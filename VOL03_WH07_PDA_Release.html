01
Overview
화면 개요
PDA-guided FIFO picking workflow for material release to production lines. Three sequential steps: (1) Scan WO barcode to load picking list with system-suggested FIFO locations (oldest stock first), (2) Scan item barcode at suggested location — wrong items are immediately blocked, and (3) Confirm picked quantity via numeric keypad. Inventory is deducted only after final confirmation. The system enforces FIFO order strictly — if the worker tries a different location, a warning is shown with the correct location code.
생산 라인 자재 출고를 위한 PDA 기반 FIFO 피킹 흐름입니다. 3단계: (1) WO 바코드 스캔으로 시스템이 제안한 FIFO 위치 목록(최고령 재고 우선) 로드, (2) 제안 위치에서 품목 바코드 스캔 — 오품목 즉시 차단, (3) 숫자패드로 피킹 수량 확인. 최종 확정 후에만 재고 차감. FIFO 순서를 엄격히 강제하며, 다른 위치 시도 시 경고와 함께 올바른 위치 코드를 안내합니다.

04
Event & Action Flow
이벤트 · 액션 흐름
1
Userlaunch · from WH-06 [Start Picking]
PDA launches with picking_id context. Loads WO + material list + FIFO-suggested locations.
WH-06 [Start Picking] 클릭으로 PDA 실행. WO·자재 목록·FIFO 권장 위치 로드.
CONTEXTpicking_id, wo_no, materials[] (with FIFO locations).
2
PDA Scantrigger · WO barcode
Worker scans WO barcode label on the work order travel sheet to confirm picking session.
작업자가 워크오더 종이의 WO 바코드를 스캔하여 피킹 세션 확정.
VALIDATE Scanned WO == session WO · else block.
3
Systemcompute · FIFO route
System calculates pick route: for each required material, finds locations with stock and orders by inbound_date ASC (oldest first). Displays first FIFO location.
자재별로 재고 보유 위치를 입고일 오름차순(최고령 우선)으로 정렬하여 피킹 경로 산출. 첫 FIFO 위치 표시.
QUERYSELECT location, on_hand, MIN(inbound_date) FROM wh_inventory ORDER BY inbound_date ASC.
4
PDA Scantrigger · location QR
Worker walks to suggested location and scans the shelf QR code. System validates it matches the FIFO suggestion.
작업자가 권장 위치로 이동 후 선반 QR 스캔. 시스템이 FIFO 일치 여부 검증.
VALIDATE Scanned == suggested · else E-WH07-001 (FIFO violation).
5
PDA Scantrigger · item barcode
Worker scans the item barcode at the location. System verifies it matches the required item code for this WO.
작업자가 위치의 품목 바코드 스캔. 시스템이 WO 필요 품목과 일치 여부 검증.
VALIDATEscanned == required_item · else E-WH07-002 (wrong item).
6
Usertap · numpad / [All Required]
Worker enters picked qty. System checks vs on_hand at location and vs required qty.
수량 입력. 위치 현재고 및 필요량과 대조.
CHECK qty ≤ location.on_hand · if < required, partial mode → re-route to next FIFO location.
7
Usertap · [Confirm Release]
Worker confirms. Backend commits transaction atomically: deduct inventory at location, insert release row, log transaction, increment WO picked qty.
확정. 백엔드 트랜잭션 원자적 커밋: 재고 차감, 출고 행 INSERT, 거래 로그, WO 피킹량 증가.
APIPOST /api/wh/release/confirm · BEGIN TRAN ... COMMIT.
✓
Systemcomplete · loop or finish
If more materials in picking list, loop to step 4 with next material. When all picked, WO status = Released, line dashboard notified, return to WH-06.
자재가 더 있으면 step 4로 루프. 전부 완료 시 WO Released 처리, 라인 대시보드 알림, WH-06 복귀.
DOWNSTREAM PP-04 WO ready signal · INJ/IMG/PNT line · WH-03/04 update · WH-08 log.
05
Exception & Error Handling
예외 · 에러 처리
E-WH07-001
FIFO ViolationFIFO 위반
Cause
Worker scans location with newer stock instead of FIFO-suggested oldest. FIFO 권장 위치가 아닌 다른 위치 스캔.
Detect
Step ② location scan comparison.
Action
Red full-screen + vibration + beep. Show suggested location + age difference. [Re-scan] button.
Recovery
Re-scan correct FIFO location · OR supervisor PIN override (audit logged).
⛔ FIFO VIOLATION — SCAN OLDEST
E-WH07-002
Wrong Item Scanned오품목 스캔
Cause
Scanned item ≠ required item for this WO. 스캔 품목이 WO 필요 품목과 다름.
Detect
Step ② item scan validation.
Action
Block. Red screen + beep + vibration. Required item code shown prominently.
Recovery
Re-scan correct item. If physical mislabel suspected, escalate to QC-04.
⛔ WRONG ITEM — RE-SCAN
E-WH07-003
Insufficient at Location위치 재고 부족
Cause
Location on_hand < required (other WO already consumed). 다른 WO가 먼저 소비하여 부족.
Detect
Step ③ qty entry validation.
Action
Allow partial pick of available qty. Auto-route to next FIFO location for remainder.
Recovery
Continue picking at suggested next location.
⚠ PARTIAL — CONTINUE NEXT LOC
E-WH07-004
Supervisor Override슈퍼바이저 예외
Cause
FIFO violation accepted via supervisor PIN. 슈퍼바이저 PIN으로 FIFO 예외.
Detect
Override applied at Step ② re-scan.
Action
Allow pick to proceed. Mandatory reason text input. Logged with supervisor ID, timestamp, original/actual location.
Recovery
N/A. Audit reviewed weekly via RPT-04.
⚠ OVERRIDE LOGGED — AUDIT
E-WH07-005
Network Disconnected네트워크 단절
Cause
WiFi signal lost during picking. 피킹 중 WiFi 단절.
Detect
Heartbeat timeout 30s.
Action
Queue confirmation locally (4 hr buffer). Yellow banner. Pick can continue offline; sync on reconnect.
Recovery
On reconnect, queued releases commit in FIFO order. Conflict resolution if inventory changed meanwhile.
⚠ OFFLINE — QUEUE MODE
E-WH07-006
WO Cancelled During Picking피킹 중 WO 취소
Cause
PP-04 cancels WO while worker is picking. 피킹 진행 중 WO 취소.
Detect
WO status push event from PP module.
Action
Pause picking. Modal: "WO cancelled. Return items already picked to original locations?" Auto-create return adjustment ticket.
Recovery
Worker returns items, system creates compensating release-back transactions.
⚠ WO CANCELLED — RETURN ITEMS

07
Applicable Business Rules
적용 업무 규칙
Rule IDDescriptionAction
BR-WH-04
All outbound picking must follow FIFO based on inbound date. System auto-suggests oldest stock location; deviations require supervisor PIN override with mandatory reason.
출고 피킹은 입고일 기준 FIFO 자동 적용. 권장 위치 외 피킹은 슈퍼바이저 PIN + 사유 입력 필수.
Auto + Override
BR-WH-15
Items scanned during picking must match the required item_code for the WO. Wrong item is immediately blocked with no override option.
피킹 중 스캔 품목은 WO 필요 품목과 일치해야 함. 오품목 즉시 차단 (예외 없음).
Block
BR-WH-16
FIFO override events are logged and reviewed weekly via RPT-04. More than 5 overrides in 30 days by a single supervisor triggers compliance review.
FIFO 예외 이벤트는 RPT-04에서 주간 검토. 30일 내 1인 5건 초과 시 컴플라이언스 리뷰 트리거.
