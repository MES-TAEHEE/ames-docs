01Overview개요
EN · Description MD_LineTimePattern gives every production line its own minute-level time pattern rather than relying on a single shared factory calendar. A pattern is scoped to a line and a day type, and decomposes a 1,440-minute day into ordered segments (MD_LineTimeSegment) — each segment is either OPERATING (가동) or PLANNED_DOWN (계획 비가동, carrying a reason code for PM, changeover or breaks). The pattern is the baseline against which PP-ODM compares real-time operating state and against which actual / unplanned downtime is measured.
KO · 설명 MD_LineTimePattern은 단일 공용 공장 캘린더에 의존하지 않고 각 생산 라인에 고유한 분단위 시간패턴을 부여한다. 패턴은 라인·요일유형 단위로 정의되며, 1,440분의 하루를 순서가 있는 세그먼트(MD_LineTimeSegment)로 분해한다 — 각 세그먼트는 OPERATING(가동) 또는 PLANNED_DOWN(계획 비가동 — PM·금형교체·휴게 사유코드 보유) 중 하나다. 이 패턴은 PP-ODM이 실시간 가동 상태를 비교하는 기준선이며, 실제/비계획 비가동을 측정하는 기준이 된다.

⚙ Minute Precision · 분단위 정밀도StartMin·EndMin을 자정 기준 분(0~1440) 정수로 저장하여 라인별로 시(時)가 아닌 분(分) 단위 시간창을 설정할 수 있다. 한 패턴의 세그먼트는 빈틈·중첩 없이 0~1440을 완전히 덮어야 한다(BR-MD-33). · Storing minute-of-day integers gives true minute-level windows; segments must tile the full day.

⚙ Full-day Tiling · 하루 완전 분할BR-MD-33 — 한 패턴의 세그먼트 집합은 StartMin=0 에서 EndMin=1440 까지 빈틈(gap)·중첩(overlap) 없이 연속해야 한다. 저장 시 트리거가 검증하며, 위반 시 ACTIVE 전환이 차단된다. · A pattern's segments must tile 0–1440 with no gap or overlap.

05Validation & Business Rules검증 + 업무 규칙
RuleDescriptionAction
BR-MD-33
A pattern's segments must tile 0–1440 minutes with no gap or overlap.
패턴 세그먼트는 0~1440분을 빈틈·중첩 없이 분할해야 함.
Block
BR-MD-34
A PLANNED_DOWN segment must carry a ReasonCode of ReasonType=DOWNTIME with PlannedFlag=TRUE.
PLANNED_DOWN 세그먼트는 DOWNTIME·PlannedFlag=TRUE 사유코드 필수.
Block
BR-MD-35
Only one ACTIVE pattern may exist per (LineID, DayType) on any given date.
(라인, 요일유형)당 특정 일자에 ACTIVE 패턴 1건만 허용.
Block
BR-MD-36
TotalOperatingMin / TotalPlannedDownMin are auto-summed from segments on save.
가동·계획비가동 합계는 저장 시 세그먼트에서 자동 집계.
Auto
BR-MD-37
An ACTIVE pattern is consumed by PP-LSB capacity planning and PP-ODM as the operating baseline.
ACTIVE 패턴은 PP-LSB 캐파 계획·PP-ODM 가동 기준선으로 사용.
Auto
VR-LTP-01
Pattern create / activate requires approval workflow + SYS-05 audit log.
패턴 생성·활성화는 승인 워크플로 + SYS-05 감사 로그.
Audit
