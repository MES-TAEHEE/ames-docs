01Overview개요
EN · Description The Audit Log is the tamper-proof record of every consequential action across all A-MES modules. Each entry captures who (user), what (action), where (module/screen), when (timestamp), and the before/after values of changed data. Logs are written automatically — no module bypasses it — and are retained for 5 years and can never be edited or deleted, not even by an Admin. The screen lets auditors filter by user, module, action type, and date range, view the field-level diff of any change, and export results for compliance and customer audits.
KO · 설명 감사 로그는 전 A-MES 모듈의 모든 중요 행위에 대한 위변조 불가 기록이다. 각 항목은 누가(사용자), 무엇을(행위), 어디서(모듈/화면), 언제(타임스탬프), 변경 데이터의 전후 값을 담는다. 로그는 자동 기록되며 — 어떤 모듈도 우회하지 못한다 — 5년간 보존되고 Admin조차 수정·삭제할 수 없다. 화면은 감사자가 사용자·모듈·행위 유형·기간으로 필터링하고, 변경의 필드 단위 diff를 보고, 컴플라이언스·고객 감사를 위해 결과를 출력하게 한다.
📌 Write-Once · 1회 기록감사 로그는 append-only(추가 전용)다 — 한 번 기록된 항목은 변경·삭제 경로 자체가 존재하지 않는다. 이것이 감사 증거로서의 신뢰성을 보장한다. · The audit log is append-only — there is no edit or delete path at all, which is what makes it trustworthy evidence.

03Logged Action Types기록 대상 행위 유형
Action TypeWhat Is Logged · 기록 내용
CREATE신규 레코드 생성 — 생성된 전체 값 기록 (계정·계획·WO·마스터 등).
UPDATE레코드 수정 — 변경된 필드의 전/후 값 모두 기록.
DELETE레코드 삭제·비활성화 — 삭제 직전 전체 값 기록.
APPROVE승인 행위 (생산계획·QC 판정·WO 종결 등) — 승인자·대상 기록.
LOGIN / LOGOUT로그인 성공·실패·잠금·로그아웃 — 보안 이벤트.
EXPORT데이터 출력 (Excel/PDF) — 무엇을 누가 반출했는지 추적.
CONFIG시스템 설정·권한·캘린더 변경 — 운영 정책 변경 추적.
⚙ Universal Coverage · 전 모듈 적용감사 로그는 특정 화면이 아니라 데이터 계층에서 작동한다 — 모든 모듈의 CRUD·승인이 자동 기록되며 개발자가 누락할 수 없는 구조다. · The audit log operates at the data layer, so every module's CRUD/approve is captured automatically.
04Workflow작업 흐름
A user performs any consequential action in any module (create, edit, approve, etc.).
사용자가 어떤 모듈에서든 중요 행위(생성·수정·승인 등)를 수행한다.
The data layer auto-writes an audit entry with actor, action, target, and before/after values.
데이터 계층이 주체·행위·대상·전후값을 담은 감사 항목을 자동 기록한다.
The entry is append-only — no edit or delete path exists for any role.
항목은 추가 전용 — 어떤 역할에도 수정·삭제 경로가 없다.
An auditor opens SYS-05, filters by user/module/action/date, reviews the trail.
감사자가 SYS-05를 열고 사용자·모듈·행위·기간으로 필터링해 추적 로그를 검토한다.
A row click shows the field-level before/after diff; results are exported for audits.
행 클릭 시 필드 단위 전/후 diff 표시 — 감사용으로 결과를 출력한다.
After 5 years, entries are archived to cold storage but never deleted.
5년 경과 항목은 콜드 스토리지로 아카이브되나 삭제되지 않는다.
05Business Rules업무 규칙
RuleDescriptionAction
BR-SYS-20
Every create/update/delete/approve action across all modules is auto-logged with before/after values.
전 모듈 생성·수정·삭제·승인은 전후값과 함께 자동 기록.
Always
BR-SYS-21
Audit log entries are append-only — they can never be edited or deleted, not even by Admin.
감사 로그는 추가 전용 — Admin조차 수정·삭제 불가.
Immutable
BR-SYS-22
Audit logs are retained for a minimum of 5 years; aged entries are archived, not purged.
감사 로그는 최소 5년 보존 — 경과분은 아카이브, 삭제 안 함.
Retain
BR-SYS-23
Audit log access is restricted to Admin/Auditor; viewing and exporting the log is itself logged.
감사 로그 접근은 Admin/감사자 한정 — 조회·출력 행위도 기록.
Restrict
