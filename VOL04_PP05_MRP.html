01
Overview
화면 개요 · 목적 · 컨텍스트
Calculates material requirements by exploding BOM for each open WO and comparing against current stock and on-order quantities. Shortage = Required − Stock − OnOrder. Shortages over zero block the affected WOs (IsBlocked=1) and surface for PR generation in PP-06. Auto-runs nightly at 00:00; can be manually triggered for partial recalculation.
각 활성 WO의 BOM 분해 → 현재고·발주중 수량과 대조하여 자재 소요량 계산. 부족량 = 소요 − 재고 − 발주중. 부족 발생 시 해당 WO 차단(IsBlocked=1)·PP-06 PR 생성 대상 목록 표시. 매일 00:00 자동 실행, 수동 부분 재실행 지원.

04
Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
SCHEDULED cron · 00:00 daily MRP
Auto-runs nightly at 00:00 KST. Iterates all open WOs and explodes BOM via recursive query.
매일 00:00 KST 자동 실행. 활성 WO 전체에 대해 BOM 재귀 분해.
EFFECTBR-PP-005 · runtime 5-15 min · result in PP_MRPLog
2
USER click · [▶ Run Now]
Manual MRP trigger. Useful after PP-03 plan changes or PP-02 import.
수동 MRP. PP-03 계획 변경 또는 PP-02 import 후 사용.
APIPOST /api/pp/mrp/run · returns job_id, polling for status
3
SYSTEM calc · BOM explosion
For each WO, recursively expand BOM to leaf-level materials. Apply WO.Qty × BOM.ChildQty.
WO별 BOM을 leaf까지 재귀 분해. WO.Qty × BOM.ChildQty로 소요량 누적.
DBRecursive CTE on MD_BOM
4
SYSTEM calc · shortage
Stock = WH_Inventory.AvailableQty (OnHand - Reserved). OnOrder = WH_PurchaseOrder.pending. Shortage = Required - Stock - OnOrder.
Stock = WH_Inventory.AvailableQty (OnHand - Reserved). OnOrder = WH_PurchaseOrder 미입고. 부족 = 소요 - 재고 - 발주중.
OUTPUTIf > 0 → WO.IsBlocked=1 + insert PP_MRPLog
5
SYSTEM block · affected WOs
WOs with material shortage are auto-blocked. PLAN team notified.
부족 자재 보유 WO 자동 차단. PLAN팀에 알림.
EFFECTBR-PP-002 · WO.Status unchanged but IsBlocked=1
6
USER click · [+ PR 생성]
From shortage row, clicks PR. Pre-fills PP-06 form with item, shortage qty, required date.
부족 행에서 [PR 생성] 클릭 시 PP-06 폼에 자재·부족량·필요일 사전 입력.
NAV/pp/pr?item=...&qty=...&due=...
7
SYSTEM complete · ready for PR
MRP cycle complete. Shortages visible. PR generation ready in PP-06.
MRP 사이클 완료. 부족 자재 표시. PP-06에서 PR 생성 가능.
DOWNSTREAMPP-06 (PR), PP-04 (WO unblock after PO arrival)

Exception Handling
예외 처리 · 오류 코드 · 복구 절차
E-PP05-001
BOM Not FoundBOM 미정의
CauseItem in WO has no BOM in MD-02.
DetectBOM lookup.
ActionSkip item, log error. WO marked partial calc.
RecoverMD team defines BOM.
⛔ NO BOM · SEE MD-02
E-PP05-002
Circular BOM ReferenceBOM 순환참조
CauseBOM contains cycle (A→B→A).
DetectCycle detection.
ActionHalt calculation, alert MD team.
RecoverMD team fixes BOM.
⛔ CYCLE DETECTED
E-PP05-003
Long Runtime장시간 실행
CauseMRP run exceeds 15 min.
DetectRuntime monitor.
ActionContinue but flag. Investigate WO count or BOM depth.
RecoverOptimize: archive Closed WOs.
⚠ SLOW MRP
E-PP05-004
Database LockDB 잠금
CauseConcurrent inventory update during MRP.
DetectLock timeout 60s.
ActionAbort run, retry once. Then alert.
RecoverCheck WH transaction load.
⛔ DB LOCK · RETRY
E-PP05-005
Stale Inventory재고 데이터 노후화
CauseWH_Inventory.last_updated > 1 hour ago.
DetectPre-run staleness check.
ActionYellow warning. Continue with current data.
RecoverCheck WH module health.
⚠ STALE STOCK
E-PP05-006
Massive Shortage대규모 부족
CauseShortage count exceeds 50 materials in single run.
DetectPost-run summary.
ActionAlert plant director. Likely supply chain issue.
RecoverConvene S&OP meeting.
⚠ MASS SHORTAGE
