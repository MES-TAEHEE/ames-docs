PP-01
Customer Forecast Management
고객사 Forecast 수신 및 관리
🖥 Web PC 1st Dev
📋 L3 Detail →
EN · Description
Receives and manages customer demand forecasts (M+1 to M+6) from SAV/GEO SCM Portal. Downloads Forecast Excel from the customer SCM portal and imports it into A-MES. Tracks gaps between forecast and confirmed orders.
KO · 설명
고객사(SAV/GEO) SCM 포탈에서 Forecast Excel을 다운로드하여 A-MES에 Import한다. Forecast 대비 확정 수주 편차를 추적한다.
PP-02
SAP B1 Supply Plan Import
SAP B1 공급계획 수신
🖥 Web PC ⇄ SAP B1
📋 L3 Detail →
EN · Description
Receives confirmed supply plans from SAP Business One. Current method: SAP B1 Excel Export → A-MES manual import. Future roadmap: SAP B1 Service Layer REST API auto-polling every 1 hour.
KO · 설명
SAP Business One의 확정 공급계획(판매오더)을 A-MES로 수신한다. 현행: SAP B1 Excel Export → 수동 Import. 향후: SAP B1 Service Layer REST API 자동 폴링(1시간)으로 전환 예정.
PP-03
Production Planning & Confirmation
생산계획 수립 및 확인
🖥 Web PC 1st Dev
📋 L3 Detail →
EN · Description
PLAN manager reviews the received supply plans against current FG inventory, equipment load, and Forecast reference. Confirms feasible items for WO batch creation.
KO · 설명
PLAN 담당자가 공급계획을 완제품 재고·설비 부하·Forecast 참고치와 대조하여 생산 가능 여부를 검토하고 WO 일괄 생성 대상을 확정한다.

Manages Work Orders from Draft through Released, In Progress, to Closed. Displays routing type badge (A/B), progress bar, DDay indicator, and SAP B1 supply plan link.

WO를 Draft → Released → In Progress → Closed 상태로 관리한다. 라우팅 A/B 배지·진행률·납기 D-Day·SAP B1 공급계획 연결을 표시한다.

PP-05
MRP — Material Requirements Planning
자재 소요 계획
🖥 Web PC ⏱ Batch
📋 L3 Detail →
EN · Description
Runs MRP to calculate material shortage quantities. A-MES MRP handles real-time WO blocking. SAP B1 MRP handles mid/long-term procurement planning and PO generation.
KO · 설명
A-MES MRP는 실시간 WO 차단·가용 여부 판단. SAP B1 MRP는 중장기 자재 소요·공급업체 발주 담당. 두 MRP를 역할 분리하여 병행 운영한다.
PP-06
Purchase Requisition → SAP B1
자재발주 요청서 (PR) → SAP B1 전송
🖥 Web PC ⇄ SAP B1
📋 L3 Detail →
EN · Description
Generates Purchase Requisition records from MRP shortage results and transmits them to SAP B1 via Service Layer API (POST /PurchaseRequests). Tracks SAP B1 DocNum after successful transmission.
KO · 설명
MRP 부족 자재를 기반으로 발주요청서(PR)를 생성하고, SAP B1 Service Layer API(POST /PurchaseRequests)로 전송한다. 전송 성공 시 SAP B1 DocNum을 수신하여 저장한다.
PP-07
Work Order Release
WO 발행 및 생산 착수
🖥 Web PC Phase 0 + 1st
📋 L3 Detail →
EN · Description
Validates all release conditions before transitioning WO to Released status: Phase-0 complete, MRP no shortage, equipment available, delivery lead time sufficient.
KO · 설명
WO Released 전환 전 4가지 조건을 순서대로 검증한다: Phase-0 완료, MRP 자재 충족, 설비 가용, 납기 여유. 모두 통과 시 WO 발행 → 자재 자동 예약 → POP 화면 노출.
