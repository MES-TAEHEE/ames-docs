01Overview개요
EN · Description Customer return processing flow. When a customer (SAV/GEO) flags defective or surplus parts, Sales raises an RMA (Return Merchandise Authorization) which auto-feeds FG-RTN. On physical receipt, FG Operator scans pallets, photo-documents condition, and links return to original SO/DN. Items enter a special Returns staging zone (Status=RETURNED). QC reviews each batch and decides disposition: RESTOCK (recovered, after rework if needed), REWORK (repair workflow), SCRAP (write-off), or CLAIM (customer-side liability, no internal action). NCR is auto-issued for tracking + CAPA evaluation.
KO · 설명 고객 반품 처리 흐름. 고객(SAV/GEO)이 불량/과잉 부품 신고 시 Sales가 RMA(Return Merchandise Authorization) 발행 → FG-RTN 자동 큐 적재. 물리 수령 시 FG 작업자가 팔레트 스캔, 상태 사진 기록, 원본 SO/DN 연결. 반품 staging zone에 입고 (Status=RETURNED). QC가 배치별 검토 후 Disposition 결정: RESTOCK(회수, 필요 시 rework), REWORK(수리), SCRAP(폐기), CLAIM(고객 측 책임, 내부 조치 없음). 추적 및 CAPA 평가용 NCR 자동 발행.

03Disposition Options처분 옵션
DispositionCriteriaDownstream Effect
✅ RESTOCK외관·기능 양호, 원본 LOT 추적 가능, 고객 반품 사유가 과잉/오발주 등 품질 외FG-02 재고 +Qty (Status=AVAILABLE) · 신규 LOT 부여 (RTN-... 접미사)

🗑 SCRAP복구 불가 손상, 안전 우려, 비용 대비 효율 낮음FG_Stock에서 영구 차감 · Scrap LOT 이관 · 회계 손실 처리

04Workflow작업 흐름
Customer files claim → Sales reviews → issues RMA with reason + expected qty + return-by date.
고객 클레임 접수 → Sales 검토 → RMA 발행 (사유·예상 수량·반품 기한).
RMA auto-creates FG-RTN record in RMA state; FG receiving dock notified.
RMA가 FG-RTN 레코드 자동 생성, RMA 상태, FG 수령 dock 알림.
Truck arrives → FG Operator scans pallets on PDA + photos condition + links original DN.
차량 도착 → FG 작업자 PDA 팔레트 스캔 + 상태 사진 + 원본 DN 연결.
RECEIVED → items in Returns staging zone with Status=RETURNED in FG-02.
RECEIVED → 반품 staging zone 입고, FG-02 Status=RETURNED.
QC reviews each pallet, references photos/condition, drills into original QC-03 inspection for context.
QC가 팔레트별 검토, 사진/상태 참조, 원본 QC-03 검사 이력 드릴.
QC Lead+ selects per-pallet disposition (mixed OK) → PIN approval per pallet.
QC Lead+가 팔레트별 처분 선택(혼합 가능), 팔레트별 PIN 승인.
Auto-trigger downstream: RESTOCK → FG-02 stock+; REWORK → PP WO; SCRAP → write-off; CLAIM → Sales/Carrier billing.
하류 자동: RESTOCK→FG-02 입고, REWORK→PP WO, SCRAP→재고차감, CLAIM→Sales/운송사 청구.
QC-04 NCR auto-issued; 3 RTNs/30d for same defect → CAPA recommended (BR-QC-03).
QC-04 NCR 자동 발행, 30일 동일 사유 3건 시 CAPA 권고.
All dispositions complete → CLOSED; 5y retention; FG-09 Return Rate updated.
모든 처분 완료 → CLOSED, 5년 보관, FG-09 Return Rate 갱신.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-48
Returns without RMA are blocked at receiving dock; Sales must issue RMA first.
RMA 없는 반품 수령 차단, Sales가 RMA 선발행.
Block
BR-FG-49
Receiving requires PDA scan + ≥2 photos per pallet + condition note; missing blocks RECEIVED.
수령 시 PDA 스캔 + 팔레트당 사진 ≥2장 + 상태 노트 필수.
Block
BR-FG-50
Per-pallet disposition allowed (mixed OK); each requires QC Lead+ PIN approval.
팔레트별 차등 처분 가능, 각각 QC Lead+ PIN 승인.
Approve
BR-FG-51
RESTOCK creates new LOT with RTN- suffix; old LOT remains for traceability link.
RESTOCK 시 RTN- 접미사 신규 LOT, 원본 LOT 추적 유지.
Auto
BR-FG-52
Return + QC-04 NCR linked; 3 returns/30d for same defect → CAPA recommended.
반품과 NCR 연계, 30일 동일 사유 3건 시 CAPA 권고.
Alert
BR-FG-53
RMA expires after 30 days if not received; Sales must re-issue.
RMA 30일 미수령 시 만료, Sales 재발행 필요.
Expire
BR-FG-54
All return records retained ≥ 5 years for audit and recurrence analysis.
반품 기록 5년 이상 보관 (감사·재발 분석).
Retain
