01Overview개요
EN · Description The core outbound screen. FG Picker receives a Pickslip from FG-03 on PDA, walks the optimized route (Slotting algorithm), scans each location QR + pallet label per allocation. System enforces FIFO: scanning a newer LOT when an older one exists for the same item is blocked at scan time (BR-FG-17). Partial picks allowed (e.g., out-of-stock cell), with auto-reallocation to next FIFO LOT. Pick completion releases Reservation → SHIPPED-ready state; Order Status → PICKED.
KO · 설명 핵심 출고 화면. FG 작업자가 PDA로 FG-03 Pickslip 수령, 최적 동선(Slotting 알고리즘) 따라 이동, 할당별 위치 QR + 팔레트 라벨 스캔. 시스템이 FIFO 강제: 동일 품목에 더 오래된 LOT이 있는데 신규 LOT 스캔 시 즉시 차단 (BR-FG-17). 부분 피킹 가능(빈 셀 등), 자동 다음 FIFO LOT 재할당. 피킹 완료 시 Reservation 해제 → SHIPPED 대기, Order Status PICKED.

03FIFO EnforcementFIFO 강제 메커니즘
Check PointAction
Pickslip GenerationFG-03 확정 시 FIFO Allocator가 가장 오래된 LOT부터 차례로 할당. Pickslip에는 정확한 LOT/Loc 명시.
Scan TimePDA가 스캔된 LOT의 StockTS를 조회 → 동일 ItemNo+CustomerCode에 더 오래된 Available LOT 존재 시 BLOCK + alert.
Mid-Pick새 FG-01 Stocking으로 더 오래된 LOT이 출현하지 않음 (StockTS는 단조 증가). 진행 중 Pickslip은 영향 없음.
Partial Pick특정 셀 부족·파손 시 시스템이 자동 다음 FIFO LOT을 재할당, Pickslip 자동 업데이트 + Reservation 보정.
Manager OverrideFG Manager+ PIN 입력 + 사유 (예: "고객 긴급 요청, 신규 LOT 우선") → 허용. OverrideCount + 사유 기록. 월간 audit 검토 의무.
04Workflow작업 흐름
Picker opens FG-04 on PDA → sees available Pickslips queue (priority: ship-date proximity).
작업자가 PDA로 FG-04 오픈, 사용 가능 Pickslip 큐 표시(우선순위: 출하일 임박).
Selects Pickslip → optimal route shown (Slotting algorithm minimizes walk distance).
Pickslip 선택, 최적 동선 표시 (Slotting 알고리즘이 이동 거리 최소화).
Walks to first location → scans Location QR → confirmed; then scans each pallet label.
첫 위치로 이동, Location QR 스캔 확정, 팔레트 라벨 순차 스캔.
If scan triggers FIFO violation → PDA blocks + directs to correct location; Override requires Manager PIN.
FIFO 위반 시 PDA 차단 + 올바른 위치 안내, Override는 Manager PIN.
Partial pick (short/damaged): tap "Short/Damaged" → system auto-reallocates next FIFO LOT.
부분 피킹 시 [Short/Damaged] 터치, 시스템이 자동 다음 FIFO LOT 재할당.
Continues to next location; PDA tracks running total vs ordered qty.
다음 위치 이동, PDA가 실시간 누적 / 주문 수량 추적.
On completion: Status → COMPLETE; Reservation released; SO Status → PICKED; FG-05 enabled.
완료 시 Status COMPLETE, Reservation 해제, SO PICKED, FG-05 활성화.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-17
FIFO mandatory: scanning newer LOT when older one exists for same item+customer is blocked at scan time.
FIFO 필수, 스캔 시 강제. 신규 LOT 스캔 차단.
Block
BR-FG-18
FIFO override requires FG Manager+ PIN + Reason; all overrides logged for monthly audit.
Override는 Manager+ PIN + 사유, 월간 audit 의무.
Approve
BR-FG-19
Partial pick auto-reallocates next FIFO LOT; cannot skip allocation manually.
부분 피킹 시 시스템 자동 재할당, 수동 스킵 불가.
Auto
BR-FG-20
Held stock (QC-05) cannot be picked even if mistakenly in Pickslip; physical scan blocks.
QC-05 Hold 재고는 Pickslip에 있어도 피킹 불가, 물리 스캔 차단.
Block
BR-FG-21
Pause > 30 min auto-releases Pickslip lock; another picker can take over.
30분 이상 일시정지 시 Pickslip 락 해제, 타 작업자 인계 가능.
Release
