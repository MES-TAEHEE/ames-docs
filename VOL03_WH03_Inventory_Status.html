01
Overview
화면 개요
Real-time inventory dashboard showing stock by item, material type (Raw Material / Fabric / Paint), and location. Items below the minimum stock level (from MD-09 · Min/Max Stock Master) are highlighted in red and surface in the KPI strip. The screen supports Excel export of the filtered view and shows daily snapshot delta vs. the previous day's closing inventory, so staff can quickly spot abnormal consumption or replenishment needs.
품목·유형(원자재/원단/도료)·로케이션별 실시간 재고 현황 대시보드입니다. 최소재고(MD-09 · 최소·최대재고 마스터) 미달 항목은 빨간색으로 강조되고 KPI에 별도 집계됩니다. 필터링된 결과의 엑셀 출력을 지원하며, 전일 마감 재고 대비 증감을 표시하여 비정상 소비·보충 필요 항목을 빠르게 식별할 수 있습니다.

04
Event & Action Flow
이벤트 · 액션 흐름
1
Userpage.load · WH-03 진입
User opens WH-03. Default filter (All types, all warehouses) applied. WebSocket subscribes to inventory updates.
WH-03 진입 시 기본 필터 적용 + WebSocket 재고 업데이트 구독.
APIGET /api/wh/inventory + WS /ws/inventory.subscribe.
2
Systempush · inventory.updated
When WH-02 inbound or WH-07 release commits, server pushes inventory delta to all subscribed clients. Row updates in place with brief highlight.
WH-02 입고·WH-07 출고 커밋 시 서버가 재고 변경분을 구독 클라이언트에 푸시. 해당 행이 잠시 하이라이트되며 갱신.
PAYLOAD{item_code, location_id, on_hand_qty, delta, timestamp}.
3
Userclick · KPI [Below Min]
User clicks a KPI card. System auto-applies the corresponding filter and scrolls to the data table.
KPI 카드 클릭 시 해당 필터 자동 적용 + 데이터 표로 스크롤.
UI Status filter set, table re-rendered, deep-link URL: ?status=below_min.
4
Systemcron · 00:00 daily snapshot
Daily 00:00 KST job writes a full inventory snapshot to wh_inventory_snapshot for Δ calculation and audit.
매일 00:00 KST 배치가 전체 재고 스냅샷을 wh_inventory_snapshot에 기록 (전일 대비 계산용).
EFFECT Δ column populated next day; data retained 365 days.
5
Systemtrigger · min-stock alert
On every inventory commit, system checks if on_hand crossed below min_qty threshold. If so, generates alert + email to warehouse manager and procurement team.
재고 커밋 시마다 최소재고 임계 통과 여부 확인. 미달 시 창고관리자·구매팀에 알림+이메일 발송.
SIDE-EFFECT INSERT sys_alert, dedupe within 24h to prevent spam.
✓
Userclick · row → location
Clicking a Location cell navigates to WH-04 Location Map centered on the clicked location, with the item highlighted.
위치 셀 클릭 시 WH-04 로케이션 맵으로 이동(해당 위치 중심·품목 하이라이트).
NAV/wh/location-map?focus=WH-FAB-D-01-1&item=FAB-GY-C03.
05
Exception & Error Handling
예외 · 에러 처리
E-WH03-001
WebSocket DisconnectedWebSocket 단절
Cause
Browser idle > 5min, network issue, or server restart. 유휴·네트워크·서버 재시작.
Detect
No heartbeat received in 30 seconds.
Action
Yellow badge in toolbar: "Live updates paused." Auto-reconnect with exponential backoff (2s, 4s, 8s, 16s...).
Recovery
On reconnect, full reload triggered to sync any missed updates.
⚠ LIVE PAUSED — RECONNECTING
E-WH03-002
Snapshot Missing스냅샷 없음
Cause
Yesterday's snapshot job failed or DB cleanup removed data. 배치 실패·데이터 정리.
Detect
No row in wh_inventory_snapshot for DATE = CURRENT_DATE - 1.
Action
Δ column shows "—" with tooltip: "No snapshot available". KPIs and other columns work normally.
Recovery
SYS admin re-runs snapshot job manually from SYS-05 batch monitor.
⚠ NO SNAPSHOT — Δ UNAVAILABLE
E-WH03-003
Excel Export Overflow엑셀 출력 초과
Cause
Filtered result > 10,000 rows. 결과가 1만 행 초과.
Detect
Server-side count check before file generation.
Action
Reject. Modal: "Result exceeds 10,000 rows. Narrow filter or use RPT-02 inventory report."
Recovery
Apply more specific filter or use Reports module.
⚠ TOO MANY ROWS — NARROW FILTER
E-WH03-004
No Min/Max Defined최소·최대 미설정
Cause
Item has no row in md_minmax (master data incomplete). 기준정보 미완성.
Detect
LEFT JOIN returns NULL for min_qty.
Action
Status column shows gray "—" badge with tooltip: "Min/Max not set. Configure in MD-09."
Recovery
Master Data manager defines min/max in MD-09 → item appears in Below Min / Low Stock KPIs.
ℹ MIN/MAX NOT SET — SEE MD-09
E-WH03-005
Negative Inventory음수 재고
Cause
Race condition or data anomaly causes on_hand_qty < 0. 동시성·데이터 이상.
Detect
Row-level check on load. Should never occur (BR-WH-05 enforces).
Action
Highlight row with red border + warning icon. Auto-create incident ticket to SYS team. Send email to WH manager.
Recovery
WH-05 inventory adjustment with reason code SYSTEM_ANOMALY + supervisor approval.
⚠ NEGATIVE STOCK — INCIDENT CREATED
E-WH03-006
Empty Result조회 결과 없음
Cause
Filter combination yields zero items. 필터 조합으로 결과 없음.
Detect
API returns empty array.
Action
Empty state illustration + "No items match your filter." Reset Filter button.
Recovery
User adjusts or resets filters.
ℹ NO RESULTS — ADJUST FILTER

Applicable Business Rules
적용 업무 규칙
Rule IDDescriptionAction
BR-WH-03
When on-hand qty falls below the minimum stock level (MD-09), an alert is sent to the warehouse manager and procurement team within 5 minutes.
현재고가 최소재고(MD-09) 미만이 되면 5분 이내 창고관리자·구매팀에 알림 발송.
Alert
BR-WH-05
Inventory quantity must never be negative. Any operation that would result in negative on-hand is rejected at DB constraint level.
재고 수량은 음수가 될 수 없으며, 음수 발생 작업은 DB 제약 조건으로 거부.
Block
BR-WH-06
Daily inventory snapshot at 00:00 KST is mandatory and retained for 365 days for audit and trend analysis.
매일 00:00 KST 재고 스냅샷 필수 + 감사·추이 분석용 365일 보관.
