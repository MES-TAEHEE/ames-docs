01Overview개요
EN · Description MD_Vendor registers every external trading partner — both material suppliers and the ship-to customers SAV / GEO, distinguished by VendorType. It holds contact data, the SCM/EDI integration link, the customer OTD target rate, and payment terms. It is the very first table loaded in the Phase-0 sequence (①) because Item, Spare Parts and every order / delivery transaction reference a vendor. Vendors are deactivated, never deleted.
KO · 설명 MD_Vendor는 모든 외부 거래 파트너를 등록한다 — 자재 공급사와 납품 고객 SAV·GEO를 VendorType으로 구분한다. 연락처, SCM/EDI 연동 링크, 고객 납기 목표율, 결제 조건을 보유한다. Phase-0 순서에서 가장 먼저(①) 입력되는 테이블이다 — 품목·정비자재·전 발주/출하 트랜잭션이 거래선을 참조하기 때문이다. 거래선은 비활성화하며 삭제하지 않는다.

05Validation & Business Rules검증 + 업무 규칙
RuleDescriptionAction
VR-Vendor-01
Only a CUSTOMER vendor may carry a ScmURL and an OtdTargetRate.
SCM URL·OTD 목표율은 CUSTOMER 거래선만 보유.
Validation
VR-Vendor-02
SAV and GEO must both exist as CUSTOMER vendors before FG / SCM go-live.
SAV·GEO는 FG/SCM 가동 전 CUSTOMER 거래선으로 등록 필수.
Block
BR-MD-04
Vendor master must be complete in Phase-0; it is loaded first (①).
거래선 마스터 Phase-0 완성 필수 — 최초 입력(①).
Block
BR-MD-23
A vendor referenced by any order / delivery cannot be deleted — set ActiveFlag = FALSE.
발주·출하 참조 거래선은 삭제 불가 — 비활성화만.
Block
BR-MD-24
Vendor create / update requires an approval workflow + SYS-05 audit log.
거래선 생성·수정은 승인 워크플로 + SYS-05 감사 로그.
Audit
