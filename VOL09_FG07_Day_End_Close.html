01Overview개요
EN · Description End-of-day FG closing process. Daily at 21:00 (configurable), system auto-collects: total stocked (FG-01), total shipped (FG-05), open orders, unshipped/over-aged stock, OTD count, returns. Manager reviews via 6-tile KPI dashboard + 5-item pre-close checklist. Mandatory items (no unverified shipments, no orphan reservations, all DNs ACK'd) must be resolved before close. On Close, an immutable Daily Snapshot is written for FG-08/FG-09 and ERP receives a settlement feed (shipped qty per customer per item).
KO · 설명 일일 FG 마감 프로세스. 매일 21:00(설정 가능) 자동 집계: 입고 합계(FG-01), 출하 합계(FG-05), 미출하 주문, 체류 재고, OTD 카운트, 반품. Manager가 6-타일 KPI + 5종 사전 체크리스트로 검토. 필수 항목(검증 미완료 출하 없음·고아 예약 없음·모든 DN ACK 수신)이 해결되어야 마감 가능. 마감 시 immutable Daily Snapshot 기록(FG-08/09 데이터 원천) + ERP에 정산 feed 송신.

03Pre-Close Checklist마감 전 필수 체크
Check ItemCondition · 조건If FAIL
1. Loadings Confirmed모든 PICKED SO가 FG-05 SHIPPED 또는 익일 미루기 표시 완료FG-05 화면으로 routing
2. No Orphan ReservationsActive SO 없이 Reserved 상태인 Stock 0건자동 해제 옵션 제공
3. DN EDI ACK금일 발행된 모든 거래명세서가 ACK 수신 또는 SLA 내재시도/Sales 알림
4. Stock-Movement ReconciliationFG_Stock 변화량 = FG-01 입고 - FG-05 출하 - 반품 + 조정차이 표시 + 조정 화면
5. No Mis-shipped Hold ItemsQC-05 HOLD 상태였던 Stock이 출하된 사례 0건위반 SO 표시 + QC 알림
04Workflow작업 흐름
21:00 cron triggers Day-End preparation: collects KPIs, runs 5-item checklist, sends notification to FG Manager.
21:00 크론이 마감 준비 시작: KPI 수집, 5종 체크리스트 실행, FG Manager 알림.
Manager opens FG-07 → reviews 6 KPI tiles + 5 checklist items + pending list.
Manager가 FG-07 오픈, 6 KPI + 5 체크리스트 + Pending 검토.
For each FAIL item: drill into source screen, resolve, re-run check.
FAIL 항목별: 소스 화면 드릴, 해결, 체크 재실행.
All checks PASS → Manager clicks Close → Daily Snapshot PDF generated, S3 immutable stored.
전 PASS 시 Close 클릭 → Daily Snapshot PDF 생성, S3 immutable 저장.
ERP settlement feed sent (shipped qty per customer per item); auto-retry 5 times on failure.
ERP 정산 feed 송신 (고객×품목별 출하량), 실패 시 5회 재시도.
FG-08 History updated with closed-day record; FG-09 Dashboard refreshes.
FG-08 이력에 마감일 기록 추가, FG-09 Dashboard 갱신.
If Manager doesn't close by 21:30 → next day 06:00 Auto-Close fires + notification.
21:30 미마감 시 익일 06:00 Auto-Close + 알림.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-33
Day-End cron fires at 21:00; Manager must close by 21:30. Auto-Close at next-day 06:00.
21:00 크론, 21:30까지 마감 의무. 미마감 시 익일 06:00 Auto-Close.
Schedule
BR-FG-34
Daily Snapshot is immutable; once Close confirmed, the day cannot be re-opened.
Daily Snapshot immutable, 재오픈 불가.
Lock
BR-FG-35
Any FAIL in pre-close checklist blocks Close; Manager must resolve OR add justification.
체크리스트 1건 FAIL 시 Close 차단, 해결 또는 사유 첨부 필수.
Block
BR-FG-36
ERP settlement feed retries 5 times with backoff; persistent fail notifies Accounting.
ERP feed 5회 재시도, 영속 실패 시 회계팀 알림.
Retry
BR-FG-37
Auto-Close (T+1 06:00) carries flag for audit; Manager required to acknowledge missed close.
Auto-Close 시 audit flag, Manager 사후 확인 의무.
Flag
