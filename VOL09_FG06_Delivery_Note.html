01Overview개요
EN · Description Customer-specific delivery document issued at FG-05 SHIPPED. Two formats supported out-of-the-box: SAV format (현대 SAV — 4-copy NCR, ASN-compatible header) and GEO format (GEO — Korean tax-invoice style + per-pallet barcode block). Auto-fills WO/Item/Qty/LOT/Seal/Driver from FG-05 record. PDF for printing (3 copies: 1 to driver, 1 file, 1 customer reply), and EDI ASN sent to customer at same TS. Issuance is one-way: re-print possible but creates new revision (rev 1, rev 2…) — auditable.
KO · 설명 FG-05 SHIPPED 시 발행하는 고객별 거래명세서. 기본 2개 양식: SAV 양식(현대 SAV — 4매 NCR + ASN 호환 헤더), GEO 양식(GEO — 한국 세금계산서 스타일 + 팔레트별 바코드 블록). FG-05 기록으로 WO/품목/수량/LOT/Seal/운전자 자동 채움. PDF 인쇄(3매: 운전자·보관·고객 반송) + EDI ASN 동시 발송. 발행은 단방향, 재인쇄 시 신규 revision 생성(rev 1, rev 2…) — 감사 가능.

04Workflow작업 흐름
FG-05 Confirm → SHIPPED → FG-06 issuance auto-triggered with FormatTemplate auto-selected by CustomerCode.
FG-05 Confirm → SHIPPED → FG-06 자동 발행, 고객 코드로 FormatTemplate 자동 선택.
PDF rendered server-side (Puppeteer/HTML→PDF), stored to S3 immutable bucket, URL returned.
서버에서 PDF 렌더(HTML→PDF), S3 immutable 버킷 저장, URL 반환.
EDI ASN message generated per customer schema; sent immediately to customer endpoint.
고객 스키마 기준 EDI ASN 메시지 생성, 고객 endpoint로 즉시 송신.
EDI send failure → auto-retry 5 times with exponential backoff; persistent failure → Sales alert.
EDI 송신 실패 시 지수 백오프 5회 재시도, 영속 실패 시 Sales 알림.
Print queue dispatches 3 copies (4 for SAV NCR) to dock printer; one given to driver.
인쇄 큐가 dock 프린터로 3매(SAV NCR은 4매) 전송, 1매는 운전자에게 전달.
Customer reply ACK received → CustomerAckTS recorded; FG-08 history updated.
고객 인수 ACK 수신 시 CustomerAckTS 기록, FG-08 이력 갱신.
Re-issue (errata): user clicks "Re-issue rev+1" → revision counter incremented, new PDF/EDI; original kept.
재발행 시 [Re-issue rev+1] 클릭 → revision +1, 신규 PDF/EDI, 원본 보존.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-28

FormatTemplate is determined by MD-08 PackSpec.CustomerFormat; cannot be manually changed.
양식은 MD-08 기준 자동, 수동 변경 불가.
Lock
BR-FG-30
PDF stored to S3 immutable bucket; revisions create new files, originals never deleted.
PDF S3 immutable 저장, revision 시 신규 파일, 원본 영구 보존.
Retain
BR-FG-31
EDI send failure → exponential backoff 5 retries; persistent failure → Sales alert.
EDI 실패 시 5회 재시도, 영속 실패 Sales 알림.
Retry
BR-FG-32
Customer ACK SLA: SAV 24h, GEO 12h; over → auto Sales notification.
고객 ACK SLA: SAV 24h, GEO 12h, 초과 시 Sales 알림.
Alert
