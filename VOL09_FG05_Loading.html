01Overview개요
EN · Description The final outbound checkpoint before the truck leaves. Loading Operator verifies the arriving truck (license plate scan or carrier API match), checks driver ID, scans each picked pallet onto the truck (3-side scan: pallet QR → SO match → truck door), captures driver e-signature on PDA, prints/issues Delivery Note (FG-06) and Seal Number. On Confirm: SO → SHIPPED, OTD evaluated against ShipDate, FG-02 stock physically deducted, all events sealed for traceability and chain-of-custody.
KO · 설명 차량 출고 전 마지막 체크포인트. Loading 작업자가 도착 차량 검증(번호판 스캔 또는 운송사 API 매칭), 운전자 ID 확인, 피킹 팔레트를 차량에 적재하며 3-side 스캔(팔레트 QR → SO 매칭 → 차량 도어), 운전자 전자서명 PDA 캡처, FG-06 거래명세서 + Seal Number 발행. Confirm 시 SO → SHIPPED 전환, ShipDate 대비 OTD 평가, FG-02 재고 물리 차감, 모든 이벤트 봉인 (이력 추적 + Chain-of-Custody).

03OTD EvaluationOTD 평가 메커니즘
OutcomeCriteria · Action
ON_TIMEConfirmTS ≤ ShipDate 23:59:59 → OTD=ON_TIME. FG-09 Dashboard에 정시 출하 카운트 +1.
LATEConfirmTS > ShipDate 23:59:59 → OTD=LATE. SO에 LATE flag 영구 기록. 고객 ACK에 지연 사유 추가, FG-09 OTD% 차감, Sales 알림.
3 LATE in Month월간 동일 고객 LATE 3회 → CAPA(QC-06) 권고 + FG Manager 에스컬레이션.
Customer-A LateSAV/GEO 같은 A등급 고객 LATE 발생 시 영업/품질 즉시 알림 + 고객 사과 메일 자동 초안 생성.
04Workflow작업 흐름
Truck arrives at gate → OCR scans license plate → carrier API match → Dock assigned.
차량 게이트 도착 → OCR 번호판 스캔 → 운송사 API 매칭 → 하역장 배정.
Driver shows ID → Loading Operator captures driver photo + verifies against carrier records.
운전자 ID 제시 → 작업자가 사진 캡처 + 운송사 기록 대조.
Truck backs into Dock → pallets queued at staging from FG-04 picking complete.
차량 하역장 진입, FG-04에서 피킹 완료된 팔레트 staging에서 대기.
Operator scans each pallet QR as loaded → system auto-matches against SO; mismatch blocks.
작업자가 팔레트 QR 적재 시 스캔, 시스템 SO 자동 매칭, 불일치 차단.
All pallets loaded → Driver signs on PDA + ID card photographed + GPS captured.
전 팔레트 적재 후 운전자 PDA 서명 + ID 사진 + GPS 캡처.
Seal applied to truck doors → photo of seal captured → Seal No recorded.
차량 도어 봉인 부착, 봉인 사진 캡처, Seal No 기록.
Confirm → SO Status SHIPPED, OTD evaluated, FG-02 stock deducted, FG-06 Delivery Note printed.
Confirm → SO SHIPPED, OTD 평가, FG-02 재고 차감, FG-06 거래명세서 인쇄.
Truck departs → DepartureTS recorded; carrier API gets ETA to destination.
차량 출발 → DepartureTS 기록, 운송사 API로 도착 ETA 전달.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-22
Unregistered truck/driver cannot load. Carrier API mismatch blocks loading at gate.
미예약 차량/운전자 적재 불가, 운송사 API 불일치 시 게이트 차단.
Block
BR-FG-23
Pallet must belong to the assigned SO; scanning a wrong-SO pallet blocks the load.
팔레트가 배정 SO에 속해야 함, 다른 SO 스캔 차단.
Block
BR-FG-24
Driver e-signature + ID photo + GPS coordinates required to Confirm; missing any blocks.
서명 + ID 사진 + GPS 모두 필수, 누락 시 Confirm 차단.
Block
BR-FG-25
Seal Number must be applied and photographed before departure; missing → loading flagged for QC.
봉인 번호 부착 + 사진 필수, 누락 시 QC 플래그.
Flag
BR-FG-26
OTD evaluated at Confirm: ON_TIME ≤ ShipDate 23:59:59, otherwise LATE; SO carries flag permanently.
OTD Confirm 시 평가, ShipDate 23:59:59 기준, LATE 영구 기록.
Auto
BR-FG-27
All loading events (scans/signature/photos/seal) retained ≥ 5 years for chain-of-custody audit.
전 적재 이벤트(스캔·서명·사진·봉인) 5년 봉인.
Retain
