01Overview개요
EN · Description Corrective and Preventive Action workflow following the PDCA (Plan-Do-Check-Act) cycle. Auto-recommended by the system when the same defect code recurs ≥ 3 times in a 30-day rolling window, or manually opened by QC Lead for high-severity NCRs. Captures 5-Why root cause analysis, immediate Corrective Action, long-term Preventive Action, responsible party, due date, and Effectiveness Verification at 30/60/90 days post-implementation. Critical for IATF 16949 audit and customer claims.
KO · 설명 PDCA(Plan-Do-Check-Act) 사이클을 따르는 시정·예방조치 워크플로우. 30일 이동 동일 불량 3회 이상 발생 시 시스템 자동 권고, 또는 고심각도 NCR에 대해 QC Lead가 수동 발의. 5-Why 근본원인 분석, 즉시 시정조치, 장기 예방조치, 담당자, 기한, 그리고 실행 후 30/60/90일 시점 효과성 검증 포함. IATF 16949 감사 및 고객 클레임 대응 핵심 산출물.

▼ 5-WHY ROOT CAUSE ANALYSIS · 근본 원인 분석Why-Because chain
PROBLEM
!
PNT-D05 미경화 (Insufficient Cure) — 30일 내 동일 불량 3회 발생.
도장 미경화 불량이 30일 동안 3차례 반복 발생.
WHY 1
→
Oven temperature dropped to 173°C for 4 minutes during cure cycle.
왜? 오븐 온도가 cure 사이클 중 173°C로 4분간 하락.
WHY 2
→
Burner gas pressure regulator drifted out of spec; PLC compensation lagged.
왜? 버너 가스 압력 레귤레이터가 사양 이탈, PLC 보정 지연.
WHY 3
→
Regulator inspection interval is 6 months, but field stress halves its true life.
왜? 레귤레이터 점검 주기 6개월이나 현장 부하로 실 수명이 절반.
WHY 4
→
MNT preventive plan based on supplier spec, not actual field data.
왜? MNT 예방계획이 공급사 사양 기준, 실제 현장 데이터 미반영.
ROOT
⛏
📌 No data feedback loop between PLC trend logs and MNT preventive plan.
근본원인: PLC 트렌드 로그와 MNT 예방계획 간 데이터 피드백 루프 부재.

04Workflow작업 흐름
Trigger: 3 NCRs (same defect, 30-day window) auto-recommend CAPA OR QC Lead opens manually for high-severity.
트리거: 30일 내 동일 불량 3건 자동 권고 또는 QC Lead가 고심각도에 수동 발의.
PLAN: Owner conducts 5-Why analysis (min 3-Why depth), identifies root cause + 4M classification.
PLAN: 담당자가 5-Why 분석 (최소 3-Why), 근본원인 + 4M 분류 식별.
Define Corrective (immediate) + Preventive (system) actions with owners and due dates; QC Manager approves.
시정(즉시) + 예방(시스템) 조치 정의, 담당자·기한 부여, QC Manager 승인.
DO: Owners execute, attach evidence (photos, MNT WOIDs, SOP IDs); status auto-tracked.
DO: 담당자 실행, 증빙 첨부 (사진/MNT WO/SOP ID), 상태 자동 추적.
CHECK: At D+30 → measure effectiveness; PASS → continue to D+60 → D+90; FAIL → return to PLAN.
CHECK: D+30 효과성 측정, PASS 시 D+60→D+90 계속, FAIL 시 PLAN으로 회귀.
ACT: Update SOP/Insp Std, deploy laterally (other lines/products), QC Manager final approval.
ACT: SOP/검사기준 갱신, 횡전개 (타 라인/제품), QC Manager 최종 결재.
Status → CLOSED, sealed for ≥ 5 years; visible in QC-07 dashboard and QC-TRC traceability.
CLOSED 전환, 5년 이상 봉인 보관, QC-07 대시보드 + QC-TRC에서 조회.
If same root cause recurs in another CAPA within 1 year → Status: REPEAT, escalated to Plant Manager.
1년 내 동일 근본원인이 다른 CAPA에서 재발하면 REPEAT 등급 → 공장장 보고.
05Business Rules업무 규칙
RuleDescriptionAction
BR-QC-03
3 NCRs (same defect code) within 30-day rolling window auto-recommend CAPA.
30일 동일 불량 3회 → CAPA 자동 권고.
Auto
BR-QC-18
CAPA must include ≥1 Corrective AND ≥1 Preventive action. Cannot proceed PLAN→DO without both.
시정·예방 각 1개 이상 필수, 미충족 시 DO 단계 진입 불가.
Block
BR-QC-19
Effectiveness verification at D+30/60/90 all must pass to CLOSE; any FAIL returns CAPA to PLAN.
D+30/60/90 효과성 모두 통과해야 CLOSED, 1건 FAIL 시 PLAN 회귀.
Verify
BR-QC-20
Same root cause recurring within 1 year → Status: REPEAT, escalated to Plant Manager.
1년 내 동일 근본원인 재발 시 REPEAT 등급 + 공장장 보고.
Escalate
BR-QC-21
Customer Impact = HIGH → CAPA report PDF auto-generated and routed to sales/customer team.
고객 영향 HIGH 시 CAPA 보고서 PDF 자동 생성 + 영업/고객팀 라우팅.
Auto
BR-QC-08
Closed CAPA records (with 5-Why, actions, evidence, effectiveness) retained ≥ 5 years.
CLOSED CAPA 기록 (5-Why·조치·증빙·효과성) 5년 이상 보관.
Retain
