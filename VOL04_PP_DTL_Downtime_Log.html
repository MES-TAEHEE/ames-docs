01
Overview
화면 개요 · 목적 · 컨텍스트
A POP touch-panel screen where the line operator records the reason for an unplanned downtime event detected by PP-ODM. The event arrives pre-filled with line, start/end minute and live duration; the operator taps a downtime reason code (MD-25, filtered to ReasonType=DOWNTIME) and an optional 4M1E cause (MD-22), enters a comment if the reason requires it, and confirms. The closed event with its reason flows to PP-ODM and PP-OEE so availability loss is correctly attributed.
PP-ODM이 감지한 실제 비가동 이벤트의 사유를 라인 작업자가 기록하는 POP 터치패널 화면. 이벤트는 라인·시작/종료 분·실시간 경과시간이 미리 채워져 도착하며, 작업자는 비가동 사유코드(MD-25, ReasonType=DOWNTIME 필터)와 선택적 4M1E 원인(MD-22)을 터치 선택하고, 사유가 요구하면 코멘트를 입력한 뒤 확정한다. 사유가 기록된 종료 이벤트는 PP-ODM·PP-OEE로 전달되어 가용성 손실이 정확히 귀속된다.

04
Event & Action Flow
이벤트 흐름 · 사유 입력 → 종료
1
SYSTEMpush · downtime event open
PP-ODM detects an unplanned downtime and pushes an open event to the line's POP panel.
PP-ODM이 실제 비가동을 감지하여 라인 POP 패널에 진행중 이벤트를 푸시.
PUSHWS event: downtime.open {lineId, startMin}
2
USERtap · reason + cause
Operator taps a DOWNTIME reason tile (MD-25) and, if needed, a 4M1E cause tile (MD-22).
작업자가 DOWNTIME 사유 타일(MD-25)과 필요 시 4M1E 원인 타일(MD-22)을 터치.
STATEreason set · cause optional
3
SYSTEMvalidate · comment requirement
If the chosen reason has RequiresComment=TRUE, the comment field is enforced before Confirm enables.
선택 사유가 코멘트 필수면 Confirm 활성화 전 코멘트 입력 강제.
RULEBR-PP-046 · MD_ReasonCode.RequiresComment
4
USERtap · [사유 확정]
Writes a PP_LineDowntimeLog row (line, start/end minute, reason, cause, comment, operator) and closes the event.
PP_LineDowntimeLog 행 기록(라인·시작/종료분·사유·원인·코멘트·작업자) → 이벤트 종료.
APIPOST /api/pp/downtime-log
5
SYSTEMcomplete · attribute loss
The reasoned downtime updates PP-ODM (event cleared), feeds PP-OEE availability loss and mirrors to MNT-06 if reason is a breakdown.
사유 기록된 비가동이 PP-ODM 갱신(이벤트 해소)·PP-OEE 가용성 손실 반영·고장 사유면 MNT-06 동기화.
DOWNSTREAMPP-ODM clear · PP-OEE recalc · MNT-06 downtime
