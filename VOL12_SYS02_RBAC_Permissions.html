01Overview개요
EN · Description The core SYS screen — the Role-Based Access Control (RBAC) permission matrix. It defines, for each of 7 roles (Admin, Production Manager, Warehouse Staff, QC Inspector, Maintenance Engineer, POP Operator, Read-Only), the access level to every module and screen: Read (R) / Write (W) / Approve (A) / All / None. This single matrix governs what every A-MES user can see and do — every screen in every module checks it on entry. Permission changes take effect immediately on save and are written to the SYS-05 audit log. The screen is Admin-only.
KO · 설명 SYS 모듈 핵심 화면 — 역할 기반 접근 제어(RBAC) 권한 매트릭스. 7개 역할(Admin·생산관리자·창고담당·QC검사원·정비기술자·POP작업자·읽기전용) 각각에 대해 전 모듈·화면 접근 수준을 정의한다: 읽기(R) / 쓰기(W) / 승인(A) / 전체 / 없음. 이 단일 매트릭스가 모든 A-MES 사용자가 무엇을 보고 할 수 있는지를 통제한다 — 전 모듈의 모든 화면이 진입 시 이를 확인한다. 권한 변경은 저장 즉시 반영되며 SYS-05 감사 로그에 기록된다. 화면은 Admin 전용이다.
⭐ Core Screen · 핵심 화면SYS-02는 A-MES 보안의 단일 통제점이다 — 전 모듈의 모든 화면이 진입 전 이 매트릭스를 조회한다. 매트릭스가 곧 시스템의 접근 정책이다. · SYS-02 is A-MES's single point of access control — every screen in every module consults this matrix before entry.

03Role Matrix7개 RBAC 역할
RoleScope · 권한 범위
Admin · 관리자전 모듈 All. SYS/MD 관리 권한. 항상 고정 — 변경 불가.
Production Manager · 생산관리자PP 생산계획 R/W/A (승인), 생산 모듈 R. 전 모듈 조회.
Warehouse Staff · 창고담당WH 입출고 R/W. PP·QC 조회. 생산·정비 접근 없음.
QC Inspector · QC검사원QC 검사·판정 R/W/A. 생산·재고 조회.
Maintenance Engineer · 정비기술자MNT 정비·WO R/W. 설비 관련 모듈 조회.
POP Operator · POP작업자배정 라인의 INJ/IMG/PNT POP만 R/W. 그 외 접근 없음.
Read-Only · 읽기전용전 모듈 R만. 입력·수정·승인 불가 (감사·견학용).
⚙ Permission Hierarchy · 권한 위계NONE < READ < WRITE < APPROVE < ALL — 상위 권한은 하위를 포함한다. APPROVE는 WRITE를, ALL은 전 권한을 포괄한다. · Higher levels include lower ones — APPROVE includes WRITE, ALL includes everything.
04Workflow작업 흐름
Admin opens SYS-02; the matrix shows all 7 roles × all modules.
Admin이 SYS-02를 연다 — 매트릭스에 7역할 × 전 모듈이 표시된다.
Admin clicks a cell and selects the permission level (R / W / A / All / None).
Admin이 셀을 클릭하고 권한 수준(R/W/A/All/—)을 선택한다.
On Save, the change takes effect immediately — affected users see the new access without re-login.
저장 시 변경 즉시 반영 — 해당 사용자는 재로그인 없이 새 권한이 적용된다.
The change (role, module, before/after level, actor) is written to the SYS-05 audit log.
변경 내역(역할·모듈·전후 수준·주체)이 SYS-05 감사 로그에 기록된다.
Every module screen consults the matrix on entry; a denied user gets an access-blocked message.
전 모듈 화면이 진입 시 매트릭스를 조회 — 권한 없는 사용자는 접근 차단 메시지.
05Business Rules업무 규칙
RuleDescriptionAction
BR-SYS-08
All module and screen access is governed by the RBAC matrix; every screen checks it on entry.
전 모듈·화면 접근은 RBAC 매트릭스 기준 — 모든 화면이 진입 시 확인.
Enforce
BR-SYS-09
Permission changes take effect immediately on save — no re-login required.
권한 변경은 저장 즉시 반영 — 재로그인 불필요.
Immediate
BR-SYS-10
The Admin role is fixed at All for every module and cannot be downgraded.
Admin 역할은 전 모듈 All 고정 — 하향 불가.
Lock
BR-SYS-11
POP Operators can access only the production lines assigned in their SYS-01 account.
POP 작업자는 SYS-01 계정에 배정된 라인만 접근 가능.
Restrict
BR-SYS-12
Every permission matrix change is logged in SYS-05 with before/after values and actor.
전 권한 매트릭스 변경은 전후값·주체와 함께 SYS-05 기록.
