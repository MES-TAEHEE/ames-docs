01
Overview
화면 개요
Lists upcoming material release demands derived from active Work Orders (WO) in the PP module. For each WO line, the system computes BOM-exploded material needs vs current on-hand and flags shortages. Warehouse staff use this screen to pre-pick materials ahead of production start (preparation status), and click [Start Picking] to launch WH-07 PDA flow. Filters by line, shift, WO number, and item. ETA is the planned WO start time minus a configurable lead (default: 30 min).
PP 모듈 활성 작업지시(WO) 기반 자재 출고 예정 목록입니다. WO별로 BOM 분해된 자재 소요량을 현재고와 대조하여 부족 자재를 표시합니다. 창고 담당자는 본 화면에서 생산 시작 전 사전 피킹을 준비하고(준비 상태), [Start Picking] 클릭 시 WH-07 PDA 흐름이 시작됩니다. 라인·교대·WO 번호·품목 필터 지원. ETA는 WO 계획 시작 시각에서 설정된 리드타임(기본 30분)을 차감한 값입니다.

04
Event & Action Flow
이벤트 · 액션 흐름
1
Scheduledcron · WO demand refresh @ 5min
Every 5 minutes, system recalculates material demand for all active WOs by joining wo_header × wo_bom_demand × wh_inventory. Updates shortage flags.
5분마다 WO·BOM·재고를 조인하여 자재 소요·부족 플래그 재계산.
EFFECT Materialized view vw_release_schedule refreshed · WebSocket push to clients.
2
Userpage.load · WH-06 진입
Warehouse staff opens screen. Default filter: today's date + all lines. Loads demand list sorted by ETA ascending.
WH-06 진입 시 기본 필터(오늘 + 전체 라인)로 ETA 오름차순 정렬 로딩.
APIGET /api/wh/release/schedule?date=today.
3
Userclick · [📋 Prepare]
User marks a WO as "preparing" — soft-reserves materials in inventory (status flag, not yet deducted). Other WOs see projected on-hand updated.
[Prepare] 클릭 시 자재를 소프트 예약(상태만 변경, 차감 아님). 다른 WO는 예상 가용량 갱신 확인.
APIPOST /api/wh/release/prepare · wo_header.prep_flag=1 · reserved_qty set.
4
Userclick · [▶ Start Picking]
User initiates picking. System checks RBAC WH.Release and material availability. Creates picking session and launches WH-07 PDA flow.
[Start Picking] 클릭 시 권한 + 자재 가용성 확인 후 피킹 세션 생성, WH-07 PDA 흐름 시작.
REDIRECTPOST /api/wh/release/start → returns picking_id → redirect /wh/pda/release?pid=....
5
Systemalert · ETA approaching
When ETA < 15 min and status is still Pending/Prep, system sends alert to warehouse staff and line supervisor.
ETA 15분 이내인데 Pending/Prep 상태인 경우 창고 담당·라인 슈퍼바이저에 알림 발송.
SIDE-EFFECT In-app push notification · email · sys_alert row inserted.
!
Systemdetect · shortage
When required material > projected on_hand, WO turns red. [Start Picking] disabled. Suggestion: trigger MRP rerun or expedite PO from procurement.
소요량 > 예상 현재고 시 WO가 빨간색으로 전환되고 [Start Picking] 비활성. MRP 재계산·긴급 발주 권고.
ALERT WO marked shortage · email to procurement + plant mgr · ref to PP-05 MRP.
✓
Systemreturn · from WH-07 success
After WH-07 picking completes, WO status = Released. Inventory deducted. Row badge updates. Production line notified that material is staged.
WH-07 피킹 완료 시 WO 상태 = Released. 재고 차감. 생산 라인에 자재 준비 완료 알림.
DOWNSTREAM WH-08 logs · WH-03 push update · PP-04 WO ready signal · INJ/IMG/PNT line dashboards.
05
Exception & Error Handling
예외 · 에러 처리
E-WH06-001
Material Shortage자재 부족
Cause
Required material qty > projected on-hand. 소요량 > 예상 가용량.
Detect
5-minute cron + real-time on WH-02/05/07 commits.
Action
Row red. [Start Picking] disabled. Email procurement + plant mgr. Link to PP-05 MRP rerun.
Recovery
Expedite PO via WH-01 [Register Schedule] or transfer stock from another location.
⛔ SHORTAGE — EXPEDITE PROCUREMENT
E-WH06-002
ETA Already PassedETA 경과
Cause
Current time > ETA but WO not yet picked. 현재 시각이 ETA를 초과했지만 미피킹.
Detect
Per-row time check on every render.
Action
Row blinks red. ETA badge "OVERDUE". Auto-escalate notification every 5 min until picked or cancelled.
Recovery
Start picking immediately, or adjust WO plan_start in PP-04.
⚠ OVERDUE — START NOW
E-WH06-003
Concurrent Picking Lock중복 피킹 시도
Cause
Another user has already started picking this WO. 동일 WO를 다른 사용자가 이미 시작.
Detect
DB row lock on picking_id already exists.
Action
Block. Show "Picking already in progress by [user] since [time]. View only?"
Recovery
Wait or coordinate with the other user.
⛔ LOCKED — IN PROGRESS
E-WH06-004
WO Cancelled UpstreamWO 상위 취소
Cause
WO was cancelled in PP-04 after warehouse already prepared. PP-04에서 WO 취소.
Detect
WO status change event from PP module.
Action
Row removed from list with notification: "WO cancelled. Return prepared materials." Soft-reservation released automatically.
Recovery
Return materials to storage. No inventory deduction needed (was only reserved).
⚠ WO CANCELLED — RETURN MATERIALS
E-WH06-005
BOM Not DefinedBOM 미정의
Cause
WO created for an item with incomplete BOM in PP module. PP BOM 미정의 품목 WO.
Detect
LEFT JOIN wo_bom_demand returns 0 rows.
Action
Material column shows "—". Row marked as "BOM ERROR" with link to MD-03 BOM Master.
Recovery
Master data team defines/completes BOM, then WH-06 row updates.
⚠ NO BOM — SEE MD-03
E-WH06-006
No Picker Assigned피커 미지정
Cause
No warehouse staff with WH.Release role is online during shift. 교대 중 출고 권한 담당자 없음.
Detect
Hourly check during active shifts.
Action
Banner: "No picker available for [shift]." Alert sent to WH manager to assign or escalate.
Recovery
Manager assigns staff or covers personally.
ℹ NO PICKER — ASSIGN STAFF

Applicable Business Rules
적용 업무 규칙
Rule IDDescriptionAction
BR-WH-04
All outbound picking must follow FIFO (First In First Out) based on inbound date. WH-07 auto-suggests the oldest stock location.
출고 피킹은 입고일 기준 FIFO 자동 적용. WH-07이 최고령 재고 위치 자동 제안.
Auto
BR-WH-13
Picking cannot start if projected on-hand < required after upstream WO allocations. [Start Picking] button is disabled until materials are sufficient.
상위 WO 할당 후 예상 가용량이 소요량 미달이면 피킹 시작 불가. 자재 확보 시까지 버튼 비활성.
Block
BR-WH-14
Concurrent picking on the same WO is blocked by row lock. Only one active picking session per WO at any time.
동일 WO 동시 피킹 차단(행 잠금). 한 WO에 동시에 한 피킹 세션만 활성.
Block
