01Overview개요
EN · Description MD_BOM defines the multi-level material structure of every semi-finished and finished good. Each row is one parent-component link — a parent item, the component item it consumes, the quantity per parent unit and a scrap factor. The BOM is a self-referencing tree against MD_Item (a component may itself be a parent at a lower level). Each row belongs to a versioned set (VersionID → MD_BOMVersion) so the effective structure as-of a date can be reconstructed. PP-05 MRP explodes this tree to compute material requirements.
KO · 설명 MD_BOM은 전 반제품·완제품의 다단계 자재 구조를 정의한다. 각 행은 하나의 부모-구성 링크 — 부모 품목, 소비하는 구성 품목, 부모 1단위당 수량, 스크랩률을 담는다. BOM은 MD_Item에 대한 자기참조 트리다 (구성 품목이 하위 단계에서 다시 부모가 될 수 있다). 각 행은 버전 집합(VersionID → MD_BOMVersion)에 속해 특정 일자 기준 유효 구조를 재구성할 수 있다. PP-05 MRP가 이 트리를 전개해 자재 소요를 산출한다.

05Validation & Business Rules검증 + 업무 규칙
RuleDescriptionAction
VR-BOM-01
BOM tree must be acyclic — an app-level traversal check rejects any circular structure on save.
BOM 트리는 비순환 — 저장 시 순회 검증으로 순환 구조 거부.
Block
VR-BOM-02
ParentItemNo must be a SEMI or FINISHED item; RAW / FABRIC / POWDER / PAINT cannot be a parent.
부모 품목은 SEMI·FINISHED만 — 원자재류는 부모 불가.
Validation
BR-MD-02
BOM must exist for an item before MRP (PP-05) can calculate material requirements.
BOM 없는 품목은 PP-05 MRP 실행 불가.
Validation
BR-MD-23
A BOM line referenced by an open WO cannot be deleted — set ActiveFlag = FALSE in a new version.
진행 WO 참조 BOM 라인은 삭제 불가 — 신규 버전에서 비활성화.
Block
BR-MD-24
BOM changes require an approval workflow + SYS-05 audit; a new MD_BOMVersion is created.
BOM 변경은 승인 워크플로 + 감사 — 신규 버전 생성.
Audit
