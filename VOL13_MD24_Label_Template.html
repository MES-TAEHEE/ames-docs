01Overview개요
EN · Description MD_LabelTemplate stores the print designs used to issue part, box, pallet and shipping labels. Each template carries a label type, paper size, a barcode type (QR / Code128 / DataMatrix / EAN13), a ZPL layout body and a JSON field-map binding each barcode/text region to an A-MES data field. A template may be customer-specific or shared (NULL customer). Templates are versioned; only the highest ACTIVE version of a key is used at print time.
KO · 설명 MD_LabelTemplate는 부품·박스·팔레트·출하 라벨 발행에 사용하는 인쇄 디자인을 저장한다. 각 템플릿은 라벨 유형, 용지 규격, 바코드 유형(QR/Code128/DataMatrix/EAN13), ZPL 레이아웃 본문, 각 바코드·텍스트 영역을 A-MES 데이터 필드에 바인딩하는 JSON 필드맵을 보유한다. 템플릿은 고객사 전용 또는 공용(고객 NULL)일 수 있다. 버전 관리되며 인쇄 시 키의 최신 ACTIVE 버전만 사용된다.

05Validation & Business Rules검증 + 업무 규칙
RuleDescriptionAction
BR-MD-27
A customer-specific template overrides the shared template at print time.
인쇄 시 고객사 전용 템플릿이 공용 템플릿보다 우선.
Auto
BR-MD-28
The highest ACTIVE Version of a template key is selected for printing.
템플릿 키의 최신 ACTIVE 버전이 인쇄에 선택됨.
Auto
VR-Lbl-01
FieldMapJSON keys must resolve to valid A-MES data fields.
필드맵 JSON 키는 유효한 A-MES 데이터 필드로 해석되어야 함.
Validate

(TemplateName, Version) must be unique; a duplicate version is rejected.
(템플릿명, 버전) 조합 고유 — 중복 버전 거부.
Block
VR-Lbl-03
Editing an ACTIVE template creates a new version, leaving the prior immutable.
ACTIVE 템플릿 수정은 신규 버전 생성 — 이전 버전은 불변.
Auto
