01
Overview
화면 개요
Displays all open purchase orders received from the SCM portal and allows warehouse staff to register expected inbound schedules. ETA D-Day is auto-calculated; overdue deliveries are highlighted in red and trigger alerts to the procurement and warehouse manager. The [Receive] action launches the WH-02 PDA Inbound flow for the selected PO, while QC-01 incoming inspection is auto-queued upon confirmation.
SCM 포탈에서 수신한 모든 미입고 발주(PO)를 표시하고, 창고 담당자가 입고 예정 일정을 등록할 수 있는 화면입니다. ETA D-Day는 자동 계산되며, 지연 발주는 빨간색으로 강조되고 구매팀·창고 관리자에게 알림이 발송됩니다. [Receive] 클릭 시 선택한 PO에 대해 WH-02 PDA 입고 흐름이 시작되며, 확정 즉시 QC-01 수입검사가 자동 대기열에 등록됩니다.

UI Component Specification
UI 컴포넌트 명세

3.1 Toolbar Components · 툴바 컴포넌트

ID Component 컴포넌트 Type 타입 Req 필수 Validation / Behavior 검증·동작 Default 기본값
CMP-01 Status Filter상태 필터 SELECT N Options: All / Pending / Arriving Today / Overdue / Completed. Changes trigger table reload. All Status
CMP-02 Vendor Filter공급사 필터 SELECT N Loaded from md_vendor where is_active=1. Multi-select disabled in v1. All Vendors

04
Event & Action Flow
이벤트 · 액션 흐름
1
Scheduled scm.sync.cron @ */30min
System fetches new/updated POs from SCM portal every 30 minutes via SAP B1 SLD adapter.
시스템이 SCM 포탈에서 신규/갱신 PO를 30분마다 SAP B1 SLD 어댑터로 수신합니다.
EFFECT UPSERT into po_header · UPDATE last_synced_at · trigger overdue re-evaluation.
2
User page.load · WH-01 진입
Warehouse staff opens the screen. System auto-applies default filter (Today ~ +14d, All Status).
창고 담당자가 화면에 진입하면 기본 필터(오늘~+14일, 전체 상태)가 자동 적용됩니다.
API GET /api/wh/inbound/schedule?from=...&to=...&status=all → KPI strip + table populated.
3
User click · [🔍 Search]
User adjusts filter (status / vendor / date range / search box) and clicks Search. Same effect when pressing Enter in any filter field.
필터(상태/공급사/기간/검색) 변경 후 [검색] 클릭 또는 Enter 입력 시 동일 동작.
VALIDATE date_from ≤ date_to, range ≤ 90 days, search ≥ 2 chars → re-query & redraw table.
4
User click · [📥 Receive]
User clicks Receive on a row. System checks RBAC WH.Receive and PO status validity, then launches WH-02 PDA flow with PO context.
[Receive] 클릭 시 RBAC 권한·PO 상태 검증 후 WH-02 PDA 입고 흐름으로 PO 정보와 함께 전환됩니다.
REDIRECT POST /api/wh/inbound/start → returns session_id → redirect /wh/pda/inbound?po=...&sid=....
5
System job.overdue.check @ 09:00 daily
Daily 09:00 KST job re-evaluates ETA. POs past ETA without receipt become Overdue → alert sent to procurement & WH manager via in-app + email.
매일 09:00 KST 배치가 ETA를 재평가하여 미입고 PO를 지연 상태로 전환 + 구매팀·창고관리자에게 알림 발송.
EFFECT UPDATE po_header SET status='Overdue' · INSERT into sys_alert · email via SYS-04.
6
User click · [+ Register Schedule]
Manager opens modal to manually register an inbound schedule for non-SCM POs (e.g. local consumables, sample shipments).
SCM 외 발주(소모성·샘플 등)에 대해 매니저가 수동으로 입고 예정을 등록할 수 있는 모달을 엽니다.
REQUIRE Role = WH.Manage · fields: Vendor, Item, Qty, ETA, Note → POST /api/wh/inbound/manual.
✓
System return · from WH-02 success
After WH-02 PDA inbound completion, user returns to WH-01. Received PO row disappears (or shows Received badge depending on filter).
WH-02 PDA 입고 완료 후 WH-01로 복귀하면 입고 완료된 PO 행이 사라지거나 (필터에 따라) [Received] 상태 배지로 표시됩니다.
DOWNSTREAM QC-01 incoming inspection auto-queued · min-stock alert re-evaluated · WH-08 transaction log written.
05
Exception & Error Handling
예외 · 에러 처리
E-WH01-001
SCM Sync FailureSCM 동기화 실패
Cause
SAP B1 SLD endpoint unreachable / timeout > 30s. SAP B1 SLD 연결 실패·30초 초과.
Detect
Last last_synced_at > 60 min ago.
Action
Show yellow banner at top: "Last sync: HH:MM. Connection issue." Retry button available. Cached data still displayed.
Notify
SYS admin via in-app alert + email after 3 consecutive failures.
⚠ SCM SYNC DELAYED — DATA MAY BE STALE
E-WH01-002
Receive Without Permission권한 없는 입고 시도
Cause
User clicks [Receive] without WH.Receive role. 권한 없는 사용자가 입고 시도.
Detect
Backend RBAC check on POST /api/wh/inbound/start.
Action
Block API call. Return 403 Forbidden. Modal: "You don't have permission to receive. Contact your manager."
Log
Record attempt in sys_audit with user_id, timestamp, attempted_po.
⛔ ACCESS DENIED — ROLE WH.RECEIVE REQUIRED
E-WH01-003
PO Already Received이미 입고된 PO
Cause
Another user already completed WH-02 for this PO (race condition). 동시 작업으로 이미 다른 사용자가 입고 완료.
Detect
On [Receive] click: SELECT status FROM po_header → status = 'Received'.
Action
Refuse navigation to WH-02. Show toast: "This PO has been received by [user] at [time]." Refresh table.
Recovery
User views WH-08 Transaction History for confirmation.
⚠ ALREADY RECEIVED — SEE WH-08 HISTORY
E-WH01-004
Invalid Date Range잘못된 기간 입력
Cause
date_from > date_to or range > 90 days. 시작일 > 종료일 또는 기간 90일 초과.
Detect
Client-side validation before API call.
Action
Highlight date inputs with red border. Inline error: "Range must be valid and ≤ 90 days." Search button disabled.
Recovery
User corrects dates → validation passes → button re-enabled.
⚠ INVALID RANGE — MAX 90 DAYS
E-WH01-005
Excel Export Overflow엑셀 출력 초과
Cause
Filtered result > 10,000 rows. 필터 결과가 1만 행 초과.
Detect
Server-side count check before file generation.
Action
Reject export. Modal: "Result exceeds 10,000 rows. Narrow your filter or use Reports module."
Recovery
User reduces date range or applies vendor filter, then retries.
⚠ TOO MANY ROWS — NARROW FILTER
E-WH01-006
Empty Result Set조회 결과 없음
Cause
No PO matches current filter. 필터 조건에 맞는 발주 없음.
Detect
API returns empty array.
Action
Show empty state illustration + message: "No inbound schedules match your filter." KPI strip shows zeros but remains visible.
Recovery
Reset filter button shown.
ℹ NO RESULTS — TRY CLEARING FILTER

07
Applicable Business Rules
적용 업무 규칙
Rule ID Description Action
BR-WH-01
Items without a valid open PO cannot be received. Barcode scan is blocked immediately if no matching PO is found in the system.
유효한 미입고 PO가 없는 품목은 입고할 수 없으며, 일치하는 PO가 없는 바코드 스캔은 즉시 차단됩니다.
 Block
BR-WH-02
All inbound items automatically trigger QC-01 incoming inspection. Inventory is updated only after QC pass.
모든 입고 품목은 QC-01 수입검사가 자동 트리거되며, 검사 합격 후에만 재고에 반영됩니다.
 Auto
BR-WH-03
When on-hand quantity falls below minimum stock level (MD-09/10), an alert is sent to the warehouse manager and procurement team.
현재고가 최소재고(MD-09/10) 미만이 되면 창고 관리자·구매팀에 즉시 알림 발송.
 Alert
