Overview
화면 개요 · 목적 · 컨텍스트
Receives confirmed supply plans from SAP Business One. Current (Phase 1): manual Excel import. Future (Phase 2): direct integration via SAP B1 Service Layer API. Each supply plan row corresponds to a confirmed sales order with order qty, due date, and customer reference. Once imported, plans flow to PP-03 for feasibility review and WO batch creation.
SAP B1에서 확정된 공급계획 수신. 1차: Excel 수동 import. 2차: SAP B1 Service Layer API 직접 연동. 각 행은 확정 판매오더(고객사·수량·납기). Import 후 PP-03에서 가용성 검토·WO 일괄 생성으로 연계.

Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
SCHEDULED cron · 07:00 daily sync
[Phase 2 future] System auto-calls SAP B1 Service Layer to fetch /Orders?filter=DocStatus eq 'O' and inserts new SOs.
[2차 예정] 매일 07:00 SAP B1 Service Layer /Orders 호출하여 미확정 SO 자동 import.
APIGET /b1s/v1/Orders?$filter=DocStatus%20eq%20%27O%27
2
USER click · [⬆ Import]
Manager uploads SAP B1 Excel export. System validates schema and parses rows.
관리자가 SAP B1 Excel export 업로드. 스키마 검증 + 행 파싱.
VALIDATERequired cols: SO No, Customer, Item Code, Qty, Due Date, Status
3
SYSTEM save · transactional insert
Inserts header row in PP_SupplyPlan and detail rows in PP_SupplyPlanDetail atomically.
PP_SupplyPlan 헤더 + PP_SupplyPlanDetail 상세를 원자적으로 INSERT.
DBBEGIN TRAN; INSERT header; INSERT details; COMMIT
4
SYSTEM match · forecast vs SO
Cross-references confirmed SO against PP-01 Forecast for the same item/period.
확정 SO와 PP-01 Forecast를 품목·기간별로 매칭.
ALERTGap > 20% triggers BR-PP-F002 alert (same as PP-01).
5
SYSTEM complete · ready for PP-03
Imported SOs appear in PP-03 with Forecast reference and FG stock context.
Import된 SO가 PP-03에서 Forecast 참조·FG 재고와 함께 표시.

Exception Handling
예외 처리 · 오류 코드 · 복구 절차
E-PP02-001
Schema Mismatch스키마 불일치
CauseExcel column names don't match expected.
DetectPre-parse header check.
ActionBlock import. Show missing/extra columns.
RecoverRe-download SAP B1 export or use Sample Template.
⛔ COLUMN MISMATCH
E-PP02-002
Duplicate SO NoSO 중복
CauseSO No already exists in PP_SupplyPlan.
DetectUnique key constraint.
ActionShow duplicate count, ask user: update or skip.
RecoverUpdate overwrites Qty/Due Date.
⚠ DUPLICATE SO
E-PP02-003
SAP API TimeoutSAP API 타임아웃
Cause[Phase 2] Service Layer call exceeds 30s.
DetectHeartbeat monitor.
ActionRetry 3× with exponential backoff. Then alert.
RecoverManual Excel fallback or wait for SAP recovery.
⛔ SAP TIMEOUT
E-PP02-004
Unknown Item Code미등록 품목
CauseSO item not in MD_Item master.
DetectPer-row validation.
ActionSkip row, log to error report. Other rows continue.
RecoverMD team adds item or removes invalid SO in SAP.
⛔ ITEM NOT IN MD
E-PP02-005
Past Due Date과거 납기
CauseSO due date is in the past.
DetectDate validation.
ActionAllow import but flag yellow. Cannot create WO from past-due SO without override.
RecoverUpdate due date in SAP B1.
⚠ PAST DUE
E-PP02-006
Authentication Failed인증 실패
Cause[Phase 2] SAP B1 Service Layer login token expired.
DetectAPI 401 response.
ActionRe-authenticate via stored credentials. Retry once.
RecoverSYS admin updates SAP credential secret.
⛔ AUTH FAILED
