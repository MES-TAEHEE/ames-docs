01Overview개요
EN · Description The Custom Report Builder is a no-code tool that lets power users design their own reports without IT support. The user picks a data source (a curated, permission-scoped dataset), selects columns, defines filters and grouping, chooses a chart type, and sees a live preview. The finished design is saved and published to the RPT-09 catalog as a reusable custom report — runnable on-demand or on a schedule like any standard report. The builder only exposes pre-approved datasets, so it cannot bypass data-access governance.
KO · 설명 사용자 정의 리포트 빌더는 파워 유저가 IT 지원 없이 자체 보고서를 설계하는 노코드 도구다. 사용자는 데이터 소스(권한 범위가 정해진 정제 데이터셋)를 고르고, 컬럼을 선택하며, 필터·그룹화를 정의하고, 차트 유형을 선택한 뒤 실시간 미리보기를 확인한다. 완성된 설계는 저장되어 RPT-09 카탈로그에 게시되며, 표준 보고서처럼 온디맨드 또는 스케줄 실행할 수 있다. 사전 승인된 데이터셋만 노출되므로 데이터 접근 거버넌스를 우회할 수 없다.
📌 Governed Self-Service · 통제된 셀프서비스RPT-10은 자유로운 보고서 제작을 허용하되, 노출 데이터셋·컬럼·권한은 모두 사전 정의된 범위 내로 제한한다 — 셀프서비스와 거버넌스의 균형. · RPT-10 enables free report-building but only within pre-defined datasets and permissions.

03Builder Components빌더 4단계 구성요소
ComponentFunction · 기능
① Data Source정제·승인된 데이터셋 선택 (생산·불량·출하·재고·OEE). 사용자 권한 범위만 노출 — 원시 테이블 직접 접근 불가.
② Columns데이터셋 내 컬럼을 체크박스로 다중 선택 + 표시 순서 지정.
③ Filter필드별 조건 빌더 (= / ≠ / > / < / 범위 / IN). AND 결합.
④ Group / Aggregate그룹화 축 + 집계 함수 (합계·평균·건수·최대·최소).
⑤ Chart Type막대 / 선 / 파레토 / 파이 / 테이블 — 데이터 형태에 맞게 선택.
⑥ Live Preview설정 변경 시 즉시 샘플 데이터로 차트·테이블 재렌더.
⚙ Dataset Governance · 데이터셋 거버넌스데이터셋은 SYS/MD 관리자가 사전 정의하며, 어떤 컬럼을 어떤 역할에 노출할지 결정한다. RPT-10 사용자는 이 경계를 넘을 수 없다. · Datasets are pre-defined by admins, deciding which columns are exposed to which roles — RPT-10 users cannot cross that boundary.
04Workflow작업 흐름
Power user launches the builder from RPT-09 "+ Custom Report".
파워 유저가 RPT-09 "+ Custom Report"에서 빌더를 실행한다.
User picks a data source, selects columns, defines filters and grouping, chooses a chart type.
데이터 소스 선택, 컬럼 선택, 필터·그룹화 정의, 차트 유형 선택.
Live preview updates with each change; the user iterates until satisfied.
변경마다 실시간 미리보기가 갱신 — 만족할 때까지 반복한다.
User saves a DRAFT, or publishes the design to the RPT-09 catalog.
사용자가 DRAFT 저장, 또는 설계를 RPT-09 카탈로그에 게시한다.
Published custom report behaves like a standard report — on-demand or scheduled, with permissions.
게시된 커스텀 보고서는 표준 보고서처럼 동작 — 온디맨드/스케줄, 권한 적용.
05Business Rules업무 규칙
RuleDescriptionAction
BR-RPT-35
The builder exposes only pre-approved datasets; raw table access is never available.
빌더는 사전 승인 데이터셋만 노출 — 원시 테이블 접근 불가.
Restrict
BR-RPT-36
Columns visible in a dataset are scoped to the user's role; restricted columns are hidden.
데이터셋 노출 컬럼은 사용자 역할 범위 — 제한 컬럼 숨김.
Restrict
BR-RPT-37
A custom report must be published to RPT-09 before it can be scheduled or shared.
커스텀 보고서는 RPT-09 게시 후에만 스케줄·공유 가능.
Block
BR-RPT-38
The 8 standard reports cannot be edited in the builder; they may be cloned as a starting point.
표준 8종은 빌더에서 편집 불가 — 복제 후 시작점으로만 사용.
Clone
