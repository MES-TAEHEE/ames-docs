01
Overview
화면 개요
PDA-driven receiving workflow consisting of three sequential steps: (1) Scan item barcode against the open PO, (2) Confirm quantity via numeric keypad, and (3) Scan shelf QR code to assign storage location. The system validates each step in real time — barcode mismatches against the PO item are immediately blocked, and the worker cannot proceed until correction. Upon confirmation, inventory is incremented, a location label is printed, and the QC-01 incoming inspection queue is auto-triggered.
PDA 기반 3단계 입고 처리: (1) 품목 바코드 스캔으로 PO와 대조, (2) 숫자패드로 수량 확인, (3) 선반 QR 스캔으로 로케이션 지정. 각 단계에서 실시간 검증되며, PO 품목과 불일치 시 즉시 차단되어 수정 전까지 진행 불가. 확정 시 재고 증가·위치 라벨 출력·QC-01 수입검사 큐 자동 등록이 이루어집니다.

04
Event & Action Flow
이벤트 · 액션 흐름
1
Userlaunch · from WH-01 [Receive]
User clicks [Receive] on a PO row in WH-01. PDA launches WH-02 with PO context. Step ① is shown.
WH-01에서 [Receive] 클릭 시 WH-02가 PO 정보와 함께 실행되며 Step ① 화면 표시.
CONTEXTsession_id, po_no, expected_item loaded into PDA local state.
2
PDA Scantrigger · item barcode
Worker pulls trigger to scan item barcode. System matches against PO expected item.
작업자가 트리거를 당겨 품목 바코드 스캔. 시스템이 PO 예정 품목과 대조.
VALIDATEscanned_code == po.item_code → success (green) · mismatch → block (E-WH02-001).
3
Usertap · numpad / [All]
Worker enters received quantity via numpad or taps [All] to auto-fill expected quantity. Display shows qty with tolerance color coding.
숫자패드 입력 또는 [All] 버튼으로 수량 입력. 디스플레이는 허용오차 색상 표시.
DETECT qty < expected → auto-enable Partial Inbound mode. qty > expected → warn (E-WH02-003).
4
PDA Scantrigger · shelf QR
Worker scans the shelf QR code at the target storage location. System validates capacity vs incoming qty.
대상 선반의 QR 코드를 스캔. 시스템이 용량 대비 입고량 검증.
VALIDATElocation.capacity - location.on_hand >= qty → OK · else block (E-WH02-004).
5
Usertap · [Confirm Inbound]
Worker confirms. Backend commits transaction atomically: insert inbound, update inventory, update PO status, queue QC-01.
작업자 확정. 백엔드가 트랜잭션을 원자적으로 커밋: 입고 INSERT, 재고 UPDATE, PO 상태 변경, QC-01 큐 등록.
APIPOST /api/wh/inbound/confirm · BEGIN TRAN ... COMMIT · returns inbound_id.
6
Systemprint · location label
Optionally prints a location label via Bluetooth-paired Zebra ZD230 printer. Worker affixes label to the lot.
블루투스 페어링된 Zebra ZD230 프린터로 위치 라벨 출력(선택). 작업자가 로트에 부착.
OUTPUT ZPL command via Bluetooth · label contains item, qty, location, date, inbound_id barcode.
✓
Systemcomplete · return to WH-01
PDA returns to WH-01. Received PO is no longer in pending list. QC-01 inspector receives notification.
PDA가 WH-01로 복귀. 처리된 PO는 미입고 목록에서 제외. QC-01 검사자에게 알림.
DOWNSTREAM QC-01 queue · WH-03 inventory updated · WH-08 transaction log · min-stock check.
05
Exception & Error Handling
예외 · 에러 처리
E-WH02-001
Item Barcode Mismatch품목 바코드 불일치
Cause
Scanned item code ≠ PO expected item. 스캔 품목이 PO 예정 품목과 다름.
Detect
Step ① real-time comparison.
Action
Red full-screen warning + vibration + beep. Re-scan button only. 3 consecutive failures → manager alert.
Recovery
Re-scan correct item to resume.
⛔ ITEM MISMATCH — INPUT BLOCKED
E-WH02-002
PO Not Found / ExpiredPO 없음·만료
Cause
PO context invalid: cancelled, already received, or session timeout. PO 취소·이미 입고·세션 만료.
Detect
Server check on Step ① scan.
Action
Block scan. Modal: "PO no longer valid. Return to WH-01." Auto-redirect after 5 sec.
Recovery
Re-select PO from WH-01.
⛔ PO INVALID — RETURN TO LIST
E-WH02-003
Qty Exceeds PO발주량 초과
Cause
Entered qty > expected by >5%. 입력 수량이 발주 수량 5% 초과.
Detect
Step ② numeric validation.
Action
Yellow display. Confirmation popup: "Receive QTY (over by N%)? Comment required." Free-text reason input.
Recovery
User confirms with reason, or corrects qty.
⚠ OVER-RECEIPT — REASON REQUIRED
E-WH02-004
Location Capacity Exceeded로케이션 용량 초과
Cause
Target location's free capacity < incoming qty. 대상 위치 잔여 용량 부족.
Detect
Step ③ on QR scan.
Action
Block confirmation. Show free capacity. Suggest alternative location.
Recovery
Scan different location, or split into multiple locations.
⛔ LOCATION FULL — SCAN ANOTHER
E-WH02-005
Network Disconnected네트워크 단절
Cause
WiFi signal lost or backend unreachable. WiFi 단절 또는 백엔드 응답 없음.
Detect
PDA heartbeat check every 10s.
Action
Switch to offline mode. Queue transactions locally (up to 4hr). Yellow banner. Auto-sync on reconnect.
Recovery
Reconnect to WiFi → queued transactions sync in FIFO order.
⚠ OFFLINE — QUEUE MODE ACTIVE
E-WH02-006
Printer Not Ready프린터 미응답
Cause
Bluetooth printer disconnected or out of paper. 블루투스 단절·용지 없음.
Detect
Print request timeout 8s.
Action
Inbound transaction NOT rolled back. Show: "Label print failed. Retry?" Inventory commit already complete.
Recovery
Re-print from WH-08 transaction history.
⚠ PRINT FAILED — RETRY FROM WH-08

07
Applicable Business Rules
적용 업무 규칙
Rule IDDescriptionAction
BR-WH-01
Items without a valid open PO cannot be received. Barcode scan blocked immediately if no matching PO is found.
유효한 미입고 PO 없는 품목 입고 불가. 미일치 바코드 즉시 차단.
Block
BR-WH-02
All inbound items automatically trigger QC-01 incoming inspection. Inventory updated only after QC pass.
모든 입고 품목 QC-01 자동 트리거. 검사 합격 후 재고 반영.
Auto
BR-WH-03
When on-hand qty falls below minimum stock level (MD-09/10), an alert is sent to manager and procurement.
최소재고 미달 시 창고관리자·구매팀 알림 발송.
Alert
