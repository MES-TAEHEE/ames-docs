01Overview개요
EN · Description Forward and backward traceability across the full A-MES — from raw-material LOT in WH, through production WOs (INJ/IMG/PNT), inspections (QC-01/02/03), NCRs, Holds, CAPAs, and downstream FG/shipment events. Search by any handle (LOT/WO/Customer/Item/Date/Defect), then drill back from a customer claim to the originating raw material, or forward from a suspect raw LOT to every shipped unit affected. Read-only views available to Sales and external Auditors. All trace data is sealed for ≥ 5 years. Used heavily for IATF 16949 audits and customer recall analysis.
KO · 설명 A-MES 전체에 걸친 정방향/역방향 이력 추적 — WH 원자재 LOT부터 생산 WO(INJ/IMG/PNT), 검사(QC-01/02/03), NCR, Hold, CAPA, 그리고 하류 FG/출하 이벤트까지. 다양한 핸들(LOT/WO/고객/품목/기간/불량)로 검색, 고객 클레임으로부터 원천 원자재까지 역추적 또는 의심 원자재 LOT으로부터 영향 받은 모든 출하분으로 정추적. Sales·외부 감사관은 읽기 전용 뷰. 모든 추적 데이터 5년 이상 봉인 보관. IATF 16949 감사·고객 리콜 분석에 핵심 사용.

03Trace Directions추적 방향
DirectionUse Case · 활용 사례
← Backward (역방향)고객 클레임 / Shipped Unit / VIN으로 시작 → 부품 LOT → WO → 공정 이벤트 → 원자재 LOT → 공급사. 리콜 시 표준 방향.
→ Forward (정방향)의심 원자재 LOT / 공급사로 시작 → 사용된 모든 WO → 검사 결과 → 출하된 모든 단위 → 고객. 결함 자재 발견 시 영향 범위 파악.
↔ Lateral (수평)동일 시점 동일 라인의 다른 WO, 동일 금형 사용 WO 등. CAPA 근본원인 분석 시 활용.
⊕ Multi-hop3-hop까지 확장 (예: 고객 → 부품 → 원자재 → 공급사의 다른 LOT). 시스템 응답 ≤ 500ms 보장.
04Workflow작업 흐름
User enters one or more search handles (LOT/WO/Customer/VIN, etc.); system identifies trace target.
사용자가 1개 이상 검색 핸들 입력, 시스템이 추적 대상 식별.
System auto-determines direction: Customer/VIN input → Backward; LOT/Supplier input → Forward.
시스템이 방향 자동 결정: 고객/VIN 입력 → 역방향, LOT/공급사 → 정방향.
Renders unified Event Timeline: WH movements + Production + RFID + Inspections + NCR/Hold/CAPA + Shipments.
통합 이벤트 타임라인 렌더링: WH 이동 + 생산 + RFID + 검사 + NCR/Hold/CAPA + 출하.
Each event has drill-link to its source screen (Inspection detail, Oven curve, RFID event chain, etc.).
각 이벤트는 소스 화면으로 드릴 링크 (검사 상세, 오븐 곡선, RFID 이벤트 체인 등).
User can pivot: clicking any event triggers a new trace centered on that event.
사용자는 피벗 가능: 이벤트 클릭 시 그 이벤트 중심 신규 추적 시작.
"Affected Items" panel lists all WOs/LOTs/shipments matching the suspected root cause for impact assessment.
"영향 항목" 패널이 의심 근본원인과 일치하는 모든 WO/LOT/출하 표시, 영향 범위 평가.
Export options: PDF Recall Report, Excel impact list, JSON API for external systems.
출력 옵션: PDF 리콜 보고서, Excel 영향 목록, JSON API (외부 시스템 연동).
All search/view actions logged ≥ 5 years; Sales/Auditor permissions enforce data scope.
모든 검색/조회 행위 5년 이상 로그, Sales/Auditor 권한별 데이터 범위 적용.
05Business Rules업무 규칙
RuleDescriptionAction
BR-QC-33
QC-TRC is Read-only; no data modification possible. Edits must occur in source modules.
QC-TRC는 읽기 전용, 데이터 변경 불가. 수정은 소스 모듈에서.
Lock
BR-QC-34
Sales role can trace by customer only; cannot see suppliers/internal cost data.
Sales 권한은 고객 기준만 추적, 공급사·내부 원가 데이터 접근 불가.
Scope
BR-QC-35
External Auditor role gets time-boxed access (visit period only); auto-revoked after.
외부 감사관은 방문 기간만 접근, 종료 후 자동 회수.
Time-Box
BR-QC-08
All trace data (events, links) retained ≥ 5 years (regulatory + customer audit).
전 추적 데이터 5년 이상 보관 (법규 + 고객 감사).
Retain
BR-QC-36
Search response time ≤ 500ms for 95% of queries; multi-hop ≤ 2s.
검색 응답 시간 95% 쿼리 500ms 이하, 멀티홉 2s 이하.
SLA
BR-QC-37
All search/view actions logged with userID, timestamp, query params for audit accountability.
모든 검색/조회 행위 사용자ID·시각·쿼리 기록 (감사 책임).
Log
