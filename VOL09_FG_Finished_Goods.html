FG-01
PDA Finished Goods Stocking
PDA 완제품 적재
📱 PDA1st Dev
EN · Description
After QC-03 final inspection passes, operator scans the WO barcode via PDA to confirm the item and quantity. The system verifies QC pass status before allowing stocking. Operator then scans a FG shelf QR code to assign the storage location. All stocked items are tagged with the production date and time for FIFO picking later. Stocking for a WO that has not passed QC-03 is completely blocked.

FG-04
PDA FIFO Picking
PDA FIFO 피킹
📱 PDA1st Dev
EN
PDA-guided FIFO picking for outbound shipment. System displays the oldest-stocked location first (by FG-01 stocking timestamp). Operator navigates to the suggested location and scans the item barcode to confirm. Wrong item barcode is immediately blocked with a red alert — mis-picks are prevented. Quantity is confirmed via numeric keypad. After picking, location inventory is auto-deducted.
KO

FG-07
Day-End Close
일 마감
🖥 Web PC1st Dev
EN
Day-end closing procedure locks all shipment transactions for the day. Checklist must be completed: all PDA picks confirmed, all delivery notes issued, FG inventory snapshot taken. After close, no backdated entries allowed. Closing generates the daily shipment summary report (RPT-03). Requires supervisor approval (digital signature).
KO
일 마감으로 당일 출하 거래 전체 잠금. 체크리스트 완료 필수: 전체 PDA 피킹 확정, 전체 거래명세서 발행, FG 재고 스냅샷 생성. 마감 후 소급 입력 불가. RPT-03 일별 출하 요약 보고서 자동 생성. 관리자 전자서명 승인 필요.
⚠️
Day-End Close is irreversible · 일 마감은 취소 불가 — Once closed, all transactions for the day are locked. No backdated adjustments allowed after close. Supervisor digital approval required. · 마감 후 당일 거래 전체 잠금. 소급 수정 불가. 관리자 전자서명 필수.

PDA-based loading confirmation: operator scans each carton barcode as it is loaded into the truck. System counts loaded qty vs. pick list qty. When all items are confirmed, one-tap loading complete — triggers FG-06 delivery note auto-generation.
