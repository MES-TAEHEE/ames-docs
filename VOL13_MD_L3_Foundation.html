01Overview개요
EN · Description Category A holds the foundation masters on which every other master and every transaction depends. Item Master (MD-01) is the spine — every raw material, fabric, powder, paint, semi-finished and finished good. BOM (MD-02) defines the multi-level material structure, version-controlled by BOM Version (MD-03). UOM (MD-13) supplies the units every quantity is expressed in. Production Calendar (MD-14) provides the working-day / shift time base for capacity and OEE. These 5 entities are entered earliest in the Phase-0 sequence — nothing operational can run without them.
KO · 설명 카테고리 A는 다른 모든 마스터와 모든 트랜잭션이 의존하는 기반 마스터다. 품목 마스터(MD-01)가 척추 — 전 원자재·원단·분체·도료·반제품·완제품. BOM(MD-02)은 다단계 자재 구조를 정의하고 BOM 버전(MD-03)으로 버전 관리된다. 단위(MD-13)는 모든 수량의 단위를 공급한다. 생산 캘린더(MD-14)는 능력·OEE 산정의 근무일/교대 시간 기반을 제공한다. 이 5개 엔티티는 Phase-0 순서에서 가장 먼저 입력되며 — 이것 없이는 어떤 운영도 불가능하다.
📌 The Spine · 척추 마스터MD-01 Item이 없으면 BOM·BOP·검사기준·재고 어느 것도 정의할 수 없다 — 카테고리 A는 Phase-0 입력 순서 ②③④⑤㉘에 배치되어 가장 먼저 완성된다. · Without MD-01 Item nothing else can be defined — Category A is entered first in Phase-0.

04Maintenance Workflow마스터 유지보수 흐름
During Phase-0, UOM and Item are loaded first (often bulk-imported from ERP).
Phase-0에서 UOM·Item을 먼저 입력 (ERP 일괄 가져오기).
BOM is built per finished good, then a BOM Version record fixes its effective period.
완제품별 BOM 작성 후 BOM 버전으로 발효 기간을 확정한다.
A change request to any foundation master enters an approval workflow (creator ≠ approver).
기반 마스터 변경 요청은 승인 워크플로 진입 (생성자 ≠ 승인자).
On approval, a new BOM Version is created — the old version becomes SUPERSEDED, not deleted.
승인 시 신규 BOM 버전 생성 — 구버전은 SUPERSEDED 처리, 삭제하지 않음.
Every master change is written to the SYS-05 audit log and retained ≥ 7 years.
전 마스터 변경은 SYS-05 감사 로그 기록, 7년 이상 보존.

기반 마스터 Phase-0 완성 필수 — 누락 시 모듈 차단.

Customer item-no mapping (MD-01) must exist for SAV & GEO before FG-06 delivery note.
MD-01 고객사 품번 매핑 후 FG-06 거래명세서 발행.
Validation

A master referenced by any active transaction cannot be deleted — only deactivated.
운영 트랜잭션 참조 마스터는 삭제 불가 — 비활성화만.
Block

All master changes require an approval workflow (creator ≠ approver) + 7-year audit.
마스터 변경은 승인 워크플로 + 7년 감사 로그.
