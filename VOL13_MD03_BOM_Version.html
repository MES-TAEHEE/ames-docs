01Overview개요
EN · Description MD_BOMVersion is the version-control header for BOM changes. Each row is one version of the BOM for a root item, carrying an effective period, the change reason and type, and an approval record. MD_BOM lines reference a VersionID, so the structure as-of any date can be reconstructed. Only one version per root item may be ACTIVE at a time; superseded versions are kept, never deleted. A Work Order created on a given date pins itself to whichever version was ACTIVE then — later BOM changes do not retroactively alter in-flight production.
KO · 설명 MD_BOMVersion은 BOM 변경의 버전 통제 헤더다. 각 행은 한 루트 품목의 BOM 한 버전으로, 발효 기간·변경 사유·변경 유형·승인 기록을 담는다. MD_BOM 라인이 VersionID를 참조하므로 임의 일자 기준 구조를 재구성할 수 있다. 루트 품목당 동시에 ACTIVE 버전은 하나뿐이며, 대체된 버전은 보존하고 삭제하지 않는다. 특정 일자에 생성된 작업지시는 그 시점 ACTIVE 버전에 고정되어 — 이후 BOM 변경이 진행 중 생산을 소급 변경하지 않는다.

03Constraints & Index제약조건 + 인덱스

PKPK_MD_BOMVersionPRIMARY KEY (VersionID)
FKFK_BOMVer_ItemFOREIGN KEY (RootItemNo) REFERENCES MD_Item(ItemNo)
CKCK_BOMVer_TypeCHECK (ChangeType IN ('DESIGN','LOCALIZE','COST_DOWN','ERROR_FIX','INITIAL'))

UKUK_BOMVer_NoUNIQUE (RootItemNo, VersionNo)
IXIX_BOMVer_ActiveINDEX (RootItemNo, Status, EffFrom) — as-of 버전 조회 가속.
⚠ One ACTIVE per Item · 품목당 ACTIVE 1개루트 품목당 동시 ACTIVE 버전은 1개만 허용된다 — 발효 기간 비중첩과 단일 ACTIVE는 버전 활성화 트랜잭션에서 애플리케이션 레벨로 강제한다. · One ACTIVE version per root item; non-overlapping effective periods enforced at the activation transaction.

05Validation & Business Rules검증 + 업무 규칙
RuleDescriptionAction
VR-BVer-01
Only one version per RootItemNo may be ACTIVE; activating a new one supersedes the old.
루트 품목당 ACTIVE 버전 1개 — 신규 활성화 시 구버전 SUPERSEDED.
Block
VR-BVer-02
A version cannot become ACTIVE without an approver (RequestedBy ≠ ApprovedBy).
승인자 없이 ACTIVE 전환 불가 (요청자 ≠ 승인자).
Validation
VR-BVer-03
A Work Order pins the BOM version ACTIVE as-of its creation date; later changes do not apply.
WO는 생성일 기준 ACTIVE 버전 고정 — 이후 변경 미적용.
Snapshot
BR-MD-23
A SUPERSEDED version is retained for traceability — never deleted.
SUPERSEDED 버전은 추적성 보존 — 삭제 불가.
Block
BR-MD-24
Version create / activate requires an approval workflow + SYS-05 audit log.
버전 생성·활성화는 승인 워크플로 + SYS-05 감사 로그.
Audit
