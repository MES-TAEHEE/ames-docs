01Overview개요
EN · Description The Defect Pareto Report ranks defects by frequency for a selected period so improvement effort targets the "vital few" causes. It draws a Pareto chart — descending bars of defect count with an overlaid cumulative percentage line — plus a backing table. Defects can be grouped by defect type, process, line, or item. The top contributor (and the set reaching 80% cumulative) is highlighted. Each defect type links to QC-05 CAPA so a corrective action can be initiated directly. The report is run on-demand for any date range.
KO · 설명 불량 파레토 보고서는 선택 기간의 불량을 빈도순으로 정렬해 개선 노력이 '핵심 소수' 원인에 집중되도록 한다. 파레토 차트(불량 건수 내림차순 막대 + 누적 비율 선 오버레이)와 백업 테이블을 함께 표시한다. 불량은 불량 유형·공정·라인·품목 기준으로 그룹화할 수 있다. 최다 기여 항목(및 누적 80% 도달 집합)이 강조 표시된다. 각 불량 유형은 QC-05 CAPA로 연결되어 시정조치를 즉시 개시할 수 있다. 임의 기간에 대해 온디맨드로 실행한다.
📌 80/20 Principle · 파레토 원칙대개 불량 건수의 80%는 불량 유형의 20%에서 발생한다. RPT-02는 그 20%(vital few)를 시각적으로 분리해 CAPA 우선순위를 정한다. · Typically 80% of defects come from 20% of types — RPT-02 isolates that vital few to prioritize CAPA.

03Pareto Logic파레토 산출 로직
StepLogic · 산출 로직
1. Collect기간 내 QC-02 검사 판정 NG + 공정 불량 기록을 GroupByAxis 기준으로 수집.
2. Aggregate그룹별 DefectCount 집계, 전체 합계(Total) 산출.
3. SortDefectCount 내림차순 정렬 → RankNo 부여.
4. ShareSharePct = DefectCount / Total × 100.
5. CumulativeCumulativePct = 순위까지 SharePct 누계.
6. Vital FewCumulativePct ≤ 80% 도달 행까지 VitalFewFlag=TRUE → 황색 강조.
7. Render막대(좌축 건수) + 누적선(우축 %) 콤보 차트 + 테이블 동시 렌더.
⚙ Long-tail 처리순위 하위 그룹 중 SharePct < 1% 항목들은 "Etc · 기타"로 묶어 차트 가독성을 유지한다. 테이블에서는 펼침으로 개별 확인 가능. · Sub-1% groups are bundled as "Etc" for chart readability; the table expands them individually.
04Workflow작업 흐름
User opens RPT-02, selects period, group-by axis, and optional process/line/item filters.
사용자가 RPT-02를 열고 기간·그룹축·공정/라인/품목 필터를 선택한다.
On Run, the engine collects QC-02 NG judgments and aggregates per the Pareto logic.
Run 시 엔진이 QC-02 NG 판정을 수집하고 파레토 로직대로 집계한다.
Combo chart (bars + cumulative line) and the detail table render; vital-few rows highlighted.
콤보 차트(막대+누적선)와 상세 테이블이 렌더되고 vital-few 행이 강조된다.
User reviews the vital few; clicking a row opens QC-05 CAPA pre-filled with defect type & period.
사용자가 vital few를 검토 — 행 클릭 시 불량유형·기간이 채워진 QC-05 CAPA가 열린다.
Report can be exported (Excel/PDF) or saved as a scheduled report in RPT-09.
보고서는 Excel/PDF 출력하거나 RPT-09에 정기 보고서로 등록할 수 있다.
05Business Rules업무 규칙
RuleDescriptionAction
BR-RPT-08
Pareto bars are always sorted by defect count descending; cumulative % is computed in that order.
파레토 막대는 항상 건수 내림차순, 누적 %도 그 순서로 산출.
Auto
BR-RPT-09
Rows within cumulative 80% are flagged vital-few and highlighted for CAPA prioritization.
누적 80% 이내 행은 vital-few 플래그·강조 — CAPA 우선순위.
Flag
BR-RPT-10
Initiating CAPA from a row passes defect type + period to QC-05 and records CapaRefID back on the run.
행에서 CAPA 개시 시 불량유형·기간을 QC-05에 전달, CapaRefID 역기록.
Link
