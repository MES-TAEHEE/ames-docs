01Overview개요
EN · Description SYS-01 is where Admins manage every A-MES user account. Each account carries an employee ID, name, department, an assigned RBAC role (one of 7, defined in SYS-02), assigned production lines (for POP users), and an active status. Admins create, edit, and deactivate accounts and reset passwords — password reset is Admin-only. After 5 consecutive failed logins an account is auto-locked; only an Admin can unlock it. Every account action is written to the SYS-05 audit log. The screen is restricted to Admin and System Manager roles.
KO · 설명 SYS-01은 Admin이 전 A-MES 사용자 계정을 관리하는 화면이다. 각 계정은 사번, 이름, 부서, 배정 RBAC 역할(SYS-02에 정의된 7종 중 하나), 담당 생산 라인(POP 사용자), 활성 상태를 보유한다. Admin은 계정을 생성·수정·비활성화하고 비밀번호를 초기화한다 — 비밀번호 초기화는 Admin 전용이다. 로그인 5회 연속 실패 시 계정이 자동 잠금되며 Admin만 잠금을 해제할 수 있다. 모든 계정 행위는 SYS-05 감사 로그에 기록된다. 화면은 Admin·시스템 관리자 역할로 제한된다.
📌 Account ≠ Permission · 계정과 권한의 분리SYS-01은 '누가 사용자인가'를, SYS-02 RBAC는 '각 역할이 무엇을 할 수 있는가'를 정의한다. 계정에 역할을 배정하면 그 역할의 권한이 자동 적용된다. · SYS-01 defines who the users are; SYS-02 defines what each role can do.

StatusDescription & Transition
CREATEDAdmin이 계정 생성, 초기 비밀번호 발급 — 첫 로그인 시 변경 강제.
ACTIVE정상 로그인 가능. 역할 권한 적용. 정상 운영 상태.
LOCKED로그인 5회 연속 실패 시 자동 전환. Admin Unlock으로만 ACTIVE 복귀. 사용자·Admin 이메일 알림.
INACTIVE퇴직·휴직 시 Admin이 비활성화. 로그인 차단, 계정·이력은 보존 (삭제 아님).
⚙ No Hard Delete · 물리 삭제 없음퇴직자 계정도 삭제하지 않고 INACTIVE 처리한다 — 과거 트랜잭션·감사 로그의 작성자 추적성을 유지하기 위함이다. · Departed-employee accounts are never hard-deleted, only set INACTIVE, to preserve audit traceability.
04Workflow작업 흐름
Admin opens SYS-01, clicks Add User, enters employee details and assigns an RBAC role.
Admin이 SYS-01에서 Add User 클릭, 사원 정보 입력 후 RBAC 역할을 배정한다.
For POP operators, Admin assigns the production line(s) the user may operate.
POP 작업자는 Admin이 운영 가능한 생산 라인을 지정한다.
Account is created with an initial password; the user must change it on first login.
계정이 초기 비밀번호와 함께 생성 — 사용자는 첫 로그인 시 변경해야 한다.
On 5 consecutive failed logins, the account auto-locks and emails are sent to user + Admin.
로그인 5회 연속 실패 시 계정 자동 잠금, 사용자·Admin에게 이메일 발송.
Admin reviews and unlocks the account; the unlock event is logged in SYS-05.
Admin이 검토 후 계정 잠금 해제 — 해제 이벤트는 SYS-05에 기록된다.
On resignation, Admin sets the account INACTIVE — never hard-deleted.
퇴직 시 Admin이 계정을 INACTIVE 처리 — 물리 삭제하지 않는다.
05Business Rules업무 규칙
RuleDescriptionAction
BR-SYS-04
5 consecutive failed logins auto-lock the account; only an Admin can unlock it.
로그인 5회 연속 실패 시 자동 잠금 — Admin만 해제.
Auto-Lock
BR-SYS-05
Password reset is Admin-only; passwords are stored hashed, never in plain text.
비밀번호 초기화는 Admin 전용 — 해시 저장, 평문 미저장.
Restrict
BR-SYS-06
Accounts are never hard-deleted; departed users are set INACTIVE to preserve traceability.
계정은 물리 삭제 불가 — 퇴직자는 INACTIVE 처리해 추적성 보존.
Lock
BR-SYS-07
Every account create/edit/lock/unlock action is written to the SYS-05 audit log.
전 계정 생성·수정·잠금·해제 행위는 SYS-05 감사 로그 기록.
