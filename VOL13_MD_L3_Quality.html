01Overview개요
EN · Description Category E is the largest, holding the masters for the quality and logistics chain. The quality side: Inspection Standard (MD-06), Defect Code (MD-21) and Defect Cause (MD-22, 4M1E root-cause catalog). The customer / logistics side: Customer (MD-12), Shipment Destination (MD-11), Packaging Spec (MD-23) and Label Template (MD-24). The common side: Reason Code (MD-25) and Common Code Group (MD-26) — the shared catalogs every module's enumerations resolve through.
KO · 설명 카테고리 E는 가장 큰 카테고리로 품질·물류 체인의 마스터를 보유한다. 품질 측: 검사기준(MD-06), 불량코드(MD-21), 불량원인(MD-22, 4M1E 근본원인 카탈로그). 고객·물류 측: 고객사(MD-12), 출하처(MD-11), 포장사양(MD-23), 라벨 템플릿(MD-24). 공통 측: 사유코드(MD-25), 공통코드(MD-26) — 전 모듈의 열거형이 해석되는 공용 카탈로그.
📌 Inspect → Disposition → Ship · 검사에서 출하까지이 카테고리는 QC 검사·불량 판정·시정조치(MD-06·21·22)부터 고객·출하처·포장·라벨(MD-12·11·23·24), 그리고 전 모듈이 공유하는 사유·공통코드(MD-25·26)까지 품질·물류·공통 마스터를 한 곳에 모은다. · From QC inspection and defect disposition through customer shipment, plus the shared reason/code catalogs.

StageMaster-Driven Logic · 마스터 기반 로직
검사 (QC)MD-06 검사기준이 검사 특성·규격(LSL/USL)·샘플링 계획·CTQ를 제공 — QC-02/03이 이를 읽어 검사 시트를 구성하고 합부를 판정한다.
불량 판정·원인NG 판정 시 MD-21 불량코드로 코드화(분류·심각도·처리), MD-22 불량원인(4M1E)으로 CAPA 근본원인 분석 → RPT-02 파레토 집계.
고객·출하 (FG)MD-12 고객사가 수주·EDI를, MD-11 출하처가 납품 지점·리드타임·시간창을 제공 — FG-04 출하 계획·RPT-04 납기 산정.
포장·라벨 (FG)합격품은 MD-23 포장사양의 3단계 계층대로 박스·팔레트화, MD-24 라벨 템플릿(고객 전용/공용)으로 라벨 발행.
공통 (All)MD-25 사유코드가 비가동·폐기·반품 등 예외 사유를, MD-26 공통코드가 전 모듈 드롭다운·열거형을 공급한다.
⚙ One Chain, Nine Masters · 하나의 체인, 9개 마스터검사 → 불량 판정·원인 → 고객·출하처 → 포장·라벨 → 공통 사유·코드 — 이 한 흐름의 각 단계가 카테고리 E의 마스터에 의존한다. 어느 하나라도 누락되면 QC 또는 FG 화면이 차단된다. · Each step of the inspect-to-ship flow depends on a Category E master.
04Maintenance Workflow마스터 유지보수 흐름
Inspection Standards (MD-06) are defined per item / process step with spec limits, sampling plan and CTQ flag.
품목·공정 단계별 검사기준(MD-06)을 규격 한계·샘플링 계획·CTQ 플래그와 함께 정의한다.
Defect Codes (MD-21) and the 4M1E Defect Cause hierarchy (MD-22) are maintained as production reveals new modes.
불량코드(MD-21)와 4M1E 불량원인 계층(MD-22)을 관리 — 생산 중 새 불량 모드 발견 시 추가한다.
Customers (MD-12) and Shipment Destinations (MD-11) are registered with EDI flags and transit lead-times.
고객사(MD-12)·출하처(MD-11)를 EDI 플래그·운송 리드타임과 함께 등록한다.
Packaging Specs and Label Templates are set per item / customer before FG go-live; templates are versioned.
FG 가동 전 품목·고객별 포장사양·라벨 템플릿을 설정 — 템플릿은 버전 관리된다.
Reason Codes and Common Code groups are governed centrally; a code used by transactions is deactivated, never deleted.
사유코드·공통코드 그룹은 중앙 관리 — 트랜잭션 참조 코드는 삭제 대신 비활성화한다.

BR-MD-03

BR-MD-13
Registering a CRITICAL-severity MD-21 Defect Code auto-holds the lot and notifies the quality manager.
CRITICAL MD-21 불량코드 등록 시 LOT 자동 HOLD + 품질 관리자 통보.
Auto
BR-MD-14
A CAPA final cause must be a RootCause leaf node in the MD-22 4M1E hierarchy.
CAPA 최종 원인은 MD-22 4M1E 계층의 RootCause 말단이어야 함.
Block
BR-MD-15
MD-22 Defect Cause is classified by the 4M1E axis; RPT-02 aggregates causes along it.
MD-22 불량원인은 4M1E 축 분류 — RPT-02가 이 축으로 원인 집계.
Auto
BR-MD-16
MD-23 Packaging Spec three-tier hierarchy must be defined before FG-03 packing.
MD-23 포장사양 3단계 계층이 있어야 FG-03 포장 가능.
Validation
BR-MD-17
A customer-specific MD-24 Label Template overrides the shared one; the highest ACTIVE version prints.
고객 전용 MD-24 라벨 템플릿이 공용보다 우선 — 최신 ACTIVE 버전 인쇄.
Auto
BR-MD-18
An MD-25 Reason Code with RequiresComment=TRUE blocks the action until a comment is entered.
코멘트 필수 MD-25 사유코드는 코멘트 입력 전 행위 차단.
Block
BR-MD-19
Every A-MES dropdown loads from MD-26 Common Code by GroupCode; a code used by transactions cannot be deleted.
A-MES 드롭다운은 MD-26 공통코드를 GroupCode로 로드 — 참조 코드 삭제 불가.
Block
