01
Overview
화면 개요 · 목적 · 컨텍스트
PLAN manager reviews supply plans against FG inventory, equipment load, and forecast reference. For each plan line, the system calculates Net Required Qty (= Order Qty − FG Stock), shows current line load %, and compares vs forecast. Items selected via checkbox are batch-released to PP-04 as Work Orders.
PLAN 담당자가 공급계획을 완제품 재고·설비 부하·Forecast와 대조하여 생산 가능성 검토. 행별로 순생산 필요량(Order − FG Stock) 자동 계산, 라인 부하율 표시, Forecast 대비. 체크박스로 선택된 행은 PP-04 WO 일괄 생성으로 전송.

04
Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
USER page.load · open PP-03
Loads all confirmed SOs from PP_SupplyPlan with status Open in date range. Calculates Net Req for each.
기간 내 확정 SO 로드 + 행별 Net Req 자동 계산.
APIGET /api/pp/plan?from=...&to=...
2
SYSTEM fetch · FG inventory
Real-time fetch from WH_Inventory for each item's FG (finished goods) on-hand qty.
각 품목별 완제품 재고를 WH_Inventory에서 실시간 조회.
QUERYSELECT item_code, SUM(on_hand) FROM wh_inventory WHERE warehouse='WH-FG' GROUP BY item_code
3
SYSTEM fetch · line load
Queries MNT_Equipment for current capacity utilization per line.
MNT_Equipment에서 라인별 현재 가동률 조회.
EFFECTEach row shows line load bar
4
USER check · row selection
User selects rows by checkbox. System dynamically updates batch size and total qty in footer.
행 체크 시 일괄 처리 건수·총 수량 푸터에 실시간 갱신.
UIFooter counter updates · disable conflicting rows
5
USER click · [Batch Create WO]
Triggers WO creation for all checked rows. Each becomes a Draft WO with routing per BOP (MD-04).
체크된 행에 대해 WO 일괄 생성. 각각 Draft 상태, BOP(MD-04) 기준 라우팅 자동 적용.
APIPOST /api/pp/wo/batch · returns array of wo_ids
6
SYSTEM validate · Phase-0
Before WO creation, validates that all required master data is complete (BOM/BOP/Inspection Standard).
WO 생성 전 기준정보 완성도 검증(BOM/BOP/검사 기준).
BLOCKIf any missing → BR-PP-001 block · skip row · report
7
SYSTEM redirect · to PP-04
Successful batch redirects to PP-04 with new WOs highlighted.
성공 시 PP-04로 이동하며 신규 WO 하이라이트.
NAV/pp/wo?highlight=WO-20250801-001,002,003

Exception Handling
예외 처리 · 오류 코드 · 복구 절차
E-PP03-001
Phase-0 IncompletePhase-0 미완료
CauseSelected item has missing master data (BOM/BOP/Insp).
DetectValidation on Batch Create.
ActionBlock creation for that row. Show specific missing master.
RecoverMD team completes master in MD-01~06.
⛔ PHASE-0 INCOMPLETE
E-PP03-002
Insufficient Capacity능력 부족
CauseLine load already > 95% for the period.
DetectCapacity check on select.
ActionYellow warning. Allow creation but suggest scheduling later.
RecoverAdjust due date or split WO.
⚠ CAPACITY TIGHT

Negative Net Req음수 순생산
CauseFG stock exceeds Order Qty (overstock).
DetectCalc check.
ActionShow negative in green (good). Disable WO creation (no need to produce).
RecoverSkip this row.
ℹ SUFFICIENT STOCK

BOP Not DefinedBOP 미정의

ActionBlock creation. Cannot determine production sequence.
