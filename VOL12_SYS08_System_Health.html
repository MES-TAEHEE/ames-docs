01Overview개요
EN · Description The System Health Dashboard gives the Admin a single operations view of the A-MES platform itself — not the factory, but the system running it. It shows application/DB server status, CPU / memory / disk usage, uptime, the last successful database backup, and the status of scheduled jobs (day-end close, report generation, ERP sync). Each metric has a threshold; exceeding it turns the tile amber or red and fires a notification. It lets the Admin catch infrastructure problems — a full disk, a failed backup, a stuck job — before they cause an outage. Auto-refresh every 30 seconds.
KO · 설명 시스템 헬스 대시보드는 Admin에게 A-MES 플랫폼 자체의 단일 운영 뷰를 제공한다 — 공장이 아니라 그것을 구동하는 시스템이다. 애플리케이션/DB 서버 상태, CPU·메모리·디스크 사용률, 가동시간, 마지막 성공 DB 백업, 스케줄 잡 상태(일 마감·보고서 생성·ERP 동기)를 표시한다. 각 지표에 임계값이 있어 초과 시 타일이 주황·적색으로 바뀌고 알림이 발생한다. 디스크 풀, 백업 실패, 멈춘 잡 같은 인프라 문제를 장애로 번지기 전에 잡게 한다. 30초 자동 갱신.
📌 Platform vs. Factory · 플랫폼 감시SYS-04가 외부 인터페이스를, SYS-08은 A-MES 자체 인프라(서버·DB·백업·잡)를 감시한다 — 둘이 합쳐 시스템 안정성의 완전한 그림을 이룬다. · SYS-04 watches external interfaces; SYS-08 watches A-MES's own infrastructure.

03Health Metrics헬스 지표 구성
Metric GroupWhat Is Monitored · 감시 내용
Server · 서버앱 서버·DB 서버 응답 상태 (OK/ERROR), 가동시간(uptime), 최근 30일 가동률.
Resource · 자원CPU·메모리·디스크 사용률 — 각 임계값(80/75/85%) 초과 시 경고.
Backup · 백업마지막 성공 DB 백업 시각·크기 — 24h 초과 미실행 시 ERROR.
Scheduled Job · 스케줄 잡일 마감·월간 KPI·DB 백업·ERP 동기 등 배치 잡의 실행 결과·소요·상태.
⚙ Backup is Critical · 백업 우선 감시DB 백업 실패는 다른 어떤 경고보다 우선한다 — 24시간 내 성공 백업이 없으면 적색 + 즉시 Admin SMS 알림이 발생한다. · A missing DB backup outranks every other warning — no successful backup within 24h triggers a red flag and an immediate SMS to Admin.
04Workflow작업 흐름
A monitoring agent collects server, resource, backup, and job metrics continuously.
모니터링 에이전트가 서버·자원·백업·잡 지표를 지속 수집한다.
The dashboard auto-refreshes every 30 seconds, comparing each metric to its threshold.
대시보드가 30초마다 자동 갱신, 각 지표를 임계값과 비교한다.
A metric over threshold turns its tile amber/red and fires a SYS-06 notification.
임계 초과 지표는 타일이 주황/적색으로 바뀌고 SYS-06 알림이 발생한다.
A failed scheduled job or missing backup is flagged with the highest priority.
잡 실패·백업 누락은 최고 우선순위로 플래그된다.
Admin investigates and resolves; metrics return to OK on the next collection cycle.
Admin이 조사·조치 — 다음 수집 주기에 지표가 OK로 복귀한다.
05Business Rules업무 규칙
RuleDescriptionAction
BR-SYS-32
A metric exceeding its threshold auto-changes status to WARNING/ERROR and fires a notification.
임계 초과 지표는 WARNING/ERROR 자동 전환 + 알림 발생.
Auto
BR-SYS-33
No successful DB backup within 24 hours raises a red ERROR and an immediate SMS to Admin.
24시간 내 성공 DB 백업 없으면 적색 ERROR + Admin SMS 즉시.
Critical
BR-SYS-34
A failed or timed-out scheduled job is flagged and notified; it does not auto-retry silently.
실패·타임아웃 스케줄 잡은 플래그·알림 — 조용한 자동 재시도 안 함.
Notify
BR-SYS-35
Health metrics are read-only; SYS-08 monitors and alerts but never alters system state.
헬스 지표는 읽기 전용 — SYS-08은 감시·경보만, 시스템 상태 변경 안 함.
Read-Only
