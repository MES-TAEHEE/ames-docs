01Overview개요
EN · Description The QC-side browser and editor for inspection standards, mirroring MD-08 master data with QC-specific add-ons (sampling tables, AQL, photo references, customer spec attachments). Inspectors view current effective standards before each inspection (Read-only). QC Lead drafts revisions, QC Manager approves; old versions remain accessible for audit. Each inspection record (QC-01/02/03) links to the exact StdID + VerNo used, ensuring traceability for customer audits and CAPA root-cause analysis.
KO · 설명 MD-08 마스터를 미러링한 QC 측 검사기준서 조회·편집 화면 (QC 전용 추가: 샘플링 표·AQL·사진 참조·고객 사양 첨부). 검사원은 각 검사 전 현재 유효 기준 조회 (Read-only). QC Lead가 개정 초안, QC Manager가 승인. 구 버전은 감사 대응 위해 접근 가능. 각 검사 기록(QC-01/02/03)은 사용된 StdID + VerNo와 정확히 연결되어 고객 감사·CAPA 근본원인 분석 시 추적 가능.

03Versioning & Approval버전 관리 + 결재
StatusDescription
DRAFTQC Lead가 신규 또는 개정 버전 작성 중. 검사에 사용 불가. 자동 저장 (autosave).
IN_REVIEWQC Manager 결재 대기. 7d 미결재 시 reminder, 14d 시 Plant Manager 에스컬레이션.
APPROVED유효 버전. Effective Date 이후 검사에 자동 적용. 이전 버전은 OBSOLETE 자동 전환.
OBSOLETE구 버전. 검사에 사용 불가지만 감사 대응 위해 5년 이상 보관. 신규 검사에 노출되지 않음.
Version PinningBehavior
At Inspection StartQC-01/02/03 발행 시 그 시점 APPROVED 버전 StdID + VerNo를 검사 record에 hardcoded. 검사 중 기준 개정되어도 그 검사는 유지.
Customer Spec ChangeCustomerSpecURL 변경 감지 시 QC Lead에게 자동 알림 + DRAFT 신규 버전 자동 생성 (변경 사항 표시).
CAPA-Driven RevisionCAPA-06 ACT 단계에서 SOP/Std 갱신 결정 시 자동 DRAFT 생성, CAPALinkID로 연결.
04Workflow작업 흐름
Inspector starts QC-01/02/03 → system auto-pulls current APPROVED Std for that item/customer; reads it before measuring.
검사원이 QC-01/02/03 시작 → 시스템이 현재 APPROVED 기준 자동 로드, 측정 전 조회.
Revision trigger (4 sources): customer spec change · CAPA-06 ACT · supplier change · routine review (annual).
개정 트리거(4종): 고객 사양 변경·CAPA ACT·공급사 변경·정기 검토(연 1회).
QC Lead clicks "Draft New Version" → autosave; edits items, KPIs, sampling rules; attaches updated customer PDF.
QC Lead가 [신규 버전 초안] 클릭, 항목·KPI·샘플링 편집 + 고객 PDF 첨부.
Submit for Review → Status: DRAFT → IN_REVIEW; QC Manager notified (Slack + email).
결재 상신 → DRAFT → IN_REVIEW, QC Manager 알림 (Slack + 이메일).
QC Manager reviews "diff vs current" view; approves or sends back with comments.
QC Manager가 현 버전 대비 diff 검토, 승인 또는 코멘트와 함께 반려.
Approved → Effective Date set; previous version auto-marked OBSOLETE; inspectors notified.
승인 시 Effective Date 설정, 이전 버전 자동 OBSOLETE, 검사원 알림.
All inspections after Effective Date auto-use new version; in-flight inspections continue with old (pinned).
Effective Date 이후 검사는 신규 버전 자동 적용, 진행 중 검사는 기존 버전 유지 (Pinned).
All versions retained ≥ 5 years; auditors can browse historical versions and "what was used" for each inspection.
전 버전 5년 이상 보관, 감사 시 각 검사가 사용한 버전 추적 가능.
05Business Rules업무 규칙
RuleDescriptionAction
BR-QC-27
Only APPROVED standards can be applied to inspections; DRAFT/IN_REVIEW blocked.
APPROVED만 검사에 사용, DRAFT/IN_REVIEW 차단.
Block
BR-QC-28
Each inspection record pins (StdID, VerNo) at start; revisions during in-flight do not affect.
검사 시작 시 (StdID, VerNo) 락, 진행 중 개정은 영향 없음.
Lock
BR-QC-29
Approval requires QC Manager role (Lead alone insufficient).
승인은 QC Manager 권한 필수, Lead 단독 불가.
Approve
BR-QC-30
IN_REVIEW > 7d → reminder; > 14d → Plant Manager auto-escalation.
결재 대기 7일 reminder, 14일 시 공장장 에스컬레이션.
Escalate
BR-QC-31
Customer spec PDF change detection auto-creates DRAFT for QC Lead review.
고객 사양 PDF 변경 감지 시 DRAFT 자동 생성.
Auto
BR-QC-32
OBSOLETE standards retained ≥ 5 years for audit traceability.
OBSOLETE 기준 5년 이상 보관 (감사 추적).
Retain
