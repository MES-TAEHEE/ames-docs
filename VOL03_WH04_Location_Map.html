01
Overview
화면 개요
Visual grid map of all warehouse zones and shelf locations. Each cell represents one storage cell and is color-coded by fill rate: empty (grey), partial (yellow), full (green), or over-capacity (red). Clicking a cell opens a detail panel with stored items, quantities, inbound dates, and a link to the cell's stock movement history. Location codes follow the convention [Warehouse]-[Zone]-[Row]-[Col]-[Level].
모든 창고 구역과 선반 로케이션을 그리드 형태로 시각화한 화면입니다. 각 셀은 한 개의 저장 위치를 나타내며 충전율에 따라 색상이 다릅니다: 비어있음(회색), 일부 사용(노랑), 가득 참(초록), 초과(빨강). 셀 클릭 시 보관 품목·수량·입고일·이력 링크가 포함된 상세 패널이 열립니다. 로케이션 코드 체계: [창고]-[구역]-[행]-[열]-[층].

04
Event & Action Flow
이벤트 · 액션 흐름
1
Userpage.load · WH-04 진입
User opens WH-04. System loads location master + current inventory per location. Renders grid color-coded by fill rate. WebSocket subscribes to inventory deltas.
WH-04 진입 시 위치 마스터 + 현재 재고를 로딩하여 충전율 색상 그리드 렌더링. WebSocket 구독.
APIGET /api/wh/locations · WS /ws/location.subscribe.
2
Userhover · cell
User hovers a cell. Tooltip appears after 200ms with item code, qty, fill %, FIFO date.
셀 호버 시 200ms 지연 후 툴팁 표시(품목·수량·충전율·FIFO 일자).
UI No network call · uses cached data from initial load.
3
Userclick · cell
User clicks a cell. Side panel opens with full detail + recent activity. URL updates with ?focus=WH-FAB-D-01-1 for shareable deep link.
셀 클릭 시 상세 패널이 열리고 최근 거래 표시. URL이 ?focus=...로 갱신되어 딥링크 공유 가능.
APIGET /api/wh/location/:code/detail.
4
Systempush · location.updated
When WH-02 inbound or WH-07 release commits, server pushes the location delta. Cell color and tooltip update with smooth transition animation.
WH-02·WH-07 커밋 시 위치 변경분 푸시. 셀 색상·툴팁이 부드러운 전환과 함께 갱신.
PAYLOAD{location_code, on_hand, capacity, fill_rate}.
5
Userclick · [Print QR Labels]
Manager selects one or more cells (or entire zone) and clicks Print. System generates a printable HTML page with QR codes for each location.
매니저가 셀(또는 구역)을 선택하고 [QR 출력] 클릭 시 QR 코드 포함 인쇄 페이지 생성.
REQUIRE Role WH.Manage · QR encodes location_code · ZPL or HTML format.
✓
Userclick · "View history"
From side panel, user clicks "View full history" link. Navigates to WH-08 with location pre-filter applied.
상세 패널에서 [전체 이력] 클릭 시 WH-08로 이동(해당 위치 필터 사전 적용).
NAV/wh/history?location=WH-FAB-D-01-1.
05
Exception & Error Handling
예외 · 에러 처리
E-WH04-001
Over-Capacity Detected초과 적재 감지
Cause
Cell's on_hand exceeds defined capacity (data anomaly or manual override). 정의 용량 초과(이상·수동 입력).
Detect
Cell render: on_hand > capacity.
Action
Cell color = red. KPI "Over Cap" increments. Alert to WH manager. Recommend relocation.
Recovery
Manager redistributes via WH-05 or expands capacity in MD-11.
⛔ OVER CAPACITY — RELOCATE
E-WH04-002
Location Blocked위치 차단
Cause
Cell marked as Blocked or Maintenance in MD-11. 차단·정비 상태.
Detect
md_location.status != 'Active'.
Action
Cell shown with diagonal stripe pattern. WH-02/WH-07 reject scanning this location until reactivated.
Recovery
Manager re-activates location in MD-11.
⚠ BLOCKED — SEE MD-11
E-WH04-003
Mixed Items in Cell셀 내 품목 혼재
Cause
Multiple item codes stored in same cell (violates BR-WH-07 if not allowed). 한 셀에 복수 품목 보관.
Detect
COUNT(DISTINCT item_code) > 1 per location.
Action
Cell badge shows "Mixed". Detail panel lists all items separately. Yellow warning icon.
Recovery
If policy disallows, WH-05 adjustment to relocate.
⚠ MIXED ITEMS — REVIEW
E-WH04-004
QR Print Permission DeniedQR 출력 권한 없음
Cause
Non-manager attempts to print QR labels. 매니저 권한 없는 사용자가 출력 시도.
Detect
Server RBAC check on print request.
Action
Return 403. Modal: "QR label printing requires WH.Manage role. Contact your manager."
Recovery
Request from manager or escalate.
⛔ ACCESS DENIED — WH.MANAGE REQUIRED
E-WH04-005
Empty Warehouse View빈 창고 뷰
Cause
Selected warehouse has no defined locations in MD-11. 선택 창고에 위치 미정의.
Detect
Locations query returns empty.
Action
Show illustration: "No locations defined for this warehouse. Configure in MD-11."
Recovery
Master data manager defines locations.
ℹ NO LOCATIONS — SEE MD-11
E-WH04-006
Rendering Performance렌더링 성능
Cause
Large warehouse (> 1,000 cells) causes slow render. 대형 창고(1000셀 초과) 렌더링 지연.
Detect
Cell count check at load time.
Action
Activate virtualized rendering (only visible cells). Show progress bar during initial render.
Recovery
User applies warehouse/level filter to reduce scope.
ℹ LARGE GRID — VIRTUALIZED

07
Applicable Business Rules
적용 업무 규칙
Rule IDDescriptionAction
BR-WH-07
Each storage cell should hold a single item code. Mixed-item cells require manager approval and are flagged in the map view.
각 셀은 단일 품목 보관 원칙. 혼재 시 매니저 승인 + 맵에 플래그 표시.
Warn
BR-WH-08
Blocked or maintenance-status locations cannot receive new inbound. WH-02 / WH-05 scans are rejected until reactivated.
차단·정비 상태 위치 신규 입고 불가. WH-02·WH-05 스캔 거부.
Block
BR-WH-09
Location capacity is defined in MD-11 and is hard-enforced at WH-02 receiving time. Over-capacity stocking is rejected unless override approved.
로케이션 용량은 MD-11에 정의되며 WH-02 입고 시 강제 적용. 초과 시 거부(승인 시 예외).
Block
