Overview
화면 개요 · 목적 · 컨텍스트
Generates Purchase Requisitions from MRP shortage results and transmits them to SAP B1 via Service Layer API (POST /PurchaseRequests). Tracks SAP B1 DocNum after successful transmission. Failures auto-retry 3× with backoff; persistent failure escalates to PLAN manager. Approved PRs flow to PO creation in SAP B1 and back to WH-01 (inbound schedule) as expected receipts.
MRP 부족 결과 기반 PR 생성 → SAP B1 Service Layer API(POST /PurchaseRequests) 전송. 성공 시 SAP B1 DocNum 추적. 실패 시 3회 자동 재시도, 지속 실패 시 PLAN 관리자 에스컬레이션. 승인된 PR은 SAP B1에서 PO 생성 → WH-01 입고 예정으로 반영.

04
Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
USER click · select Drafts
User multi-selects Draft PRs from list.
사용자가 Draft PR 다중 선택.
UICheckbox column · footer shows selected count
2
USER click · [📤 Send]
Triggers transmission. Each PR sent individually via API to SAP B1 Service Layer.
전송 트리거. 각 PR이 SAP B1 Service Layer API로 개별 전송.
APIPOST /b1s/v1/PurchaseRequests per row
3
SYSTEM transmit · retry on fail
On failure, auto-retries 3× with backoff (5s, 15s, 45s). After 3 failures, marks Failed.
실패 시 지수 backoff(5s/15s/45s)로 3회 자동 재시도. 3회 실패 후 Failed 상태.
EFFECTBR-PP-006 · log all attempts in PP_PRSendLog
4
SYSTEM track · DocNum
On success, SAP B1 returns DocNum. Stored in PP_PR.SAP_DocNum. Status = Sent.
성공 시 SAP B1이 DocNum 반환. PP_PR.SAP_DocNum에 저장 + 상태 Sent.
DBStatus: Draft → Sent
5
SCHEDULED poll · PO creation
Every 1 hour, polls SAP B1 to check if any Sent PR has been converted to PO.
1시간마다 SAP B1 폴링하여 Sent PR의 PO 전환 여부 확인.
APIGET /b1s/v1/PurchaseOrders?$filter=PurchaseRequestNum eq ...
6
SYSTEM update · PO Created
When SAP B1 generates PO from PR, status → Approved (PO). Triggers WH-01 inbound schedule update.
SAP B1에서 PR이 PO로 전환되면 상태 Approved (PO) → WH-01 입고 예정 갱신.
DOWNSTREAMWH_PurchaseOrder INSERT + WH-01 KPI refresh
7
USER click · [🔁 Retry]
Manual retry for persistently failed PRs (after fixing root cause).
지속 실패 PR에 대한 수동 재시도 (근본 원인 해결 후).
RESETRetry counter reset, attempt fresh transmission
05
Exception Handling
예외 처리 · 오류 코드 · 복구 절차
E-PP06-001
SAP Auth FailureSAP 인증 실패
CauseService Layer token expired or invalid.
Detect401 response.
ActionAuto re-login. Retry. If still fails, alert SYS.
RecoverSYS updates credential.
⛔ AUTH FAILED
E-PP06-002
PR Validation RejectPR 검증 거부
CauseSAP B1 rejects PR (invalid vendor, item, etc.).
Detect400 response with reason.
ActionMark Failed with SAP error message. Skip retries.
RecoverUser fixes data in PP-06 or MD masters.
⛔ SAP REJECTED
E-PP06-003
Network Timeout네트워크 타임아웃
CauseService Layer response > 30s.
DetectHeartbeat.
ActionRetry 3× per BR-PP-006.
RecoverCheck SAP B1 health.
⚠ TIMEOUT · RETRYING
E-PP06-004
Duplicate PRPR 중복
CauseSame item+date already has open PR.
DetectPre-send check.
ActionShow duplicate, ask user: send anyway or merge.
RecoverMerge or cancel duplicate.
⚠ DUPLICATE
E-PP06-005
Insufficient Vendor거래선 부재
CauseItem has no default vendor in MD-07.
DetectPre-send.
ActionBlock send. Cannot create PR without vendor.
RecoverMD team sets default vendor in MD-07.
⛔ NO VENDOR
E-PP06-006
SAP B1 DownSAP 다운
CauseRepeated 5xx errors from SAP.
DetectStatus check.
ActionHalt all sends. Banner: 'SAP B1 unavailable'.
RecoverWait for SAP recovery, then resume.
⛔ SAP DOWN
