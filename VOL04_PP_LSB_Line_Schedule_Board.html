01
Overview
화면 개요 · 목적 · 컨텍스트
A minute-level Gantt board that schedules Work Orders onto each line's own time pattern (MD-29). Every line row renders its active MD_LineTimePattern as background bands — green OPERATING windows and amber PLANNED_DOWN windows — and WO blocks (routing A blue / B orange) are dragged onto operating windows only. Available capacity per line = Σ OPERATING minutes; the board blocks any WO that would overflow an operating window or land on planned downtime.
작업지시(WO)를 각 라인의 고유 시간패턴(MD-29) 위에 배치하는 분단위 간트 보드. 각 라인 행은 활성 MD_LineTimePattern을 배경 밴드로 렌더링한다 — 녹색 OPERATING 시간창, 호박색 PLANNED_DOWN 시간창 — 그리고 WO 블록(라우팅 A 파랑/B 주황)은 가동 시간창에만 드래그 배치된다. 라인별 가용 캐파 = Σ OPERATING 분; 가동창을 초과하거나 계획 비가동에 걸치는 WO는 차단된다.

04
Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
USERpage.load · select date
Resolves each line's DayType from MD-14 Calendar, loads the matching ACTIVE MD_LineTimePattern and renders minute-level bands.
MD-14 캘린더로 라인별 요일유형 판별 → ACTIVE 시간패턴 로드 → 분단위 밴드 렌더링.
APIGET /api/pp/line-schedule?date=2026-05-21
2
USERdrag · WO block → operating window
User drags a released WO onto an OPERATING segment; system snaps to 1-minute grid and computes end minute from WO standard run time.
발행 WO를 OPERATING 세그먼트에 드래그 → 1분 그리드 스냅 → WO 표준시간으로 종료 분 계산.
CALCendMin = startMin + ceil(WO.qty × cycleTime / 60)
3
SYSTEMvalidate · window fit
Checks the WO block stays inside one OPERATING window and does not cross a PLANNED_DOWN segment; overflow flips the block red.
WO 블록이 단일 OPERATING 창 내에 있고 PLANNED_DOWN을 가로지르지 않는지 검증 — 초과 시 빨강 표시.
RULEBR-PP-031 · overflow → block + E-LSB-001
4
USERclick · [Auto-Schedule]
Heuristically fills all unscheduled released WOs into available OPERATING minutes, ordered by D-Day ascending.
미배치 발행 WO를 D-Day 오름차순으로 가용 OPERATING 분에 자동 충전.
APIPOST /api/pp/line-schedule/auto-fill
5
USERclick · [Publish]
Commits PP_LineSchedule, pushes the day's schedule to INJ/PNT POP screens and sets the PP-ODM operating baseline.
PP_LineSchedule 확정 → INJ/PNT POP에 일정 푸시 → PP-ODM 가동 기준선 설정.
DOWNSTREAMPP-ODM baseline · PP-OEE plan · POP work queue
05
Exception Handling
예외 처리 · 오류 코드
E-LSB-001
Window Overflow가동창 초과
CauseWO block exceeds the OPERATING window or crosses a PLANNED_DOWN segment.
DetectOn drop / auto-fill.
ActionBlock turns red, Publish disabled. Split WO or move to another window.
⛔ OVERFLOW · BLOCKED
E-LSB-002
No Active Pattern패턴 미정의
CauseLine has no ACTIVE MD_LineTimePattern for the resolved DayType.
DetectOn page load.
ActionLine row greyed; prompt to define pattern in MD-29.
⛔ PATTERN MISSING
E-LSB-003
Capacity Over 100%부하율 초과
CauseΣ WO minutes exceeds Σ OPERATING minutes for a line.
DetectOn every schedule change.
ActionCapacity bar red; suggest moving WO to another line / day.
⚠ OVER-CAPACITY
E-LSB-004
Pattern Changed After Publish발행 후 패턴 변경
CauseMD-29 pattern edited after the day was published.
DetectPattern version compare.
ActionBanner: re-validate & re-publish required.
⚠ STALE SCHEDULE
