01Overview개요
EN · Description System Configuration is the no-code control panel for system-wide parameters and common codes. Parameters govern behavior — session timeout, default thresholds (defect-rate alert, OEE target), document numbering rules, retention periods. Common codes are the shared dropdown lists used across modules — defect-type codes, downtime reason codes, units of measure, delay reason codes. Admins add, edit, or deactivate these without code changes. A change propagates to every module that reads the code/parameter. All edits are audit-logged.
KO · 설명 시스템 설정은 시스템 전역 파라미터와 공통 코드의 노코드 통제판이다. 파라미터는 동작을 좌우한다 — 세션 타임아웃, 기본 임계값(불량율 경보·OEE 목표), 문서 채번 규칙, 보존 기간. 공통 코드는 모듈 전반에서 쓰이는 공유 드롭다운 목록이다 — 불량 유형 코드, 비가동 사유코드, 단위, 지연 사유코드. Admin이 코드 변경 없이 이를 추가·수정·비활성화한다. 변경은 해당 코드/파라미터를 읽는 모든 모듈에 전파된다. 전 편집은 감사 로그에 기록된다.
📌 Config vs. Master Data · 설정과 마스터의 경계SYS-07은 시스템 동작 파라미터·공통 코드를, MD(VOL13)는 품목·거래선·BOM 등 업무 마스터를 다룬다. 공통 코드는 SYS, 비즈니스 마스터는 MD. · SYS-07 holds system parameters and shared codes; MD holds business master data (items, clients, BOM).

🔐 Security · 보안세션 타임아웃, 로그인 실패 한도, 비밀번호 정책, 비밀번호 만료 주기.
⚙ Thresholds · 임계값불량율 경보, OEE 목표, 일정 허용, 부동재고 기준, 안전재고 경보 등.
🔢 Numbering · 채번 규칙WO·거래명세서·보고서 등 문서 번호 형식 (접두어·일자·시퀀스).
🗄 Retention · 보존 기간감사 로그(5년)·트랜잭션·보고서 보존 기간.
⚠ Defect Codes · 불량 코드QC·생산 불량 유형 코드 — RPT-02 파레토 그룹 기준.
⏱ Downtime Reasons · 비가동 사유MNT-06 비가동 사유코드 (계획/비계획/물류 등).
📏 Units · 단위수량·중량·길이 단위 (EA/kg/m...).
🚚 Delay Reasons · 지연 사유RPT-03 출하 지연·RPT-08 일정 지연 사유코드.
⚙ Deactivate, Don't Delete · 비활성화 원칙이미 트랜잭션에 사용된 공통 코드는 삭제할 수 없다 — 비활성화하면 신규 입력 목록에서만 제외되고 과거 데이터는 그대로 유지된다. · A common code used in past transactions cannot be deleted, only deactivated, preserving historical data.
04Workflow작업 흐름
Admin opens SYS-07 and selects a category from the left tree.
Admin이 SYS-07을 열고 좌측 트리에서 카테고리를 선택한다.
Admin edits a parameter value or adds/edits/deactivates a common code.
Admin이 파라미터 값을 수정하거나 공통 코드를 추가·수정·비활성화한다.
On Save, the new value propagates immediately to every module that reads it.
저장 시 새 값이 이를 읽는 모든 모듈에 즉시 전파된다.
A code already used in transactions can only be deactivated, never deleted.
트랜잭션에 사용된 코드는 비활성화만 가능 — 삭제 불가.
Every config change is written to the SYS-05 audit log with before/after values.
전 설정 변경은 전후값과 함께 SYS-05 감사 로그에 기록된다.
05Business Rules업무 규칙
RuleDescriptionAction
BR-SYS-28
A parameter change propagates immediately to all modules that reference it — no restart.
파라미터 변경은 참조 모듈 전체에 즉시 전파 — 재기동 없음.
Immediate
BR-SYS-29
A common code used in any transaction cannot be deleted — only deactivated.
트랜잭션에 사용된 공통 코드는 삭제 불가 — 비활성화만 가능.
Block
BR-SYS-30
Numbering rules are validated for uniqueness; a sequence cannot be reset while active.
채번 규칙은 유일성 검증 — 운영 중 시퀀스 초기화 불가.
Validate
BR-SYS-31
Every parameter and code change is audit-logged with before/after values and actor.
전 파라미터·코드 변경은 전후값·주체와 함께 감사 로그 기록.
Audit
