01Overview개요
EN · Description The Report Center is the single entry point to every A-MES report. It presents a catalog of all standard reports (RPT-01~08) and saved custom reports (RPT-10), each with its description, owner, and last-run status. From here users open past report instances, browse run history, configure scheduled generation and email recipients, and manage access permissions by role. It does not compute report content itself — it is the catalog, scheduler, and access-control layer that ties the report set together.
KO · 설명 리포트 센터는 모든 A-MES 보고서로 가는 단일 진입점이다. 전 표준 보고서(RPT-01~08)와 저장된 사용자 정의 보고서(RPT-10)의 카탈로그를 설명·소유자·최근 실행 상태와 함께 제시한다. 사용자는 여기서 과거 보고서 인스턴스를 열고, 실행 이력을 조회하며, 정기 생성·이메일 수신자를 설정하고, 역할별 접근 권한을 관리한다. 보고서 내용을 직접 계산하지 않는다 — 보고서 집합을 묶는 카탈로그·스케줄러·접근 제어 계층이다.
📌 Hub Role · 허브 역할각 보고서(RPT-01~08, 10)는 독립 화면이지만, RPT-09는 그것을 한 곳에서 발견·예약·통제하게 한다. 신규 사용자의 출발점이자 RPT Admin의 운영 콘솔이다. · Each report is its own screen, but RPT-09 makes them discoverable, schedulable, and governed from one place.

03Catalog & Permissions카탈로그 + 권한
AspectDescription · 설명
Catalog Entry표준 8종은 시스템 사전 등록 — 삭제 불가. 커스텀(RPT-10)은 소유자가 카탈로그에 게시.
Generation ModeAUTO(마감 연동) / SCHEDULED(cron) / ON_DEMAND(수동 실행) — 모드별 아이콘 표시.
Run History전 실행 인스턴스 보관 — 과거 시점 보고서 재열람 가능.
View Permission역할별 조회 허용 — 미허용 역할은 카탈로그에서 보이지 않음.
Export PermissionExcel/PDF 출력·이메일 발송은 별도 권한 — 민감 보고서 통제.
Email Schedule정기 생성 시 수신자에게 자동 발송 (SYS 이메일 큐 연계).
🔐 Permission Source · 권한 출처역할(Role)은 SYS 사용자·권한 모듈에서 정의되며, RPT-09는 보고서별로 그 역할에 조회·출력 권한을 매핑한다. · Roles are defined in the SYS module; RPT-09 maps view/export rights per report onto those roles.
04Workflow작업 흐름
User opens Report Center; the catalog shows only reports their role may view.
사용자가 리포트 센터를 연다 — 카탈로그는 역할이 조회 가능한 보고서만 표시.
User clicks a report card to open its latest instance, or picks a past period from run history.
보고서 카드 클릭 → 최신 인스턴스 열람, 또는 실행 이력에서 과거 기간 선택.
RPT Admin configures schedule (cron), email recipients, and view/export permissions per report.
RPT Admin이 보고서별 스케줄(cron)·이메일 수신자·조회/출력 권한을 설정한다.
Scheduled reports auto-generate at the cron time and dispatch emails via the SYS queue.
정기 보고서는 cron 시각에 자동 생성되고 SYS 큐로 이메일 발송된다.
"+ Custom Report" launches RPT-10 Builder; the saved report is published back to the catalog.
"+ Custom Report" → RPT-10 빌더 실행, 저장된 보고서는 카탈로그에 게시된다.
05Business Rules업무 규칙
RuleDescriptionAction
BR-RPT-31
The 8 standard reports are system-registered and cannot be deleted from the catalog.
표준 8종은 시스템 등록 — 카탈로그에서 삭제 불가.
Lock
BR-RPT-32
A report is visible in the catalog only if the user's role is in its ViewRoleSet.
사용자 역할이 ViewRoleSet에 있을 때만 카탈로그에 표시.
Restrict
BR-RPT-33
Export / email dispatch requires ExportRoleSet membership, separate from view rights.
출력·이메일 발송은 조회와 별개의 ExportRoleSet 권한 필요.
Restrict
BR-RPT-34
All report runs (auto, scheduled, on-demand) are logged with actor and timestamp; history is immutable.
전 실행(자동·정기·온디맨드)은 주체·시각 기록 — 이력 불변.
Audit
