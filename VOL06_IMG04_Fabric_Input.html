01Overview개요
EN · Description The signature screen of the IMG module. Before mounting a fabric roll on the wrapping machine, the operator scans the roll's QR code via PDA. The system instantly cross-checks the scanned color code against the WO's required color. Match → proceed. Mismatch → input blocked, screen turns red, alarm sounds (BR-IMG-02). After confirmed input, fabric length is auto-deducted from WH stock per BOM unit on every IMG-03 production confirm. This is the #1 differentiator vs INJ/PNT — eliminates wrong-color defects.
KO · 설명 IMG 모듈의 시그니처 화면. 원단 롤 장착 전 작업자가 PDA로 롤 QR 스캔. 시스템이 스캔된 색상 코드를 WO 요구 색상과 즉시 대조. 일치 → 진행. 불일치 → 투입 차단, 화면 빨간색, 경고음(BR-IMG-02). 투입 확정 후 IMG-03 실적 확정마다 BOM 단위 기준 원단 길이가 WH 재고에서 자동 차감됨. INJ/PNT 대비 #1 차별화 — 오색 투입 불량 제로화.

02QR Code FormatQR 코드 포맷
Field · 필드Value / Format
PatternFAB-{Color}-{YYMM}-{SEQ:3} 예: FAB-GY-2507-020
FAB고정 prefix (원단 식별자).
Color2자리 컬러 그룹 코드 (GY=Grey, BK=Black, BR=Brown 등). MD-17/MD-10 마스터 참조.
YYMM원단 입고 연월(2+2).
SEQ입고 월 단위 일련번호 (3자리).
EncodingQR Code (ECC L), 최소 25×25 모듈, 컬러 변색 환경 견디는 PET 라벨.
Embedded Color CodeQR 페이로드에 추가 RGB hex (시각 검증용, 옵션): FAB-GY-2507-020;#808080
03Color Verification Logic색상 검증 로직
StepDetail
1. Parse스캔된 QR payload를 정규식으로 파싱: ^FAB-([A-Z]{2})-(\d{4})-(\d{3})$. 형식 불일치 시 즉시 INVALID.
2. LookupWH_FabricStock에서 Roll ID로 잔량·색상 코드 조회. 미등록 롤 → BR-IMG-V01 차단.
3. Cross-checkWH.ColorCode == PP_WO.RequiredColor 확인. 정확 매칭(case-sensitive).
4. ExpiryFabricStock.ExpiryDate ≥ today 확인. 만료 시 라인장 PIN 필요.
5. CapacityRemainingM ≥ WO.RemainingQty × BOM.UnitUseM 검증. 부족 시 노랑 경고(차단 아님).
6. Match Decision1~4 모두 통과 → MATCH. 1~3 중 실패 → MISMATCH (BR-IMG-02 발동).
💡 Color Group Logic · 컬러 그룹 매칭 2자리 컬러 그룹(GY/BK 등)이 1차 매칭 키. 같은 그룹 내 미세 색차(GY-A vs GY-B)는 별도 sub-code로 BR-IMG-V02 적용 — 라인장 PIN 우회 가능.
04Auto Fabric Deduction원단 자동 차감
ItemFormula / Detail
TriggerIMG-03 Production Confirm event.
Source Roll현재 IMG-04에서 검증된 RollID (라인 세션에 바인딩).
Per-cycle DeductionFabricConsumedM = MD_BOM.UnitUseM × GoodQty
ExampleBOM 0.25 m/EA × 4 EA = 1.00 m per cycle.
WH Stock UpdateWH_FabricStock.RemainingM -= FabricConsumedM · 원자적 트랜잭션.
Low Stock AlertRemainingM ≤ 10m → 노랑 알림. ≤ 5m → 빨강 + IMG-04 재투입 권고 (BR-IMG-D01).
Roll ExhaustedRemainingM < expected next cycle → Confirm 차단 (BR-IMG-E03), 새 롤 IMG-04 진행 요구.
Audit Log모든 차감 이벤트 FabricDeductionLog에 기록 (7년 보관, 자재 추적성).
05Workflow작업 흐름
From IMG-02 Dashboard or low-stock alert, tap "Fabric Input" → IMG-04 opens with WO context.
IMG-02 또는 잔량 알림에서 [Fabric Input] 터치 → IMG-04 진입.
PDA shows required color (color swatch + code). Operator scans new roll's QR code.
PDA에 요구 색상 표시(스왓치 + 코드). 작업자가 신규 롤 QR 스캔.
System runs 6-step verification (§03). Match → green ✅ confirm screen.
시스템이 6단계 검증(§03) 수행. 일치 시 녹색 ✅ 확정 화면.
Mismatch → red screen + alarm + conveyor interlock. Operator must re-scan correct roll.
불일치 시 빨간 화면 + 알람 + 컨베이어 인터록. 올바른 롤 재스캔.
On Match Confirm → INSERT IMG_FabricInput · bind to current session. IMG-03 unlocks.
매칭 확정 시 IMG_FabricInput 적재 + 세션 바인딩. IMG-03 잠금 해제.
From now, every IMG-03 Confirm auto-deducts fabric length from this roll's stock.
이후 IMG-03 Confirm 시마다 본 롤 잔량에서 자동 차감.
When remaining low → push notification + IMG-04 re-entry prompt for new roll.
잔량 부족 시 푸시 알림 + IMG-04 재진입 권유.
06Business Rules업무 규칙
RuleDescriptionAction
BR-IMG-01
Roll QR scan is mandatory before production start. Unscanned input blocked.
QR 스캔 없이 생산 불가.
Block
BR-IMG-02
Scanned color ≠ WO color → immediate block + red screen + alarm. Supervisor PIN cannot bypass.
색상 불일치 시 즉시 차단 + 빨간 + 알람. PIN 우회 불가.
Block+Alarm
BR-IMG-03
Auto deduction = BOM unit × produced qty on every IMG-03 Confirm.
IMG-03 확정마다 BOM 기준 자동 차감.
Auto
BR-IMG-V01
Roll ID not in WH_FabricStock → block (rogue / counterfeit roll prevention).
미등록 롤 차단.
Block
BR-IMG-V02
Sub-color mismatch within same group → supervisor PIN required (sample comparison).
동일 그룹 내 미세 색차 → 라인장 PIN 필요.
Audit
BR-IMG-V03
Expired fabric LOT (ExpiryDate < today) → supervisor PIN + reason.
유효기간 만료 LOT → PIN + 사유.
Audit
BR-IMG-V04
All FabricInput attempts logged (success + failure) ≥ 5 years.
모든 시도(성공·실패) 5년 보관.
Retain
BR-IMG-V05
3 consecutive mismatches → escalate to supervisor (possible roll mix-up at WH).
연속 3회 불일치 시 라인장 에스컬레이션 (WH 혼동 의심).
Escalate
