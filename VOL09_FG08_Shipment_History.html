01Overview개요
EN · Description Historical record of every shipment — past SOs, Loadings, DNs, and Returns — searchable by customer, period, item, license plate, driver, seal number, or VIN (if applicable). Read-only view; data cannot be modified. Filters narrow to top results, drill into each shipment opens full audit trail (FG-03 → FG-04 → FG-05 → FG-06 with all e-signatures, photos, GPS, EDI ACKs). Exports to Excel/PDF for tax/customer audits. Role-scoped: Sales sees their customer only; Auditors get time-boxed full access.
KO · 설명 모든 출하 이력 — 과거 SO, Loading, DN, 반품 — 고객·기간·품목·번호판·운전자·봉인번호·VIN(해당 시)로 검색. 읽기 전용, 데이터 변경 불가. 필터로 결과 좁히고, 출하 클릭 시 전체 audit trail 오픈 (FG-03 → FG-04 → FG-05 → FG-06 + 전자서명·사진·GPS·EDI ACK). Excel/PDF 출력 (세무/고객 감사). 권한별 범위: Sales는 자기 고객만, Auditor는 기간 한정 전체 접근.

03Search SLA & Retention검색 SLA + 보관 정책
AspectSpecification
Search SLA95% 쿼리 ≤ 500ms (Elasticsearch + DB hybrid index). Date range search up to 1 yr 즉시, >1 yr 비동기 결과 알림.
Retention - Hot최근 1년 데이터 — 실시간 PostgreSQL + Elasticsearch index (즉시 검색).
Retention - Warm1~3년 데이터 — PostgreSQL (Elasticsearch 인덱스 압축, 검색 가능하나 응답 ≤ 3s).
Retention - Cold3~5년 데이터 — S3 Glacier archived (검색 시 retrieval 12h 후 결과).
Retention - Final5년 이후 — 법적 보관 의무 종료, 단 고객 계약 의무 있으면 연장. 삭제 결정은 Plant Manager 승인 필요.
Export SLA≤ 1만 행 Excel 즉시 (브라우저 download), > 1만 행 백그라운드 + 이메일 알림.
04Workflow작업 흐름
User opens FG-08 → system loads default last-30-day view (role-scoped).
사용자가 FG-08 오픈, 기본 최근 30일 뷰 로드 (권한 범위 적용).
Apply filters (Period/Customer/Item/Truck/etc.) → results refresh; ≤500ms target.
필터(기간/고객/품목/차량 등) 적용, 결과 갱신 (≤500ms 목표).
Click row → drill-in opens full audit trail (FG-03 → FG-04 → FG-05 → FG-06 events).
행 클릭 시 드릴 — 전체 audit trail (FG-03 → FG-04 → FG-05 → FG-06) 오픈.
Each event has source-link to original screen with original data (read-only).
각 이벤트는 원본 화면 링크 (읽기 전용 원본 데이터).
Export Excel/PDF for audit; all export actions logged with userID, query, row count.
감사용 Excel/PDF 출력, 모든 export는 사용자·쿼리·행 수 로그.
Cold storage (3-5y) search: triggers retrieval job (12h SLA); user notified by email.
Cold storage(3-5년) 검색은 retrieval 작업 발사 (12h SLA), 사용자 이메일 알림.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-38
FG-08 is Read-only; data cannot be edited or deleted. Corrections require new shipment records.
FG-08 읽기 전용, 수정/삭제 불가. 정정 시 신규 출하 기록.
Lock
BR-FG-39
All search/view/export actions logged for audit accountability (userID, query, timestamp, row count).
모든 검색/조회/export 행위 사용자·쿼리·시각·행수 로그.
Log
BR-FG-40
Sales role limited to their assigned customer; cross-customer data hidden at SQL.
Sales 권한은 담당 고객만, 타 고객 SQL 차단.
Scope
BR-FG-41
External Auditor gets time-boxed access (visit period); auto-revoked after.
외부 감사관은 방문 기간만 접근, 종료 후 자동 회수.
Time-Box
BR-FG-42
Data deletion after 5y requires Plant Manager approval + Sales/Legal confirmation; default is retain.
5년 이후 삭제는 공장장 승인 + Sales/법무 확인, 기본은 보관.
Approve
