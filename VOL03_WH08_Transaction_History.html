01
Overview
화면 개요
Complete audit trail of every inventory movement: IN (WH-02 inbound), OUT (WH-07 release), ADJ+/ADJ− (WH-05 adjustment), RELOC (location move). Each row captures: timestamp, operator, transaction type, item, location, qty before/after, reason code, reference document (PO / WO / Adjustment ID), and approver (for adjustments). Searchable by date range, item, operator, transaction type, and reference ID. Excel export for audit and regulatory purposes. Records are immutable and retained for 10 years per quality system requirements.
모든 재고 이동에 대한 완전한 감사 기록: 입고(IN, WH-02), 출고(OUT, WH-07), 조정(ADJ+/ADJ−, WH-05), 위치 이동(RELOC). 각 행에는 일시·작업자·거래 유형·품목·위치·전후 수량·사유 코드·참조 문서(PO/WO/Adjustment ID)·승인자(조정의 경우)가 기록됩니다. 날짜 범위·품목·작업자·거래 유형·참조 ID로 검색 가능하며, 감사·규제 목적의 Excel 출력을 지원합니다. 기록은 수정 불가하며 품질 시스템 요건에 따라 10년 보관됩니다.

04
Event & Action Flow
이벤트 · 액션 흐름
1
Systemtrigger · transaction commit
Whenever WH-02/05/07 commits an inventory change, a corresponding row is written to wh_transaction via DB trigger. Atomic with source operation.
WH-02·05·07이 재고 변경 커밋 시, DB 트리거로 wh_transaction에 행이 자동 기록됨. 원자적 커밋.
EFFECTINSERT INTO wh_transaction (...) VALUES (...) · WebSocket push.
2
Userpage.load · WH-08 진입
Auditor or warehouse manager opens screen. Default filter: today, all types. Loads first page of 50 rows sorted by timestamp DESC.
감사자 또는 관리자가 진입. 기본 필터(오늘·전체 유형)로 첫 50행 로딩(시각 내림차순).
APIGET /api/wh/transactions?date=today&page=1&size=50.
3
Systempush · new transaction
When a new transaction commits, WebSocket pushes the row to all subscribed clients. New row appears at top with brief flash animation.
신규 거래 커밋 시 WebSocket으로 푸시. 표 상단에 새 행이 짧은 플래시 애니메이션과 함께 추가.
UI Row prepended · auto-pagination preserved · KPI counters increment.
4
Userclick · KPI card
User clicks a KPI to drill into that type. Filter applied, table redraws.
KPI 클릭 시 해당 유형 필터 자동 적용 후 표 갱신.
URL?type=ADJ&date=... · deep-linkable.
5
Userclick · ref_id link
User clicks a reference (PO / WO / ADJ ID) to navigate to source document.
참조 ID(PO/WO/ADJ) 클릭 시 원본 문서 화면으로 이동.
NAV PO → WH-01 detail · WO → PP-04 · ADJ → WH-05.
6
Userclick · [📥 Excel Export]
User requests export. System checks WH.Export role and row count. If < 50,000, generates .xlsx async via background job; download link emailed when ready.
Excel 출력 요청. 권한·행 수 검증 후 비동기 백그라운드 작업으로 생성. 완료 시 다운로드 링크 이메일 전송.
APIPOST /api/wh/export · job_id returned · email when ready.
✓
Userclick · [🖨 Audit PDF]
Generates a digitally signed PDF report with cover page, audit footer, and tamper-evident hash. Used for ISO/IATF audit submission.
디지털 서명된 PDF 감사 보고서 생성(표지·푸터·해시값 포함). ISO/IATF 감사 제출용.
OUTPUT PDF with SHA-256 hash footer · timestamped digital signature.
05
Exception & Error Handling
예외 · 에러 처리
E-WH08-001
Range Exceeds 90 Days기간 90일 초과
Cause
Date range > 90 days. Performance and storage concern. 조회 기간 90일 초과.
Detect
Client + server validation.
Action
Block. Unless user has WH.Audit.Extended role. Modal: "Range > 90 days requires extended audit role."
Recovery
Reduce range or request extended role from SYS admin.
⚠ RANGE TOO LARGE — REDUCE OR ESCALATE
E-WH08-002
Edit Attempt수정 시도
Cause
User attempts to edit/delete a transaction row via API or DevTools. 감사 행 수정·삭제 시도.
Detect
UPDATE/DELETE on wh_transaction blocked at DB level (BR-WH-17).
Action
Return 403. Log attempt with user_id, IP, timestamp in sys_security_audit. Alert SYS admin.
Recovery
If correction needed, create compensating transaction via WH-05 with reason SYSTEM_ANOMALY.
⛔ IMMUTABLE — CREATE COMPENSATION
E-WH08-003
Export Row Limit출력 행 제한
Cause
Filtered result > 50,000 rows. 결과 5만 행 초과.
Detect
Server-side count check.
Action
Reject. Suggest narrower filter or use RPT-04 audit reports for large datasets.
Recovery
Narrow filter, or request RPT-04 audit batch report.
⚠ TOO MANY ROWS — USE RPT-04
E-WH08-004
Hash Mismatch해시 불일치
Cause
Periodic integrity check detects record hash mismatch (data tampered or corruption). 레코드 해시 불일치(변조·손상).
Detect
Daily integrity cron at 02:00 KST.
Action
CRITICAL alert to SYS admin, plant director, QA lead. Affected rows flagged. Forensic backup loaded.
Recovery
SYS team investigates; restore from immutable archive; report to compliance.
⛔ INTEGRITY VIOLATION — INVESTIGATE
E-WH08-005
Old Data Archived오래된 데이터 아카이브
Cause
Searching for data older than 2 years; moved to cold storage. 2년 초과 데이터는 콜드 스토리지.
Detect
Query response time slow (> 5s).
Action
Show banner: "Querying archived data. May take longer." Background async fetch + email when ready.
Recovery
Wait for fetch or use specific filters to narrow.
ℹ ARCHIVED — FETCHING ASYNC
E-WH08-006
No Transactions Found거래 없음
Cause
Filter combination yields zero results. 필터 결과 0건.
Detect
API returns empty array.
Action
Empty state illustration: "No transactions match your filters." Reset button.
Recovery
Reset or adjust filters.
ℹ NO RESULTS — ADJUST FILTER

07
Applicable Business Rules
적용 업무 규칙
Rule IDDescriptionAction
BR-WH-17
All transactions in wh_transaction are immutable (INSERT-only). UPDATE and DELETE are forbidden at DB level. Corrections must use compensating transactions.
모든 거래는 수정 불가(INSERT 전용). UPDATE·DELETE는 DB 차원에서 금지. 보정은 별도 보정 거래로 처리.
Block
BR-WH-18
Transaction records are retained for 10 years to comply with IATF 16949 traceability requirements. Records older than 2 years move to cold storage.
IATF 16949 추적 요건에 따라 거래 기록 10년 보관. 2년 초과 데이터는 콜드 스토리지 이관.
Auto
BR-WH-19
Each transaction row includes a SHA-256 hash of its content. Daily integrity check verifies no tampering. Hash mismatch = CRITICAL incident.
각 거래 행에 SHA-256 해시 포함. 일일 무결성 검증으로 변조 감지. 불일치는 치명적 인시던트.
Integrity
