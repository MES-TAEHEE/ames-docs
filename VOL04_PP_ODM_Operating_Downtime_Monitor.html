01
Overview
화면 개요 · 목적 · 컨텍스트
A real-time wallboard that compares each line's planned time pattern (MD-29 baseline) against its actual operating state streamed from equipment signals. Every minute of the day resolves to one of three states. When the plan says OPERATING but the line signal says stopped, the gap is auto-classified UNPLANNED_DOWN and the operator is prompted to log a reason via PP-DTL. The classified state log feeds PP-OEE availability and RPT-06 OEE.
각 라인의 계획 시간패턴(MD-29 기준선)과 설비 신호로 수신되는 실제 가동 상태를 실시간 대조하는 현황판. 하루의 매 분(分)이 세 상태 중 하나로 판정된다. 계획상 OPERATING인데 라인 신호가 정지면 그 구간은 UNPLANNED_DOWN으로 자동 분류되고 작업자에게 PP-DTL 사유 입력을 요청한다. 분류된 상태 로그는 PP-OEE 가용성·RPT-06 OEE로 전달된다.

04
Event & Action Flow
이벤트 흐름 · 상태 분류 → 사유 입력
1
TICKminute.tick · equipment signal
Every minute, equipment RUN/STOP signal (via SYS-04 interface) is paired with the line's MD-29 planned state for that minute.
매 분 설비 RUN/STOP 신호(SYS-04 인터페이스)를 해당 분의 MD-29 계획 상태와 짝지음.
INsignal{lineId, minute, runFlag} + MD_LineTimeSegment.state
2
SYSTEMclassify · 3-state
Resolves the minute to OPERATING / PLANNED_DOWN / UNPLANNED_DOWN and appends a PP_LineStateLog row.
해당 분을 OPERATING / PLANNED_DOWN / UNPLANNED_DOWN으로 판정 → PP_LineStateLog 행 추가.
RULEBR-PP-035 · STOP within OPERATING window → UNPLANNED_DOWN
3
SYSTEMdetect · downtime event
Consecutive UNPLANNED_DOWN minutes are merged into one open downtime event; the board badges it as reason-unlogged.
연속 UNPLANNED_DOWN 분(分)을 하나의 진행중 비가동 이벤트로 병합 — 사유 미입력 배지 표시.
APIPOST /api/pp/downtime-event · status=OPEN
4
SYSTEMalert · downtime threshold
An unplanned downtime exceeding 10 minutes pushes an alert to PLAN / MNT supervisors.
실제 비가동 10분 초과 시 PLAN/MNT 감독자에게 경보 푸시.
ALERTBR-PP-037 · in-app + SYS-06 notification
5
USERclick · [사유 입력] → PP-DTL
Operator logs a downtime reason; the event closes and the classified state log flows to PP-OEE availability.
작업자가 비가동 사유 입력 → 이벤트 종료 → 분류 상태 로그가 PP-OEE 가용성으로 전달.
DOWNSTREAMPP-DTL close · PP-OEE recalc · RPT-06 OEE

ActionEscalation alert to supervisor; event stays OPEN.
⚠ REASON OVERDUE

Run During Planned Down계획비가동 중 가동
CauseEquipment RUN while plan says PLANNED_DOWN.
DetectClassifier mismatch.
ActionCounted as OPERATING (extra run); flagged for pattern review.
⚠ UNPLANNED RUN

No Pattern Baseline기준 패턴 없음
CauseLine has no ACTIVE MD-29 pattern for today.
DetectOn monitor load.
ActionCannot classify; lane shows 'define pattern' prompt.
