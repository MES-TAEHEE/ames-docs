01
Overview
화면 개요 · 목적 · 컨텍스트
Manages customer forecast intake (SAV/GEO). Excel files downloaded from customer SCM portals are uploaded, validated, and mapped against the PP_CustomerItemMap table to convert customer item numbers to A-MES item codes. Provides M+1 to M+6 monthly demand visibility with M+1~M+3 binding for production planning and M+4~M+6 as reference for capacity planning. Detects gaps between forecast and confirmed orders, alerting PLAN/MGR when deviation exceeds 20%.
고객사(SAV/GEO) Forecast 수신·관리 화면. 고객 SCM 포탈에서 다운로드한 Excel을 업로드하면 PP_CustomerItemMap으로 고객사 품번 → A-MES 품목코드 매핑·검증. M+1~M+3은 생산계획 구속력, M+4~M+6은 능력계획 참고. Forecast 대비 확정 수주 편차 20% 초과 시 PLAN/MGR 알림.

04
Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
USER click · [📤 Upload Excel]
User selects Excel file from customer SCM portal.
고객 SCM 포탈에서 다운로드한 Excel 선택.
VALIDATESchema check: required columns present, no empty Item No.
2
SYSTEM parse · file → rows
System parses rows and matches against PP_CustomerItemMap for item code mapping.
행을 파싱하여 PP_CustomerItemMap으로 품번 매핑.
EFFECTMapped: insert/update PP_Forecast. Unmapped: flag in warning list.
3
SYSTEM calc · gap analysis
For each mapped item, fetch confirmed orders from PP_SupplyPlan and compute gap%.
매핑된 품목별로 PP_SupplyPlan의 확정 수주와 비교하여 편차% 계산.
ALERTGap > 20% → BR-PP-F002 alert sent to PLAN/MGR via email.
4
SYSTEM history · re-upload
If duplicate month+item rows, move existing to PP_ForecastHistory before update.
동일 월·품목 중복 시 기존 행을 PP_ForecastHistory로 이관 후 갱신.
AUDITHistory retained 3 years per audit policy.
5
USER click · row [Map] (for unmapped)
User clicks Map button on unmapped row, opens dialog to manually link customer item no to A-MES item code.
미매핑 행의 [Map] 클릭 시 수동 매핑 다이얼로그 표시.
APIPOST /api/pp/customer-item-map · adds mapping for future uploads.
6
SYSTEM complete · forecast saved
Forecast tab refreshed. Downstream PP-03 plan creation can now reference these qty.
Forecast 탭 갱신. PP-03 생산계획 수립에서 이 수량 참조 가능.
DOWNSTREAMPP-03 (planning), PP-05 (MRP capacity check)
05
Exception Handling
예외 처리 · 오류 코드 · 복구 절차
E-PP01-001
Unmapped Item미매핑 품번
CauseCustomer item no not found in PP_CustomerItemMap.
DetectOn upload parse.
ActionRow flagged red, excluded from save. Warning list shown.
RecoverUser maps via [Map] button or MD team updates mapping master.
⚠ UNMAPPED · MAP REQUIRED
E-PP01-002
Invalid File Format파일 형식 오류
CauseExcel missing required columns or wrong type.
DetectSchema validation on upload.
ActionReject entire upload. Show specific missing columns.
RecoverUse [Sample Template] download.
⛔ INVALID FORMAT
E-PP01-003
Forecast Gap Alert편차 알림
CauseConfirmed orders vs forecast deviation > 20%.
DetectAuto on save.
ActionEmail alert to PLAN+MGR. Item flagged in table.
RecoverReview with sales/customer.
ℹ GAP ALERT — REVIEW
E-PP01-004
Concurrent Upload동시 업로드 충돌
CauseTwo users upload same month forecast simultaneously.
DetectRow lock on PP_Forecast.
ActionSecond user blocked with current state shown.
RecoverReload and re-apply changes if still needed.
⚠ CONCURRENT — RELOAD
E-PP01-005
History Rollback Required이력 복구 필요
CauseUser requests rollback to previous forecast version.
DetectManual via [History] tab.
ActionRequires manager approval. Restored from PP_ForecastHistory.
RecoverManager approves; new entry inserted (not in-place update).
⚠ ROLLBACK · APPROVAL
E-PP01-006
Empty Upload빈 파일
CauseExcel has zero data rows.
DetectPre-parse check.
ActionBlock save. Suggest using correct template.
RecoverRe-download from customer portal.
⚠ NO DATA
