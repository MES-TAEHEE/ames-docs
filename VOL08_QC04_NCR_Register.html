01Overview개요
EN · Description Central Non-Conformance Report (NCR) registry that consolidates all defects detected across A-MES — auto-issued from QC-01/02/03 FAILs and from line-side INJ-05/IMG-05/PNT-08, plus manually-entered customer claims. Each NCR carries a 3-tier severity (Critical / Major / Minor), root-cause classification (4M: Man/Machine/Material/Method), evidence (photos, measurements), and a Disposition decision (Rework / Scrap / UAI Use-As-Is / Return-to-Supplier). NCRs ≥ Major auto-link to Hold (QC-05) and feed CAPA evaluation (QC-06).
KO · 설명 A-MES 전체에서 발견된 부적합품(NCR)을 통합 관리하는 중앙 등록부. QC-01/02/03 FAIL 및 라인사이드 INJ-05/IMG-05/PNT-08 자동 발행, 고객 클레임 수동 등록. 각 NCR은 3등급 심각도(Critical/Major/Minor), 4M 근본원인(Man/Machine/Material/Method) 분류, 증빙(사진·측정값), Disposition 결정(Rework/Scrap/UAI/Return)을 포함. Major 이상은 자동으로 Hold(QC-05) 연계 + CAPA(QC-06) 평가 자동.

03NCR Status FlowNCR 상태 흐름
StatusDescription & Transition Rules
DRAFT검사원/오퍼레이터가 초안 작성 중. Severity·증빙·4M 입력 필수. 자동 발행 NCR은 DRAFT 단계 생략 가능.
OPEN정식 등록. Major+ 시 자동으로 QC-05 Hold 생성 + QC Lead 알림. Disposition 미정 상태.
DISPOSEDLead 이상이 처분 결정 (REWORK/SCRAP/UAI/RETURN). 후속 작업 자동 발사 (Rework → 신규 WO, Scrap → 재고 차감, UAI → 고객 승인 첨부 필수, Return → 공급사 반품 WO).
CLOSED모든 후속 작업 완료, CAPA 평가 완료. ≥ 5년 보관. 재오픈 불가 (수정 시 신규 NCR 생성).
SeverityAuto Cascade
🔴 CRITICAL(1) QC-05 Hold 자동, (2) QC Lead + Plant Manager 즉시 알림, (3) Disposition은 Plant Manager 승인 필수, (4) CAPA 자동 권고, (5) Customer notify 자동 켜짐.
🟡 MAJOR(1) QC-05 Hold 자동, (2) QC Lead 알림, (3) CAPA 평가 자동 (30일 3회 조건).
🟢 MINOR(1) Hold 선택 (수량 적은 경우 생략 가능), (2) 추세 모니터링만 (QC-07 Dashboard 반영).
04Workflow작업 흐름
Trigger: QC-01/02/03 FAIL, line defect ≥ Major, or customer claim. NCR auto-issued with pre-filled header.
트리거: QC 검사 FAIL, 라인 defect Major 이상, 또는 고객 클레임. 헤더 자동 채워진 NCR 발행.
Inspector reviews and adds: defect codes, 4M cause, photos, measurements, linked events.
검사원이 검토 후 추가 입력: 불량 코드·4M 원인·사진·측정값·연관 이벤트 링크.
Severity auto-evaluated (Customer impact × Frequency); inspector confirms or escalates.
시스템이 Severity 자동 평가(고객영향×발생빈도), 검사원이 확인 또는 상향 조정.
Submit → Status: DRAFT → OPEN. Major+ triggers Hold (QC-05) + Lead alert in < 5s.
제출 → DRAFT → OPEN 전환. Major+ 시 5초 내 Hold(QC-05) + QC Lead 알림.
QC Lead reviews evidence, decides Disposition (REWORK/SCRAP/UAI/RETURN); UAI requires customer approval PDF.
QC Lead가 증빙 검토 후 Disposition 결정(REWORK/SCRAP/UAI/RETURN). UAI 선택 시 고객 승인 PDF 첨부 필수.
Downstream actions auto-fire: REWORK → new WO, SCRAP → inventory write-off, UAI → release Hold, RETURN → supplier debit memo.
후속 작업 자동: REWORK→신규 WO, SCRAP→재고 폐기, UAI→Hold 해제, RETURN→공급사 차변 발의.
CAPA evaluation: if 30-day rolling count of same defect ≥ 3 → auto-recommend CAPA (QC-06).
CAPA 평가: 30일 이동 동일 불량 카운트 3회 이상 → CAPA 자동 권고(QC-06).
All downstream actions completed and CAPA closed (if any) → NCR Status → CLOSED, sealed for ≥ 5 years.
모든 후속 + CAPA 완료 → CLOSED 전환, 5년 이상 봉인 보관.
05Business Rules업무 규칙
RuleDescriptionAction
BR-QC-11
UAI (Use-As-Is) disposition requires customer approval document attached. Without it → blocked.
UAI 처분 시 고객 승인 문서 첨부 필수, 없으면 차단.

BR-QC-12
CRITICAL severity Disposition requires Plant Manager approval (Lead alone insufficient).
CRITICAL 처분은 공장장 승인 필수, Lead만으로 불가.
Approve
BR-QC-13
≥ Major NCR auto-creates QC-05 Hold for affected qty within 5 seconds.
Major 이상 NCR은 5초 내 QC-05 Hold 자동 생성.
Auto
BR-QC-03
Same defect code ≥ 3 within 30 days → auto-recommend CAPA (QC-06).
30일 내 동일 불량 3회 이상 → CAPA 자동 권고.
Alert
BR-QC-14
NCR CLOSED records are immutable; corrections require new NCR with reference.
CLOSED NCR 수정 불가, 정정 시 참조 포함 신규 NCR 생성.
Lock
BR-QC-08
All NCR records retained ≥ 5 years (IATF 16949 / customer audit).
NCR 5년 이상 보관 (IATF 16949 / 고객 감사).
Retain
