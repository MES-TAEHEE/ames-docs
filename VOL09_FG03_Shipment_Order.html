01Overview개요
EN · Description Customer shipment order intake and stock reservation. Receives orders from SAV/GEO via EDI auto-feed (hourly poll), customer portal push, or manual entry by Sales. System auto-validates customer/item/qty against MD-08 PackSpec, allocates Available stock by FIFO (oldest LOT first), creates atomic Reservation, generates Pickslip for FG-04. Late orders (D+1 ship-date) escalate to Logistics Lead. Order cancellation auto-releases reservation back to Available.
KO · 설명 고객 출하 주문 수신 + 재고 예약. SAV/GEO EDI 자동 수신(시간당 polling), 고객 포털 push, Sales 수동 입력. 시스템이 고객/품목/수량을 MD-08 PackSpec 기준 자동 검증, FIFO로 Available 재고 할당(오래된 LOT 우선), 원자 예약 생성, FG-04용 Pickslip 발행. 출하일 D+1 임박 주문은 Logistics Lead 에스컬레이션. 주문 취소 시 예약 자동 해제(Available 복귀).

03Order Lifecycle주문 라이프사이클
StatusDescription & Transition
RECEIVEDEDI/Portal/Manual 수신 직후. 자동 검증 진행 중. 90% 자동 통과 시 즉시 CONFIRMED 전환, 검증 실패 시 Sales 알림 (수동 보정 필요).
CONFIRMED재고 FIFO 할당 + Reservation 락 완료. 고객 ACK 발송 가능. Pickslip 생성 가능.
PICKINGFG-04에서 Pickslip 수령 + 작업 시작. 부분 피킹 가능.
PICKED전수 피킹 완료. 상차 대기.
SHIPPEDFG-05 Loading Confirm 완료. OTD 평가 (ShipDate 대비). FG-06 Delivery Note 발행 가능.
CANCELED고객 취소 또는 내부 중단. CONFIRMED~PICKING 단계만 취소 가능, PICKED 이후는 Return으로 처리. Reservation 자동 해제.
04Workflow작업 흐름
EDI / Portal / Manual entry → SO created in RECEIVED state with auto-validation queued.
EDI/포털/수동 입력 → RECEIVED 상태 SO 생성, 자동 검증 큐 적재.
System validates: customer hold/credit, item active status, PackSpec compliance, stock availability.
시스템이 고객 Hold/신용·품목 상태·PackSpec·재고 자동 검증.
FIFO Allocator picks oldest LOT first, splits across multiple LOTs if needed; atomic Reserve transaction.
FIFO 할당기가 오래된 LOT 우선, 필요시 다중 LOT 분할, 원자 예약 트랜잭션.
CONFIRMED → ACK auto-sent to customer (EDI/Email); Pickslip generated for FG-04.
CONFIRMED → 고객 ACK 자동 발송, FG-04 Pickslip 생성.
FG-04 Picker fetches Pickslip → PICKING; partial pick OK; on full pick → PICKED.
FG-04 작업자가 Pickslip 인수 → PICKING, 부분 가능, 전수 완료 시 PICKED.
FG-05 Loading Confirm → SHIPPED; OTD evaluated; FG-06 Delivery Note printable.
FG-05 상차 확정 → SHIPPED, OTD 평가, FG-06 거래명세서 출력 가능.
Late orders (no ship by D+0 24:00) auto-flag OTD=LATE; escalate to Logistics Lead.
D+0 24:00 미출하 시 OTD=LATE 자동, Logistics Lead 에스컬레이션.
05Business Rules업무 규칙
RuleDescriptionAction
BR-FG-11
EDI auto-orders bypass manual entry but still pass validation. Validation FAIL → Sales alert.
EDI 자동 주문도 검증은 동일 적용, FAIL 시 Sales 알림.
Validate
BR-FG-12
FIFO allocation mandatory: oldest LOT first. Manual override requires PIN + Reason.
FIFO 할당 필수, 오래된 LOT 우선. 수동 변경 PIN+사유.
Force
BR-FG-13
Customer Hold (credit/quality) blocks SO creation; only Manager+ can override.
고객 Hold(신용/품질) 시 SO 생성 차단, Manager 이상만 우회.
Block
BR-FG-14
Ship-date D+0 24:00 미출하 → OTD=LATE 자동 + Logistics Lead 에스컬레이션.
D+0 24:00 미출하 시 LATE 자동.
Alert
BR-FG-15
SO 수정은 RECEIVED/CONFIRMED 단계만 가능. PICKING 이후는 Cancel + 신규 SO.
PICKING 이후 수정 불가, 취소 + 신규 SO.
Lock
BR-FG-16
Cancellation auto-releases all reservations; refund/credit handled in ERP (out of scope).
취소 시 예약 자동 해제, 환불은 ERP 처리.
Auto
