A-MES 데이터베이스는 10개 모듈 · 149개 테이블로 구성됩니다. MD 기준정보가 모든 모듈에서 참조되는 중심축이고, ID(ASP.NET Identity)와 SYS가 인증·감사 영역을 담당합니다. 실 생산 데이터 흐름은 WH → PP → PR/PNT → QC → FG 방향이며, MNT는 설비 고장·PM으로 PR과 양방향 연동됩니다.

MD · MASTER DATA
기준정보 (33 tables)
자연키 PK · 전 모듈에서 참조
MD 모듈은 모든 거래 테이블의 참조 대상입니다. MD_Item·MD_Equipment·MD_Mold·MD_Line·MD_Customer가 가장 빈번하게 참조됩니다.
WH · WAREHOUSE
창고 (8 tables)
발주 수신 → 입고 → 재고 → 출고 피킹 사이클. WH_TransactionHistory는 append-only 감사 로그.
PP · PRODUCTION PLANNING
생산계획 (16 tables)
SAP B1 수주/공급계획 → PP_WorkOrder → MRP/PR/라인 스케줄/OEE. ODM이 분 단위 라인 상태를 분류하고 DTL이 비가동 사유 입력.
PR · PRODUCTION SHARED
생산공통 INJ/IMG (23 tables)
tbl_Lot은 전 모듈 추적 앵커. PR_ProductionResult가 사이클별 실적, INJ는 금형/MoldChange, IMG는 Fabric*/Bond* 전용 테이블 사용.
PNT · PAINTING
도장 (22 tables)
RFID 기반 라인 추적. PNT_VirtualLot(사전 발행) → JigBinding → R1/R2/R3 LineEvent → OvenLog → JigUnload → LotLabel → QcQueue.
QC · QUALITY CONTROL
품질 (10 tables)
IQC/IPQC/FQC 통합 헤더 → FAIL 시 NCR/Hold 자동 생성. CAPA는 PDCA + D+30/60/90 효과성 체크. 5년+ 보관.
FG · FINISHED GOODS
완제품 (10 tables)
QC PASS → FG_Stock 적치 → SO 수신 → FIFO 피킹 → 상차 → BOL 발행 → 일마감. 반품은 RMA → NCR 자동 연계.
MNT · MAINTENANCE
설비보전 (10 tables)
설비 상태(PLC 동기화) · 고장 등록·조치 · OEE 측정 · PM 일정·실행 · 정비 WO/Task · 자재 입출고 · 금형 쇼트수 운영 카운터.
SYS · SYSTEM ADMIN
시스템 (9 tables)
UserProfile (Identity 확장) + RBAC 매트릭스 + 감사로그 + 알림 (Rule/Channel/History) + 환경설정 + 인터페이스 상태 + 공장 캘린더.
ID · ASP.NET IDENTITY
인증 (7 tables)
.NET 표준 Identity 스키마. AspNetUsers.Id NVARCHAR(450)가 전 모듈 사용자 FK의 참조 대상.
