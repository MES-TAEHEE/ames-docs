01
Overview
화면 개요 · 목적 · 컨텍스트
Manages Work Orders through their full lifecycle: Draft → Released → In Progress → Closed. Each WO has a routing badge (A: INJ→IMG→QC→FG · B: INJ→PNT→QC→FG), production progress bar, D-Day indicator, and SAP B1 supply plan reference. Block reasons (Phase-0 incomplete, MRP shortage, equipment unavailable) are surfaced before release.
작업지시(WO) 전체 생명주기 관리: Draft → Released → In Progress → Closed. 각 WO는 라우팅 배지(A: INJ→IMG→QC→FG · B: INJ→PNT→QC→FG), 진척률 바, D-Day, SAP B1 공급계획 참조 표시. 발행 전 차단 사유(Phase-0 미완료·MRP 부족·설비 미가용) 표시.

04
Event & Action Flow
이벤트 흐름 · 사용자 액션 → 시스템 반응
1
USER page.load · WO list
Loads all WOs with current status, progress, D-Day. Default sort by D-Day ascending.
전체 WO 목록 로드 (상태·진척률·D-Day). 기본 D-Day 오름차순.
APIGET /api/pp/wo?date=today&sort=dday
2
USER click · row
Opens detail side panel with WO header, BOM allocation, routing steps, and history log.
행 클릭 시 상세 패널 표시 (헤더·BOM 할당·라우팅·이력).
APIGET /api/pp/wo/:id/detail
3
SYSTEM push · WO status change
When downstream POP (INJ/IMG/PNT) updates routing step, WO progress and status auto-refresh via WebSocket.
POP 모듈(INJ/IMG/PNT) 공정 갱신 시 WebSocket으로 진척률·상태 자동 갱신.
PUSHWS event: wo.progress {wo_id, routing_step, qty}
4
SYSTEM alert · D-3 approaching
When WO due date is within 3 days, sends alert + highlights row red.
WO 납기 D-3 도달 시 알림 발송 + 행 빨간색 강조.
ALERTBR-PP-008 · email + in-app + row.class='rd'
5
USER click · [+ Manual WO]
Opens manual WO creation form. Requires manager role and provides bypass for non-SAP planning.
수동 WO 생성 폼. 매니저 권한 + SAP 미경유 케이스 처리.
APIPOST /api/pp/wo · source='manual', requires audit reason
6
USER click · row [Cancel]
User cancels Draft/Released WO. System auto-releases material reservations and reverses status.
Draft/Released WO 취소 시 자재 예약 자동 해제 + 상태 역전.
EFFECTBR-PP-004 · WH_Inventory.Reserved 감소
7
SYSTEM complete · status sync
WO status changes propagate to PP-CAL calendar and PP-OTD KPI calculations.
WO 상태 변경이 PP-CAL 캘린더 및 PP-OTD KPI 집계에 전파.
DOWNSTREAMPP-CAL refresh · PP-OTD recalc · RPT-05 dashboard
05
Exception Handling
예외 처리 · 오류 코드 · 복구 절차
E-PP04-001
MRP BlockMRP 차단
CauseWO blocked by PP-05 MRP due to material shortage.
DetectAuto on MRP run.
ActionRow red. Status = Blocked. [Release] disabled. Show shortage detail.
RecoverRun PP-06 PR or wait for material.
⛔ BLOCKED · MRP
E-PP04-002
Cancel In Progress진행중 취소
CauseUser attempts to cancel In Progress WO.
DetectStatus check on cancel.
ActionModal: 'Already in production. Cancel anyway?' Manager approval required.
RecoverManager confirms; partial production logged.
⚠ IN PROGRESS · APPROVE
E-PP04-003
Routing Mismatch라우팅 불일치
CausePOP screen reports routing step inconsistent with BOP.
DetectPush event validation.
ActionYellow warning. Allow but log as anomaly. QC may need review.
RecoverAudit log entry + QC review.
⚠ ROUTING DRIFT
E-PP04-004
Closed WO Edit Attempt마감 WO 수정 시도
CauseUser tries to modify a Closed WO.
DetectStatus check on save.
ActionBlock. WO is immutable after Close.
RecoverCreate compensating WO if correction needed.
⛔ CLOSED · IMMUTABLE
E-PP04-005
Manual WO Permission수동 WO 권한
CauseNon-manager attempts Manual WO.
DetectRBAC check.
ActionBlock with 'PP.WO.Manual role required'.
RecoverRequest role from sys admin.
⛔ PERMISSION DENIED
E-PP04-006
WO Qty MismatchWO 수량 불일치
CauseCompleted qty > planned qty.
DetectPost-routing validation.
ActionYellow flag. Investigate overproduction reason.
RecoverQC review + reconcile with SO.
⚠ OVER-PRODUCED
