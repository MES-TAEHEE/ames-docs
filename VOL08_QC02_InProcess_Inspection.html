01Overview개요
EN · Description Mid-process sampling inspection of production WOs while still running on the line. Auto-triggered when cumulative good qty in INJ/IMG/PNT reaches MD-08 SampleInterval. QC Inspector picks the queued job on PDA (line-side, fast scan) or Web (deeper measurement). PASS → WO continues. FAIL → line auto-hold + NCR (QC-04) + supervisor PIN required to resume. Critical-defect FAIL also triggers QC-05 Hold for current shift output.
KO · 설명 생산 중인 WO에 대한 공정 도중 샘플 검사. INJ/IMG/PNT 누적 양품이 MD-08 SampleInterval 도달 시 자동 발행. QC 검사원이 PDA(현장 빠른 스캔) 또는 Web(상세 측정)으로 검사. 합격 → WO 계속 진행. 불합격 → 라인 자동 정지 + NCR(QC-04) + 라인장 PIN으로 재가동. Critical 등급 불량 시 해당 교대 누적 생산분 QC-05 격리 자동.

03Auto Trigger Flow자동 트리거 흐름
SourceTrigger Condition
INJ-04BR-INJ-06: WO 누적 양품이 MD_InspStd.SampleInterval 도달 시 QC-02 자동 발행.
IMG-03BR-IMG-06: 동일 규칙 (IMG 라인용).
PNT-06BR-PNT-06: R3 통과(CONFIRMED) 시 LOT 단위로 QC-02 발행.
PNT-05 Oven Deviation오븐 ±5°C/2분 이탈 LOT은 강화검사 플래그로 QC-02 자동 발행 (BR-PNT-05).
Manual TriggerQC Lead가 임의 시점에 수동 발행 가능 (이력 자동 기록).
OutcomeAuto Cascade
PASS(1) WO Status 그대로 유지, (2) MD_InspStd 다음 검사 주기까지 자동 카운터 리셋, (3) RPT 모듈에 PASS 데이터 적재.
FAIL(1) 라인 자동 정지(해당 WO만), (2) NCR(QC-04) 자동 발행, (3) INJ-05/IMG-05/PNT-08 Defect 자동 연계, (4) 라인장 PDA 푸시, (5) Resume 시 PIN + CorrectiveAction 필수.
FAIL + Critical위 + (6) 해당 교대 누적 양품 전체 QC-05 격리, (7) QC Lead 자동 알림, (8) CAPA(QC-06) 후보 자동 등록.
04Workflow작업 흐름
INJ/IMG/PNT cumulative good count hits MD-08 SampleInterval → QC-02 auto-issued + PDA push.
생산 라인 누적 양품이 검사 주기 도달 → QC-02 자동 발행 + PDA 푸시.
QC Inspector arrives line-side, scans WO barcode/PDA, retrieves required sample (8 EA in example).
QC 검사원이 라인 도착, WO 바코드 PDA 스캔, 필요 샘플(예: 8개) 수령.
Inspector enters per-sample results via PDA (Visual + Dimension + Weight). 1 FAIL → batch FAIL.
검사원이 PDA로 샘플별 판정 입력(외관·치수·중량). 1개라도 FAIL이면 LOT 전체 FAIL.
On PASS: tap Confirm → WO continues; counter resets to next interval automatically.
PASS 시 Confirm 터치 → WO 계속, 다음 주기까지 카운터 자동 리셋.
On FAIL: line auto-halts (this WO only), supervisor receives push; NCR pre-filled with sample data.
FAIL 시 라인 자동 정지(해당 WO), 라인장 푸시 수신, NCR이 샘플 데이터로 자동 채워짐.
Supervisor reviews cause, enters CorrectiveAction + PIN → line resumes; downstream parts flagged for enhanced check at QC-03.
라인장이 원인 검토, 시정조치 + PIN 입력 → 라인 재가동, 후속 생산분 QC-03 강화검사 플래그.
05Business Rules업무 규칙
RuleDescriptionAction
BR-QC-05
QC-02 trigger is mandatory at MD-08 SampleInterval; cannot be skipped or deferred > 15 minutes.
검사 주기 도달 시 QC-02 필수 발행. 15분 초과 지연 불가.
Force
BR-QC-06
FAIL during in-process auto-halts the affected WO; resume requires supervisor PIN + CorrectiveAction.
FAIL 시 해당 WO 자동 정지, PIN + 시정조치 입력 후 재가동.
Halt
BR-QC-07
Critical-defect FAIL auto-quarantines the entire shift cumulative output for that WO.
Critical 불량 시 해당 교대 누적 양품 전체 자동 격리.
Multi
BR-QC-03
3 FAILs (same WO + same defect code) within 30 days → auto-recommend CAPA (QC-06).
30일 내 동일 WO·동일 불량 FAIL 3회 → CAPA 자동 권고.
Alert
BR-QC-08
All in-process inspection records retained ≥ 5 years (including photos and per-sample data).
공정검사 기록·사진·샘플 데이터 5년 이상 보관.
Retain
