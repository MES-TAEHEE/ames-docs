01Overview개요
EN · Description MD_Item is the single registry of every item in A-MES — raw material, fabric, powder, paint, semi-finished and finished good. It is the most-referenced master table: PP, WH, INJ, IMG, PNT, QC and FG all hold foreign keys to ItemNo. The table carries identity (code, name, type), the default unit of measure, the BOP routing type, stock levels, standard cost and the customer item-number mapping for SAV / GEO. Items are never physically deleted — discontinued items are set ActiveFlag = FALSE to preserve historical traceability.
KO · 설명 MD_Item은 A-MES 전 품목의 단일 등록 테이블 — 원자재·원단·분체·도료·반제품·완제품. 가장 많이 참조되는 마스터로 PP·WH·INJ·IMG·PNT·QC·FG가 모두 ItemNo를 외래키로 보유한다. 식별 정보(코드·명·유형), 기본 단위, BOP 라우팅 유형, 재고 수준, 표준 원가, SAV/GEO 고객사 품번 매핑을 담는다. 품목은 물리 삭제하지 않으며 — 단종 품목은 ActiveFlag = FALSE로 처리해 과거 추적성을 보존한다.

05Validation & Business Rules검증 + 업무 규칙
RuleDescriptionAction
VR-Item-01
ItemNo follows the pattern {TypePrefix}-{6-digit seq}; prefix must match ItemType.
ItemNo는 {유형접두어}-{6자리} 패턴, 접두어는 ItemType과 일치.
Format
VR-Item-02
SEMI / FINISHED items must have a RoutingType; RAW / FABRIC / POWDER / PAINT must not.
SEMI·FINISHED는 RoutingType 필수, 원자재류는 NULL.
Validation
BR-MD-22
FINISHED items must have CustItemNoSAV / GEO mapped before FG-06 can issue a delivery note.
완제품은 고객 품번 매핑 후 FG-06 거래명세서 발행.
Validation
BR-MD-23
An item referenced by any transaction cannot be deleted — set ActiveFlag = FALSE instead.
트랜잭션 참조 품목은 삭제 불가 — ActiveFlag=FALSE 처리.
Block
BR-MD-24
Every INSERT / UPDATE requires an approval workflow (creator ≠ approver) + SYS-05 audit log.
전 생성·수정은 승인 워크플로 + SYS-05 감사 로그.
Audit
